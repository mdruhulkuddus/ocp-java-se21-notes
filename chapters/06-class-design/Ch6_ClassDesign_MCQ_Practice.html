Q1INHERITANCE

What is the result of compiling the following?

final class Mammal {}
class Rhinoceros extends Mammal {}
  • A. It compiles successfully.
  • B. It compiles but fails at runtime.
  • C. Mammal does not compile.
  • D. Rhinoceros does not compile.
Show Answer
D
A final class cannot be extended. Mammal itself is a legal declaration, so the compiler error lands on the Rhinoceros line that tries to extend it.

Q4this

What is printed?

public class Flamingo {
   private String color;
   public Flamingo(String color) {
      color = color;
   }
   public static void main(String[] a) {
      System.out.println(new Flamingo("pink").color);
   }
}
  • A. pink
  • B. null
  • C. An empty string
  • D. It does not compile.
Show Answer
B
color = color assigns the parameter to itself; the instance field is never touched, so it keeps its default of null. Writing this.color = color would have fixed it.
Q5this

What does this program print?

public class Duck {
   private int length;
   private int height;
   private String color;
   public Duck(int length, int theHeight, String color) {
      length = this.length;
      height = theHeight;
      this.color = "white";
   }
   public static void main(String[] a) {
      Duck d = new Duck(5, 2, "yellow");
      System.out.println(d.length + " " + d.height + " " + d.color);
   }
}
  • A. 5 2 yellow
  • B. 0 2 white
  • C. 0 2 yellow
  • D. 5 2 white
Show Answer
B
length = this.length is backwards: it copies the field's default (0) into the parameter, leaving the field 0. height = theHeight correctly sets 2, and this.color = "white" ignores the parameter. Result: 0 2 white.
Q6super

What does getSpeed() return when called on a Crocodile instance?

class Reptile {
   protected int speed = 10;
}
class Crocodile extends Reptile {
   protected int speed = 20;
   public int getSpeed() { return super.speed; }
}
  • A. 10
  • B. 20
  • C. 0
  • D. It does not compile.
Show Answer
A
Variables are hidden, not overridden. super.speed explicitly reaches the parent's field, which is 10. Plain speed would have returned 20.
Q7CONSTRUCTORS

public class Bunny {

  • B. It compiles; bunny() is a method.

Q8CONSTRUCTORS

Is new Bunny() legal given this class?

  • B. Yes — Bunny() is a method and a default constructor is supplied.

Because it declares a return type (void), Bunny() is an ordinary method, not a constructor. Since no constructor is declared, the compiler inserts a default no-arg one, so new Bunny() works.

Q12CONSTRUCTORS

What is the result?

public class Gopher {
   public Gopher() { this(5); }
   public Gopher(int holes) { this(); }
}
  • A. It compiles but causes infinite recursion at runtime.
  • B. It does not compile.
  • C. It compiles and runs normally.
  • D. It throws StackOverflowError at runtime.

B
The compiler detects the cyclic this() calls (each constructor delegates to the other) and rejects the code at compile time — it never gets a chance to recurse at runtime.

Q14INIT ORDER

What is printed?

public class Hippo {
   static { System.out.print("A"); }
   static int x = init();
   static int init() { System.out.print("B"); return 1; }
   public static void main(String[] a) { System.out.print("C"); }
}
  • A. ABC
  • B. BAC
  • C. BCA
  • D. CAB
Show Answer
A
Static initializers and static variable initializers run top-to-bottom when the class loads, before main(). The block prints A, then x = init() prints B, then main prints C.

Q16INIT ORDER

What is the result?

public class MouseHouse {
   private final String type;
   public MouseHouse() {}
}
  • A. It compiles; type is null.
  • B. It compiles; type is an empty string.
  • C. It does not compile.
  • D. It compiles but throws at runtime.
Show Answer
C
A final instance field must be assigned exactly once by the time the constructor finishes — at its declaration, in an instance initializer, or in the constructor. It is never assigned here, so the code does not compile.
Q17OVERRIDING

What happens when new Kangaroo().getAverageWeight() is called?

class Marsupial {
   public double getAverageWeight() { return 50; }
}
class Kangaroo extends Marsupial {
   public double getAverageWeight() {
      return getAverageWeight() + 20;
   }
}
  • A. Returns 70.0
  • B. Returns 50.0
  • C. Does not compile
  • D. Throws StackOverflowError at runtime
Show Answer
D
The unqualified getAverageWeight() call inside the override invokes itself, recursing forever — a StackOverflowError at runtime. Using super.getAverageWeight() + 20 would correctly return 70.0.
Q18OVERRIDING

Given FileNotFoundException is a subclass of IOException, what is the result?

class Reptile {
   protected void exitShell() throws FileNotFoundException {}
}
class Tortoise extends Reptile {
   protected void exitShell() throws IOException {}
}
  • A. It compiles.
  • B. Tortoise does not compile.
  • C. Reptile does not compile.
  • D. It compiles but fails at runtime.
Show Answer
B
An override may not throw a new or broader checked exception. The parent throws FileNotFoundException; the child throws the broader IOException, so Tortoise does not compile. Narrowing (parent IOException → child FileNotFoundException) would be fine.
Q19OVERRIDING

Choose all that apply. Each child method attempts to override the parent method directly above it. Which overrides compile?

// A  parent: public CharSequence getName()
       child:  public String getName()
// B  parent: public String getColor()
       child:  public CharSequence getColor()
// C  parent: public void run()
       child:  public void run()
// D  parent: public void stop()
       child:  public int stop()
  • A. Override A
  • B. Override B
  • C. Override C
  • D. Override D
Show Answer
A · C
Return types must be covariant (same type or a subtype). A is fine because String is a subtype of CharSequence; C is an identical void/void match. B widens the return type (illegal) and D changes void to int (void is only covariant with void), so both fail.
Q20HIDING

What is printed?

class Bear {
   public static void eat() { System.out.println("Bear is eating"); }
}
class Panda extends Bear {
   public static void eat() { System.out.println("Panda is chewing"); }
}
public class Test {
   public static void main(String[] a) {
      Bear b = new Panda();
      b.eat();
   }
}
  • A. Bear is eating
  • B. Panda is chewing
  • C. It does not compile.
  • D. It throws at runtime.
Show Answer
A
static methods are hidden, not overridden, and are resolved by the reference type at compile time. The reference is Bear, so Bear.eat() runs — Bear is eating.

Q26ABSTRACT

Choose all that apply. Which snippets compile?

// A
abstract class Animal { abstract String getName(); }
class Walrus extends Animal {}

// B
abstract class Mammal { abstract void showHorn(); abstract void eatLeaf(); }
abstract class Rhino extends Mammal { void showHorn() {} }
class BlackRhino extends Rhino { void eatLeaf() {} }

// C
abstract class Llama {}
  • A. Snippet A
  • B. Snippet B
  • C. Snippet C
  • D. None of them compile.
Show Answer
B · C
Only the first concrete subclass must implement all inherited abstract methods. In A, Walrus is concrete but leaves getName() unimplemented — it fails. In B the work is split across an intermediate abstract Rhino, and BlackRhino finishes it. C is a legal abstract class.
Q27ABSTRACT

What is printed by new Platypus();?

abstract class Mammal {
   abstract String chew();
   public Mammal() { System.out.println(chew()); }
}
class Platypus extends Mammal {
   String chew() { return "yummy!"; }
   public static void main(String[] a) { new Platypus(); }
}
  • A. yummy!
  • B. null
  • C. It does not compile (abstract classes can't have constructors).
  • D. It throws at runtime.
Show Answer

Abstract classes can and do have constructors. Platypus's implicit constructor calls super(), which runs chew(); the call dispatches to the overridden version, printing yummy!.
Q28IMMUTABILITY

Which statement about this class is correct?

public final class Animal {
   private final List<String> foods;
   public Animal(List<String> foods) {
      this.foods = foods;
   }
   public List<String> getFoods() { return foods; }
}
  • A. It is immutable because the field is private final.
  • B. It is not immutable; callers can still mutate the list's contents.
  • C. It does not compile.
  • D. It is immutable because the class is final.
Show Answer
B
final stops the reference from being reassigned, but the referenced List is still mutable. The caller who passed it in keeps a reference, and getFoods() hands out the same list — both can add() or clear() it. True immutability needs a defensive copy on the way in (new ArrayList<>(foods)) and no leaking of the internal reference on the way out.

Quick Revision — traps to memorize

  • private members are never inherited; top-level classes can't be protected/private.
  • this() and super() must be the first statement; cyclic this() is a compile error.
  • No default constructor is generated once any constructor is declared.
  • A parent with only a parameterized constructor breaks a child's implicit super().
  • Static initializers/fields run top-to-bottom at class load, before main(); every final field must be assigned exactly once.
  • Overriding = runtime dispatch on the object; hiding (static methods & all fields) = compile-time dispatch on the reference type.
  • Override rules: same signature, ≥ accessibility, no broader checked exception, covariant return (void only with void).
  • abstract can't combine with final, private, or static; only the first concrete subclass must implement all abstract methods.
  • Abstract classes have constructors, and a constructor may call an overridden abstract method.
  • final on a collection field protects the reference, not the contents — defensive-copy in and out for true immutability.
