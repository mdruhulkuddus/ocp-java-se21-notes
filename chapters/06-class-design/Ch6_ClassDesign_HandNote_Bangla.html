SECTION 01

Understanding Inheritance

Inheritance-এর মাধ্যমে একটি subclass (child) তার superclass-এর (parent) কিছু member — field ও method — স্বয়ংক্রিয়ভাবে পেয়ে যায়। এটি ঘোষণা করতে হয় extends keyword দিয়ে। তবে শর্ত হলো, parent ক্লাসটি final হওয়া যাবে না।

কোন member-গুলো inherit হয়

Access levelsubclass-এ inherit হয়?
publicহ্যাঁ — সবসময়
protectedহ্যাঁ — সবসময়
package (default)শুধু একই package হলে
privateনা — কখনোই inherit হয় না
 
public class BigCat {
    protected double size;
}
public class Jaguar extends BigCat {
    public Jaguar() { size = 10.2; }       // OK: size protected, তাই inherit হয়েছে
}
public class Spider {
    public void printDetails() {
        System.out.println(size);            // DOES NOT COMPILE — Spider তো size inherit করেনি
    }
}

Inheritance transitive

যদি X extends Y এবং Y extends Z হয়, তাহলে X হলো Z-এর descendant। আর direct subclass হলো সেটাই যেটা সরাসরি extends লেখে — অর্থাৎ X শুধু Y-এর direct subclass, Z-এর নয়।

⚠ EXAM TRAP

কোনো final ক্লাসকে extend করতে দেখলে কোডটা সঙ্গে সঙ্গে compile হবে না — এটা চোখে পড়লেই উত্তর দিয়ে দাও।

public class Mammal {}
public final class Rhinoceros extends Mammal {}
public class Clara extends Rhinoceros {}   // DOES NOT COMPILE — Rhinoceros final

Single inheritance (Java)

  • প্রতিটি class-এর ঠিক একটি direct parent।
  • একাধিক level চলে (X→Y→Z)।
  • এক parent-এর অনেক child থাকতে পারে।

Multiple inheritance (Java-তে নেই)

  • একাধিক direct parent — অনুমোদিত নয়।
  • কোন parent থেকে inherit করবে — তাতে ambiguity তৈরি হয়।
  • একমাত্র ব্যতিক্রম: একটি class একাধিক interface implement করতে পারে (অধ্যায় ৭)।

Object থেকে inheritance

প্রতিটি class-ই java.lang.Object থেকে inherit করে। Object-ই একমাত্র class যার কোনো parent নেই। তোমার class যদি কিছু extend না করে, compiler নিজে থেকেই extends java.lang.Object যোগ করে দেয়। তাই নিচের দুটো equivalent:

public class Zoo { }
public class Zoo extends java.lang.Object { }   // equivalent
★ REMEMBER

যখন তুমি কোনো existing class extend করো, Java আবার Object যোগ করে না — chain-এর মাধ্যমে তুমি এমনিতেই Object পেয়ে যাও। primitive type (int, boolean) class নয়, তাই তারা Object inherit করে না; তবে তাদের wrapper class করে (autoboxing-এর মাধ্যমে)। toString() ও equals() method Object-এ থাকে, তাই সব class-এই এগুলো পাওয়া যায়।

SECTION 02

Creating Classes

একটি class extend করা

// Animal.java
public class Animal {
    private int age;                  // private — সরাসরি inherit হয় না
    protected String name;          // protected — inherit হয় ও সরাসরি accessible
    public int getAge() { return age; }
    public void setAge(int newAge) { age = newAge; }
}
// Lion.java
public class Lion extends Animal {
    protected void setProperties(int age, String n) { setAge(age); name = n; }
    public void roar() {
        System.out.print(name + ", age " + getAge() + ", says: Roar!");
    }
}

lion.setProperties(3,"kion")-এর পর lion.roar() প্রিন্ট করে kion, age 3, says: Roar!। age হলো private, তাই শুধু setAge()/getAge() দিয়ে পৌঁছানো যায়; name হলো protected, তাই সরাসরি accessible। Lion-এর ভেতরে age সরাসরি ব্যবহার করলে compile হবে না।

Class access modifier (top-level)

Top-level class-এর নিয়ম

  • একটি .java ফাইলে সর্বোচ্চ একটি public top-level class থাকতে পারে।
  • যত খুশি package-access class রাখা যায়, যেকোনো order-এ — এমনকি একটা public class না থাকলেও চলে।
  • Top-level class শুধু public বা package হতে পারে — protected/private এখানে illegal (এগুলো শুধু nested class-এ চলে, অধ্যায় ৭)।
protected class ClownFish {}    // DOES NOT COMPILE
private class BlueTang {}       // DOES NOT COMPILE

this reference

parameter আর instance variable-এর নাম এক হয়ে গেলে Java সবচেয়ে granular (সংকীর্ণতম) scope ব্যবহার করে। এই trap-টা লক্ষ করো:

public class Flamingo {
    private String color = null;
    public void setColor(String color) {
        color = color;                 // parameter-কে নিজের ভেতরেই assign — instance var অপরিবর্তিত!
    }
}
// f.setColor("PINK"); System.out.print(f.color);  => null

সমাধান: this.color = color; → তখন প্রিন্ট হবে PINK। this reference current instance-কে নির্দেশ করে এবং যেকোনো instance method, constructor বা instance initializer-এ কাজ করে — কিন্তু static method বা static initializer-এ নয় (সেখানে কোনো implicit instance নেই)।

TRACE — Duck উদাহরণ → প্রিন্ট হয় 0 2 white

  • length = this.length; — উল্টো! instance length (0) parameter-এ assign হয়; instance 0-ই থাকে।
  • height = theHeight; — ভিন্ন নাম, this লাগে না; height হয় 2।
  • this.color = "white"; — ঠিক আছে; নাম conflict না থাকলেও this. লেখা legal।

super reference

একই নামের variable/method parent ও child দুই জায়গায় থাকলে instance-এ দুটো আলাদা copy থাকে। this খোঁজে current → inherited; super খোঁজে শুধু inherited (current class-এর member বাদ দিয়ে)।

public class Reptile { protected int speed = 10; }
public class Crocodile extends Reptile {
    protected int speed = 20;
    public int getSpeed() { return speed; }        // => 20 (this.speed)
    // return super.speed;  দিলে => 10
}
⚠ EXAM TRAP — Insect / Beetle

এই প্রোগ্রামটা মাত্র একটা লাইনের জন্য compile হবে না:

class Insect { protected int numberOfLegs = 4; String label = "buggy"; }
public class Beetle extends Insect {
    protected int numberOfLegs = 6;
    short age = 3;
    public void printData() {
        System.out.println(this.label);    // buggy  (inherited)
        System.out.println(super.label);   // buggy  (inherited)
        System.out.println(this.age);      // 3      (current class)
        System.out.println(super.age);     // DOES NOT COMPILE — age inherited নয়
        System.out.println(numberOfLegs);   // 6      (সংকীর্ণতম scope = Beetle)
    }
}

this = current + inherited; super = শুধু inherited। age শুধু Beetle-এ আছে, তাই super.age illegal।

SECTION 03

Declaring Constructors

একটি constructor-এর নাম হুবহু class-এর নামের সমান এবং কোনো return type থাকে না (void-ও নয়)। new দিয়ে instance তৈরি করলে এটি চলে।

public class Bunny {
    public bunny() {}          // DOES NOT COMPILE — ছোট হাতের নাম = method, কিন্তু return type নেই
    public void Bunny() {}     // বৈধ METHOD (return type আছে) — constructor নয়
}
⚠ EXAM TRAP

Constructor parameter যেকোনো class, array, primitive বা generic হতে পারে — কিন্তু কখনোই var নয়: public Bonobo(var food){} compile হয় না।

Default constructor

Default no-arg constructor

  • class যদি একটাও constructor না লেখে, compiler একটি public no-argument constructor যোগ করে।
  • এটি compile time-এ তৈরি হয় — .java ফাইলে দেখা যায় না, শুধু .class-এ।
  • তুমি যদি একটিও constructor লেখো (এমনকি private হলেও), default যোগ হয় না।
Classconstructor লেখা আছে?default no-arg পায়?
Rabbit1 {}নাহ্যাঁ
Rabbit2 + Rabbit2()হ্যাঁনা
Rabbit3 + Rabbit3(boolean)হ্যাঁনা
Rabbit4 + private Rabbit4()হ্যাঁ (private)না

অন্য class থেকে new Rabbit4() করলে compile হয় না — constructor-টা private। শুধুই private constructor-ওয়ালা class বাইরে থেকে instantiate করা যায় না এবং default পায় না (static-only utility class-এর জন্য কাজে লাগে)।

Overloaded constructor — this() দিয়ে call

Constructor-রা একে অপরকে call করতে পারে, কিন্তু শুধু this(...) দিয়ে — নাম দিয়ে নয়, new দিয়েও নয় (তাতে একটা বাড়তি, ফেলে দেওয়া object তৈরি হবে)।

public Hamster(int weight) {
    Hamster(weight, "brown");        // DOES NOT COMPILE — constructor method-এর মতো call করা যায় না
}
public Hamster(int weight) {
    new Hamster(weight, "brown");    // compile হয়, কিন্তু বাড়তি object তৈরি হয় যা পরে discard হয়
}
public Hamster(int weight) {
    this(weight, "brown");          // সঠিক — একই instance
}

this বনাম this()

  • this → current instance-এর reference।
  • this() → একই class-এর আরেকটি constructor-এ call।

super বনাম super()

  • super → parent-এর member reference।
  • super() → parent constructor-এ call।

মুখস্থ রাখার constructor নিয়ম

  1. একটি class-এ অনেক overloaded constructor থাকতে পারে, প্রতিটির signature আলাদা।
  2. প্রতিটি constructor-এর প্রথম লাইন হলো this() অথবা super()-এ call।
  3. this()/super() থাকলে তা অবশ্যই প্রথম statement হবে (তাই প্রতি constructor-এ এমন একটাই call)।
  4. কোনোটাই না লিখলে compiler প্রথম লাইনে no-arg super() বসিয়ে দেয়।
  5. Cyclic constructor call নিষিদ্ধ।
  6. super() সবসময় সবচেয়ে কাছের direct parent-কে বোঝায়।
⚠ EXAM TRAP — this()-এর আগে statement 
public Hamster(int weight) {
    System.out.println("chew");      // একটা statement...
    this(weight, "brown");          // DOES NOT COMPILE — this() প্রথমে হতে হবে
}

Comment statement হিসেবে গণ্য হয় না, তাই this()-এর আগে comment থাকলে সমস্যা নেই। Cyclic call-ও fail করে: Gopher(int){ this(5); } নিজেকেই call করছে → compile হয় না।

Parent constructor — super() দিয়ে call

public class Zoo {
    public Zoo() {
        System.out.println("Zoo created");
        super();                       // DOES NOT COMPILE — super() প্রথমে হতে হবে
    }
}

Child constructor parent-এর যেকোনো accessible constructor call করতে পারে — নিজের signature-এর সঙ্গে মেলানোর দরকার নেই। argument-সহ super()-ও “super call” হিসেবেই গণ্য।

⚠ EXAM TRAP — parent-এ no-arg constructor নেই

Parent যদি শুধু argument-সহ constructor রাখে এবং কোনো no-arg constructor না রাখে, তাহলে auto-inserted super()-এর call করার মতো কিছু থাকে না:

public class Mammal { public Mammal(int age) {} }
public class Seal extends Mammal {}              // DOES NOT COMPILE
public class Elephant extends Mammal {
    public Elephant() {}                          // DOES NOT COMPILE — implicit super()-এর target নেই
}

সমাধান: child constructor-এ explicit super(6); দাও। (Parent-এ no-arg না থাকলেও subclass-এ no-arg constructor থাকতে পারে — শুধু একটা existing parent constructor call করলেই হলো।)

SECTION 04

Initializing Objects

Class initialization-এর order

Initialize Class X (সর্বোচ্চ একবার হয়)

  1. প্রথমে X-এর superclass initialize হয়।
  2. সব static variable declaration source order-এ process হয়।
  3. সব static initializer source order-এ process হয়।

প্রথম ব্যবহারের আগে class load হয়; main()-যুক্ত class main() চলার আগেই load হয়। main() যদি Hippo-র ভেতরে থাকে (যেটা Animal extend করে), প্রোগ্রাম প্রিন্ট করে ABC — ঠিক একবার: class init থেকে A+B, তারপর main() থেকে C, যদিও তিনটি instance তৈরি হয়। আর Hippo অন্য class থেকে ব্যবহার করলে সম্ভবত CAB দেখবে (দরকার পড়লেই load হয়)।

final field

✓ KEY RULE

constructor শেষ হওয়ার মধ্যে প্রতিটি final instance variable-কে ঠিক একবার value পেতে হবে — declaration-এ, instance initializer-এ, অথবা constructor-এ। default value শুধু non-final field-এ প্রযোজ্য।

public class MouseHouse {
    private final int volume;
    private final String type;
    { this.volume = 10; }                // instance initializer volume set করে
    public MouseHouse(String type) { this.type = type; }   // OK
    public MouseHouse() {                  // DOES NOT COMPILE
        this.volume = 2;                   // volume আগেই set; আবার type কখনো set হয়নি
    }
}

প্রতিটি constructor আলাদাভাবে যাচাই হয়। no-arg version দুইভাবে fail করে: type কখনো set করে না, আর already-final volume আবার assign করে। this(null) দিয়ে delegate করলে ঠিক হয়ে যায় — final reference-এ null একবারের বৈধ assignment।

Instance initialization-এর order

Initialize Instance of X

  1. Class X initialize করো (আগে না হয়ে থাকলে)।
  2. X-এর superclass instance initialize করো।
  3. instance variable declaration source order-এ process করো।
  4. instance initializer source order-এ process করো।
  5. constructor চালাও (this()-chained constructor সহ)।

সংক্ষেপে: declaration → initializer → constructor, superclass আগে, subclass পরে।

SECTION 05

Inheriting Members

Method override করা

Override = subclass কোনো inherited method-এর জন্য নতুন implementation দেয়, same signature (name + parameter) ও compatible return type রেখে। signature-এ return type, access modifier, specifier ও exception ধরা হয় না।

public class Kangaroo extends Marsupial {
    public double getAverageWeight() {
        return super.getAverageWeight() + 20;   // Marsupial=50.0, Kangaroo=70.0
    }
}
// super ছাড়া নিজের ভেতরেই getAverageWeight() ডাকলে => StackOverflowError (runtime-এ)

Override-এর ৪টি নিয়ম

  1. parent method-এর সঙ্গে same signature।
  2. parent-এর চেয়ে কম restrictive নয় (অন্তত সমান accessible)।
  3. কোনো নতুন বা broader checked exception declare করা যাবে না।
  4. return type same বা subtype হতে হবে (covariant)।

★ REMEMBER

Covariance টেস্ট: cast ছাড়া B-কে A reference-এ assign করা যায় কি? পারলে B, A-র সঙ্গে covariant। String হলো CharSequence-এর subtype, তাই String→CharSequence চলে, উল্টোটা নয়। return type void হলে override-ও void হতে হবে। Override-এ নতুন unchecked exception যোগ করা যায় অবাধে।

private method redeclare করা

private method override করা যায় না (এটা inherit হয় না)। child একই/ভিন্ন signature ও return type দিয়ে এটাকে redeclare করতে পারে — সেটা একটা নতুন, স্বাধীন method, কোনো override নিয়ম খাটে না।

public class Beetle { private String getSize() { return "Undefined"; } }
public class RhinocerosBeetle extends Beetle {
    private int getSize() { return 5; }   // compile হয় — override নয় (parent-এরটা private)
}

static method hiding

✓ KEY RULE

Static method override হয় না — hide হয়। ৪টি override নিয়মই খাটে, সঙ্গে ৫ম: parent method static হলে child method-ও static হতে হবে। mismatch (একটা static, একটা নয়) ⇒ compile হয় না।

public class Bear {
    public static void sneeze() {}
    public void hibernate() {}
    public static void laugh() {}
}
public class SunBear extends Bear {
    public void sneeze() {}            // DOES NOT COMPILE — parent static, child instance
    public static void hibernate() {}  // DOES NOT COMPILE — parent instance, child static
    protected static void laugh() {}   // DOES NOT COMPILE — বেশি restrictive (নিয়ম #২)
}

variable hiding

Java variable override করে না — শুধু hide করে। একই নামের child variable দুটো আলাদা copy তৈরি করে; কোনটা দেখবে তা নির্ভর করে reference type-এর ওপর।

class Carnivore { protected boolean hasFur = false; }
public class Meerkat extends Carnivore {
    protected boolean hasFur = true;
    public static void main(String[] args) {
        Meerkat m = new Meerkat();
        Carnivore c = m;
        System.out.println(m.hasFur);   // true  (Meerkat reference)
        System.out.println(c.hasFur);   // false (Carnivore reference)
    }
}
★ REMEMBER

Method override সব reference-এর জন্য সেটাকে replace করে (super ছাড়া)। method/variable hiding শুধু তখনই খাটে যখন child reference type ব্যবহৃত হয়।

final method

final method override করা যায় না — আর final static method hide-ও করা যায় না। child final লিখুক বা না লিখুক, তাতে কিছু যায় আসে না। (parent method private হলে child version বৈধ redeclaration হয়ে যেত।)

SECTION 06

Creating Abstract Classes

একটি abstract class (abstract দিয়ে চিহ্নিত) সরাসরি instantiate করা যায় না এবং এতে abstract method থাকতে পারে — যেগুলো abstract দিয়ে ঘোষিত, body-হীন, এবং subclass-কে implement করতে বাধ্য করে।

public abstract class Canine {
    public abstract String getSound();          // body নেই, ; দিয়ে শেষ
    public void bark() { System.out.println(getSound()); }
}
// Canine w = new Fox(); w.bark();  => Squeak!

Abstract নিয়ম

  • শুধু instance method abstract হতে পারে — variable, constructor বা static method নয়।
  • abstract class-এ শূন্য বা একাধিক abstract method থাকতে পারে; non-abstract class-এ একটিও নয়।
  • প্রথম concrete subclass-কে সব inherited abstract method implement করতে হবে।
  • abstract method override-এ স্বাভাবিক override নিয়ম খাটে।
  • concrete subclass instantiate হলে তবেই constructor দিয়ে abstract class initialize হয়।

✓ KEY RULE

abstract class-এ কোনো abstract method না থাকলেও চলে (যেমন abstract class Llama { void chew(){} } compile হয়)। তবু এটা instantiate করা যায় না।

ভুল abstract declaration চেনা

public abstract class Turtle {
    public abstract long eat()            // DOES NOT COMPILE — semicolon নেই
    public abstract void swim() {};       // DOES NOT COMPILE — abstract + body
    public abstract int getAge() { return 10; }  // DOES NOT COMPILE — abstract + body
    public abstract void sleep;          // DOES NOT COMPILE — () নেই
    public void goInShell();              // DOES NOT COMPILE — abstract নয়, তাই body লাগবে
}

abstract modifier access modifier-এর আগে বা পরে বসতে পারে (abstract public class Tiger ঠিক আছে), কিন্তু class keyword-এর পরে নয় (public class abstract Bear fail করে) এবং method-এর return type-এর পরেও নয়।

অবৈধ modifier সংমিশ্রণ

⚠ EXAM TRAP — প্রথম concrete subclass

abstract method একাধিক abstract class-এর মধ্য দিয়ে নিচে নামতে পারে। প্রথম concrete subclass-কে তখনো-abstract সব method implement করতে হবে। মাঝের কোনো abstract class একটা method implement করলে concrete class-কে শুধু বাকিগুলো দিতে হয়। কিন্তু মাঝের একটা class-কে concrete বানালে সেটাকে হঠাৎ সব বাকি abstract method দিতে হয়।

SECTION 07

Creating Immutable Objects

একটি immutable object তৈরি হওয়ার পর তার state বদলানো যায় না — security-র জন্য এবং একাধিক thread-এ নিরাপদে share করার জন্য দুর্দান্ত।

প্রচলিত immutability কৌশল

  1. class-কে final করো অথবা সব constructor private করো।
  2. সব instance variable private ও final করো।
  3. কোনো setter method রেখো না।
  4. referenced mutable object-কে বদলাতে দিও না।
  5. constructor দিয়ে সব field set করো, দরকার হলে copy বানিয়ে।
⚠ EXAM TRAP — mutable reference leak

নিয়ম ১–৩ মানলেও ভেতরের list সরাসরি return করলে immutability ভেঙে যায়:

public List<String> getFavoriteFoods() { return favoriteFoods; }  // caller clear()/add() করতে পারবে!

নিরাপদ বিকল্প: delegate method (getFavoriteFoodsItem(int), getFavoriteFoodsCount()) অথবা copy-on-read getter: return new ArrayList<>(this.favoriteFoods);।

Defensive copy (নিয়ম ৫)

caller যদি data সরবরাহ করে, তার reference সংরক্ষণ করলে পরে সে সেটা mutate করতে পারে। তাই constructor-এ copy বানাও:

public Animal(List<String> favoriteFoods) {
    if (favoriteFoods == null || favoriteFoods.size() == 0)
        throw new RuntimeException("favoriteFoods is required");
    this.favoriteFoods = new ArrayList<String>(favoriteFoods);  // defensive copy
}
★ REMEMBER

defensive copy ছাড়া caller তার নিজের reference রেখে পরে clear()/add() করলে “immutable” object-ও বদলে যায় — এটাই ক্লাসিক exam trap।

RECAP

সংক্ষিপ্ত রিভিউ

Inheritance

  • extends = subclass; parent final হতে পারে না। public/protected সবসময় inherit, package শুধু same-package, private কখনোই নয়।
  • সব class java.lang.Object inherit করে; primitive করে না।

Creating classes

  • প্রতি ফাইলে একটি public top-level class; top-level protected/private হয় না।
  • this = current+inherited; super = শুধু inherited; static context-এ কোনোটাই নেই (এমনকি this-ও নয়)।

Constructors

  • নাম class-এর সমান, return type নেই, var param নেই। default no-arg শুধু কোনো constructor না থাকলে।
  • প্রথম লাইন this() বা super() (নাহলে compiler super() বসায়); প্রথম statement হতে হবে; cycle নিষেধ।

Initialization order

  • Class: superclass → static var → static initializer (একবার)। Instance: superclass instance → instance var → instance initializer → constructor।
  • প্রতিটি final field constructor শেষ হওয়ার মধ্যে ঠিক একবার assign।

Overriding / hiding

  • Override: same signature, ≥ access, নতুন/broader checked exception নয়, covariant return।
  • static → hidden (static মিলতে হবে); variable → hidden (reference type ঠিক করে); final override/hide হয় না।

Abstract classes

  • instantiate হয় না; abstract method-এ body নেই, ; দিয়ে শেষ। প্রথম concrete subclass সব implement করে। abstract+final/private/static নিষেধ।

Immutable objects

  • final class বা private constructor, private final field, setter নেই, mutable leak নেই, in/out defensive copy।
