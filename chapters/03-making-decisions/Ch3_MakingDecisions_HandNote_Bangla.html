SECTION 01

Decision-Making: if / else ও Pattern Matching

একটা if statement একটা block তখনই চালায় যখন একটা boolean expression true হয়। else (ও chained else if) বিকল্পগুলো সামলায়। Java উপর থেকে নিচে branch মূল্যায়ন করে আর প্রথম true-তেই থামে।

⚠ EXAM TRAP — indentation মিথ্যা বলে; brace trace করুন 
if (hourOfDay < 11)
  System.out.println("Good Morning");
  morningGreetingCount++;  // runs ALWAYS — not part of the if

brace ছাড়া একটা if শুধু পরের একটামাত্র statement নিয়ন্ত্রণ করে। Java-তে tab শুধু whitespace। সবসময় open/close brace trace করুন, indentation উপেক্ষা করে।

⚠ EXAM TRAP — if-এ boolean লাগে 
int hourOfDay = 1;
if (hourOfDay) { ... }  // DOES NOT COMPILE (0 and 1 are not boolean)

instanceof দিয়ে pattern matching

Pattern matching type-এর পরে একটা pattern variable যোগ করে — type check আর cast এক ধাপে মিলিয়ে দেয়, আর cast শুধু তখনই চলে যখন instanceof true, ফলে ClassCastException এড়ানো যায়।

// old, verbose
if (number instanceof Integer) {
  Integer data = (Integer) number;
  System.out.print(data.compareTo(5));
}
// pattern matching
if (number instanceof Integer data)
  System.out.print(data.compareTo(5));

একটা optional guard (&& দিয়ে boolean expression) আরও filter করতে পারে, যে line-এ pattern variable declare হয় সেই line-এই ব্যবহার করে:

if (number instanceof Integer data && data.compareTo(5) > 0)
  System.out.print(data);
✓ KEY RULE — null ও reassignment

instanceof (ও pattern matching) null-এর জন্য সবসময় false। Pattern variable পুনরায় assign করা bad practice; final দিলে compile হবে না (if (n instanceof final Integer data) { data = 10; } → compile হয় না)। Pattern type-টা হতে হবে একই, subtype, বা supertype — অসম্ভব type যেমন Number n instanceof String s compile হয় না।

Flow scoping

একটা pattern variable শুধু সেখানেই scope-এ থাকে যেখানে compiler তার type প্রমাণ করতে পারে — এটা সাধারণ hierarchical scoping-এর মতো নয়।

⚠ EXAM TRAP — OR scope ভাঙে 
if (number instanceof Integer data || data.compareTo(5) > 0)  // DOES NOT COMPILE

||-এর ক্ষেত্রে, input যদি Integer না হয়, তাহলে data undefined — তাই ব্যবহার করা যায় না। (&& দিয়ে ঠিক আছে।)

কিন্তু একটা pattern variable if-এর বাইরেও বেরোতে পারে যখন flow type নিশ্চিত করে। এটা compile হয়, কারণ শেষ line-এ পৌঁছানো প্রমাণ করে number একটা Integer:

void printOnlyIntegers(Number number) {
  if (!(number instanceof Integer data))
    return;
  System.out.print(data.intValue());  // in scope here
}
SECTION 02

switch Statement

একটা switch একটা value মূল্যায়ন করে এক বা একাধিক branch-এ flow পাঠায়। একটা statement কোনো value ফেরায় না (expression-এর বিপরীতে)। দুটোই switch (var) দিয়ে শুরু হয় আর zero বা তার বেশি case clause support করে; প্রতিটা clause-এ comma দিয়ে আলাদা করা value, একটা separator (: বা ->), তারপর একটা expression বা block থাকে।

Supported switch type

★ REMEMBER — অনুমোদিত switch variable type

int/Integer, byte/Byte, short/Short, char/Character, String, enum value, যেকোনো object type (pattern matching সহ), এবং var যদি এগুলোর একটায় resolve হয়। সমর্থিত নয়: boolean, long, float, double।

গ্রহণযোগ্য case value

case value হতে হবে compile-time constant: literal, enum constant, বা literal দিয়ে initialize করা final variable। Method আর non-final variable চলে না।

final int bananas = 1;
int apples = 2;
final int cookies = getCookies();
switch (numberOfAnimals) {
  case bananas:        // OK (final + literal)
  case apples:         // DOES NOT COMPILE (not final)
  case getCookies():   // DOES NOT COMPILE (method)
  case cookies:        // DOES NOT COMPILE (final from a method, not a literal)
  case 3 * 5:         // OK (constant expression)
}

case type-টা switch variable-এর type-এর সাথে মিলতে হবে। একটা int switch-এ String case compile হয় না। যদি একটা switch-এ -> আর : মিশানো হয়, তাহলে compile হয় না — একটা style বেছে নিন।

Fall-through ও break

⚠ EXAM TRAP — colon-style fall through করে

break ছাড়া execution পরের branch-গুলোতে ক্রমে চলতে থাকে, default যেখানেই থাকুক সহ। নিচের printSeasonForMonth(2) সবকিছু print করে:

switch (month) {
  case 1, 2, 3: System.out.print("Winter-");
  case 4, 5, 6: System.out.print("Spring-");
  default:     System.out.print("Unknown-");
  case 7, 8, 9: System.out.print("Summer-");
  case 10,11,12: System.out.print("Fall-");
}
// Winter-Spring-Unknown-Summer-Fall-

Order গুরুত্বপূর্ণ! প্রতিটা branch-এর পরে break দিয়ে fall-through থামান। কোনো case clause না থাকা switch (switch (month) {}) বৈধ।

★ REMEMBER — arrow-style switch statement

যে arrow switch-এর body গুলো void ফেরায় (যেমন সব System.out.print(...)) সেটা এখনও একটা switch statement, expression নয় — কোনো value ফেরায় না। Arrow clause fall through করে না।

SECTION 03

switch Expression

একটা switch expression value ফেরায়, তাই সাধারণত =-এর ডানে বা return-এর পরে বসে, আর শেষে একটা semicolon লাগে। প্রতিটা arrow case expression-এরও নিজের semicolon লাগে; একটা case block (brace) লাগে না।

var result = switch (bear) {
  case 30 -> "Grizzly";   // semicolon after each case expression
  default -> "Panda";
};                          // semicolon after the whole expression

Consistent return type

প্রতিটা branch target type-এর সাথে compatible একটা value ফেরাতে হবে।

int size = switch (measurement) {
  case 5  -> Integer.valueOf(1); // OK (unboxed to int)
  case 10 -> (short)2;         // OK (short fits int)
  default -> 3;
  case 20 -> "4";             // DOES NOT COMPILE
  case 40 -> 5L;             // DOES NOT COMPILE
  case 50 -> null;           // DOES NOT COMPILE
};

Exhaustiveness

switch expression অবশ্যই exhaustive হতে হবে — তিন উপায়

  1. একটা default clause যোগ করুন (সবচেয়ে প্রচলিত)।
  2. enum-এর জন্য, প্রতিটা value-র জন্য একটা case দিন।
  3. Pattern matching দিয়ে সব সম্ভাব্য type cover করুন।

একটা value (যেমন FALL) বাদ পড়া enum switch ততক্ষণ compile হয় না যতক্ষণ সেই case বা একটা default না দেওয়া হয়। প্রতিটা int (এমনকি প্রতিটা byte)-এর জন্য case লেখা গ্রহণযোগ্য নয় — default লাগবেই।

yield statement

একটা case block তার value yield দিয়ে ফেরায় (একটা return পুরো method থেকে বেরিয়ে যাবে)। block-এ yield লাগে, যদি না সেটা exception throw করে।

var name = switch (fish) {
  case 1 -> "Goldfish";
  case 2 -> { yield "Trout"; }
  case 3 -> {
    if (length > 10) yield "Blobfish";
    else yield "Green";
  }
  default -> "Swordfish";
};
⚠ EXAM TRAP — case block-এ semicolon

একটা case expression-এ semicolon লাগে; একটা case block-এর closing brace-এর পরে লাগে না। case 2 -> { yield "Trout"; };-এ একটা extra semicolon আছে আর compile হয় না।

Pattern matching switch (Java 21)

প্রতিটা case-এ একটা type + pattern variable ব্যবহার করুন। একটা when guard একটা condition যোগ করে (when keyword লাগেই)। এতে switch range সামলাতে পারে।

return switch (height) {
  case Integer i when i > 10 -> "Joseph";
  case Integer i              -> "Daniel";
  case Double num when num <= 15.5 -> "Peter";
  case Double num             -> "Kelly";
  case Number num             -> "Ralph";
};
⚠ EXAM TRAP — dominance ও ordering

Pattern matching-এ, statement আর expression দুটোর জন্যই order গুরুত্বপূর্ণ। সরু case-এর আগে একটা চওড়া case সরুটাকে unreachable করে → compile error। case Integer i case Integer i when i > 10-এর আগে দিলে compile হয় না। switch variable-এর সাথে সম্পর্কহীন type (Number switch, case String s) compile হয় না।

✓ KEY RULE — case null ও runtime null

case null ছাড়া, একটা null switch variable runtime-এ NullPointerException throw করে (একটা default থাকলেও)। Java 21 case null অনুমোদন করে। case null ব্যবহার করলে switch-টা pattern-matching switch হয়ে যায় (তাই exhaustive হতে হবে), আর case null default-এর পরে আসতে পারে না।

SECTION 04

Loop: while, do-while, for, for-each

while ও do-while

একটা while প্রতিটা iteration-এর আগে condition check করে, তাই body শূন্য বার চলতে পারে। একটা do/while body-টা অন্তত একবার চালায়, তারপর check করে।

while (cond) { ... }

  • Condition আগে।
  • শূন্য বা তার বেশি iteration।
  • full=5 হলে while (full < 5) কিছুই print করে না।

do { ... } while (cond);

  • Body আগে, condition পরে।
  • এক বা তার বেশি iteration।
  • do { lizard++; } while (false); → lizard হয় 1।
⚠ EXAM TRAP — infinite loop

Condition-এর কোনো variable না বদলালে loop শেষ হয় না: pen fixed থাকলে while (pen < 10) pigs++; চিরকাল চলে। নিশ্চিত করুন loop variable termination-এর দিকে এগোয়।

basic for loop

তিনটা semicolon দিয়ে আলাদা করা অংশ: init; condition; update। Init আর update-এ comma দিয়ে একাধিক statement থাকতে পারে; declare করা variable-গুলো একই type হতে হবে।

⚠ EXAM TRAP — for-loop-এর খুঁটিনাটি 
for (int i = 0; i < 10; i++) ...;
System.out.println(i);  // DOES NOT COMPILE (i scoped to the loop)

for ( ; ; ) ...           // valid infinite loop (all sections optional)

int x = 0;
for (int x = 4; x < 5; x++) ...  // DOES NOT COMPILE (x redeclared)

for (long y = 0, int z = 4; ...) ... // DOES NOT COMPILE (mixed types)
★ REMEMBER — reverse loop পড়া

Decrement loop প্রচুর test হয়। for (var c = 4; c >= 0; c--) print করে 4 3 2 1 0। Start value আর condition > না >= খেয়াল করুন।

for-each loop

ডান পাশ হতে হবে একটা built-in array বা java.lang.Iterable implement করা কিছু (যেমন List, Set — কিন্তু Map নয়, আর String নয়)। বাঁ পাশ একটা variable declare করে যার type প্রতিটা element-এর সাথে মেলে।

for (var name : names) System.out.println(name);

String birds = "Jay";
for (String bird : birds) ...   // DOES NOT COMPILE (String isn't iterable here)
String[] sloths = new String[3];
for (int sloth : sloths) ...    // DOES NOT COMPILE (int vs String)
SECTION 05

Branching: break, continue, return, label

একটা label হলো একটা colon দিয়ে শেষ হওয়া identifier যা একটা loop-এর আগে বসে, break/continue-কে একটা outer loop target করতে দেয়। প্রথা অনুযায়ী label SNAKE_CASE।

break

  • পুরো loop থেকে বেরিয়ে যায়।
  • break LABEL; labelled outer loop থেকে বেরোয়।

continue

  • বর্তমান iteration শেষ করে, condition আবার check করে।
  • continue LABEL; labelled loop continue করে।

Label ছাড়া break/continue নিকটতম inner loop-এ লাগে। return পুরো method থেকে বেরোয় আর পরিষ্কার nested-loop search-এর জন্য label-এর বিকল্প হতে পারে।

⚠ EXAM TRAP — unreachable code 
if (checkDate > 100) {
  break;
  checkDate++;  // DOES NOT COMPILE (unreachable)
}

break, continue, বা return-এর ঠিক পরে একই block-এ থাকা যেকোনো statement unreachable আর compile হয় না — branch-টা যৌক্তিকভাবে অসম্ভব দেখালেও।

★ REMEMBER — labelled break ফল বদলায়

একটা nested search-এ, break LABEL; প্রথম match-এই সব loop থেকে বেরোয়; একটা সাধারণ break; শুধু inner loop থেকে বেরোয় (শেষ inner match খুঁজে); কোনো break না থাকলে সর্বশেষ match খুঁজে। প্রতিটা ক্ষেত্রে output আলাদা।

সংক্ষিপ্ত রিভিউ

Decision ও pattern matching

  • if-এ boolean লাগে (0/1 নয়); brace ছাড়া এটা একটা statement নিয়ন্ত্রণ করে — indentation নয়, brace trace করুন।
  • Pattern matching check + cast মিলিয়ে দেয়; null সবসময় false। Flow scoping variable-টাকে শুধু সেখানেই scope-এ রাখে যেখানে type প্রমাণিত (&& ঠিক, || নয়; early return scope বাড়াতে পারে)।

switch

  • অনুমোদিত type: int/Integer, byte/Byte, short/Short, char/Character, String, enum, object (pattern), var; কখনো boolean/long/float/double নয়। case হতে হবে switch type-এর সাথে মেলা compile-time constant।
  • Colon-style fall through করে (break দিন); arrow-style করে না। Expression exhaustive হতে হবে আর consistent type ফেরাতে হবে; yield block থেকে ফেরায়; case null pattern matching বোঝায় আর default-এর পরে আসতে পারে না।

Loop ও branching

  • while = zero-or-more; do/while = one-or-more। for-এর অংশ optional কিন্তু দুটো semicolon নয়; loop variable loop-এ scope করা। for-each-এ array বা Iterable লাগে (Map/String নয়)।
  • Label break/continue-কে outer loop target করতে দেয়; return method থেকে বেরোয়; break/continue/return-এর ঠিক পরের code unreachable।
