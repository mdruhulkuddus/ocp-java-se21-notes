Q1Pattern Switch

What is the output?

Object skips = 10;
switch (skips) {
  case a when a < 10 -> System.out.print(2);
  case b when b >= 10 -> System.out.print(4);
  case null -> System.out.print(6);
  default -> System.out.print(8);
}
  • A. 4
  • B. 6
  • C. Exactly one line does not compile.
  • D. Exactly two lines do not compile.
Show Answer D · two lines do not compile

The first two case clauses are missing the pattern variable type (e.g. case Integer a when …). With a supported type added, the code would compile and print 4. As written, both lines fail.

Q2Switch Types

Which data types can be used in a switch expression? (Choose all that apply.)

  • A. enum
  • B. int
  • C. Byte
  • D. long
  • E. boolean
  • F. double
Show Answer A, B, and C

A switch supports enum values (A), int (B), and the wrapper Byte along with byte (C). It does not support long, boolean, float, or double.

Q4for-each

Which types are permitted on the right side of a for-each? (Choose all that apply.)

  • A. Double[][]
  • B. Object
  • C. Map
  • D. List
  • E. String
  • F. char[]
Show Answer A, D, and F

for-each works with arrays (A, F — each Double[][] element is a Double[]) and types implementing java.lang.Iterable like List (D). Map doesn't implement Iterable (C), Object and String aren't iterable here (B, E).

Q5Exhaustiveness

What is the output of printReptile(6)?

var type = switch (category) {
  case 1,2 -> "Snake";
  case 3,4 -> "Lizard";
  case 5,6 -> "Turtle";
  case 7,8 -> "Alligator";
};
System.out.print(type);
  • A. Turtle
  • B. TurtleAlligator
  • C. None of the above
  • D. The code does not compile.
Show Answer D · does not compile

A switch expression must handle all possible int values, but there's no default and not every int is covered — so it isn't exhaustive. With a valid default added, it would compile and print Turtle.

Q6Unreachable

What is the output?

List<Integer> nums = new ArrayList<>();
nums.add(10); nums.add(14);
for (var a : nums) { System.out.print(a + ", "); break; }
for (int b : nums) { continue; System.out.print(b + ", "); }
for (Object c : nums) System.out.print(c + ", ");
  • A. 10, 10, 14,
  • B. 10, 10, 14, 10, 14,
  • C. Exactly one line of code does not compile.
  • D. Two or more lines do not compile.
Show Answer C · one line does not compile

In the second loop, the continue is unconditional and always part of the loop body, so the print() after it is unreachable — that line fails to compile. It's the only error; the other loops are fine.

Q7for loops

Which snippets, inserted into for (___), print all elements of a nonempty int[] weather (body uses weather[i])? (Choose all that apply.)

  • A. int i=weather.length; i>0; i--
  • B. int i=0; i<=weather.length-1; ++i
  • C. var w : weather
  • D. int i=weather.length-1; i>=0; i--

Q9Labels

Which statements, inserted into the blank, make the code print 2? (Choose all that apply.)

int count = 0;
BUNNY: for (int row = 1; row <= 3; row++)
RABBIT: for (int col = 0; col < 3; col++) {
  if ((col + row) % 2 == 0)
    ______ ;
  count++;
}
System.out.println(count);
  • A. break BUNNY
  • B. break RABBIT
  • C. continue BUNNY
  • D. continue RABBIT
  • E. break
Show Answer B, C, and E

A plain break (E) and break RABBIT (B) both exit the inner loop — equivalent. continue BUNNY (C) jumps to the outer loop. All three follow the same path that lands count at 2. break BUNNY exits everything too early (count = 1). continue RABBIT and a plain continue let the loops run on, reaching count = 5.

Q16case syntax

What is the result?

final char a = 'A', e = 'E';
char grade = 'B';
switch (grade) {
  default:
  case a:
  case 'B': 'C': System.out.print("great ");

Q17reverse loops

Given char[] wolf = {'W','e','b','b','y'};, which print it in reverse? (Choose all that apply.)

Quick Revision

  • if / pattern matching: if needs a boolean; trace braces, not indentation. Pattern matching merges check + cast; null → false. Flow scoping: && keeps the variable in scope, || doesn't; an early return can extend scope past the if.
  • switch types: int/Integer, byte/Byte, short/Short, char/Character, String, enum, object (pattern), var — never boolean/long/float/double. Cases must be compile-time constants of the matching type.
  • switch statement: colon-style falls through without break (order matters!); arrow-style doesn't. A void-bodied arrow switch is still a statement.
  • switch expression: must be exhaustive (default, all enum values, or all types) and return a consistent type. yield returns from a block; a block must not have a trailing semicolon; a case expression must.
  • pattern switch: order matters for both statements and expressions; a broader case before a narrower one is unreachable. case null implies pattern matching and can't follow default; without it a null switch variable throws NullPointerException.
  • loops: while = zero+; do/while = one+. for sections are optional, the two semicolons aren't, and loop variables are loop-scoped; init must share one type. for-each needs an array or Iterable (not Map/String).
  • branching: labels let break/continue target outer loops; return exits the method; code right after break/continue/return is unreachable.
