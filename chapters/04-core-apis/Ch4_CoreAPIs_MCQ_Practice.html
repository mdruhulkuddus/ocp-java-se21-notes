Q1Strings

What is printed?

System.out.println(1 + 2 + "c" + 3 + 4);
  • A. 12c34
  • B. 3c7
  • C. 3c34
  • D. 12c7
Show Answer C · 3c34

Evaluation is strictly left to right. 1 + 2 are both numeric → 3. Then 3 + "c" hits a String → concatenation → "3c". After a String appears, everything to the right also concatenates: "3c" + 3 → "3c3", then + 4 → "3c34".

Q2Strings

Given var s = "animals";, which calls throw an exception at runtime? (Choose all that apply.)

  • A. s.substring(3, 7)
  • B. s.substring(3, 2)
  • C. s.charAt(7)
  • D. s.indexOf('z')
Show Answer B and C

"animals" has length 7 (indexes 0–6). A is fine — end == length (7) is allowed and returns "mals". B throws because the end index (2) is before the begin index (3). C throws StringIndexOutOfBoundsException — index 7 is past the last valid index 6. D simply returns -1 (no exception when not found).

Q3Strings

Given String text = " abc\t "; (a leading space, then "abc", a tab, and a trailing space), what does text.stripLeading().length() return?

  • A. 3
  • B. 4
  • C. 5
  • D. 7
Show Answer C · 5

stripLeading() removes only the leading whitespace (the first space), leaving abc + tab + trailing space = 5 characters. The tab \t counts as a single character. (For comparison, strip() and trim() would give 3, and stripTrailing() would give 4.)

Q4Strings

What is the result?

System.out.println("Food: %d tons".formatted(2.0));
  • A. Food: 2 tons
  • B. Food: 2.0 tons
  • C. It does not compile.
  • D. It throws an exception at runtime.
Show Answer D · runtime exception

%d is for integer types, but a double (2.0) is passed. This compiles fine — the mismatch isn't caught until runtime, where it throws IllegalFormatConversionException. (To print a floating-point value you'd use %f.)

Q5Strings

After this runs, what are the values of a and b?

String a = "abc";
String b = a.toUpperCase();
b = b.replace("B", "2").replace('C', '3');
  • A. a="abc", b="A23"
  • B. a="A23", b="A23"
  • C. a="abc", b="ABC"
  • D. a="abc", b="a23"
Show Answer A · a="abc", b="A23"

String is immutable, so a never changes. b starts as "ABC"; replace("B","2") → "A2C"; then replace('C','3') → "A23". The chained calls each return a new String assigned back to b.

Q6StringBuilder

What is printed?

var sb = new StringBuilder("animals");
String sub = sb.substring(sb.indexOf("a"), sb.indexOf("al"));
int len = sb.length();
char ch = sb.charAt(6);
System.out.println(sub + " " + len + " " + ch);
  • A. anim 4 s
  • B. anim 7 s
  • C. animal 7 s
  • D. anim 7 l
Show Answer B · anim 7 s

indexOf("a") is 0, indexOf("al") is 4, so substring(0,4) = "anim". Crucially, substring() does not modify the StringBuilder, so length() is still 7 and charAt(6) is 's'.

Q7StringBuilder

What does this print?

var sb = new StringBuilder()
    .append("aaa")
    .insert(1, "bb")
    .insert(4, "ccc");
System.out.println(sb);
  • A. aaabbccc
  • B. abbaccca
  • C. abbcccaa
  • D. abbaaccc
Show Answer B · abbaccca

Start: aaa. insert(1,"bb") places "bb" before index 1 → abbaa. insert(4,"ccc") places "ccc" before index 4 → abba + ccc + a = abbaccca. The indexes refer to the string as it exists at each step.

Q8StringBuilder

Which statement about the following is correct?

var sb = new StringBuilder("abcdef");
sb.delete(1, 3);
sb.deleteCharAt(5);
  • A. sb ends up as "adef".
  • B. sb ends up as "ade".
  • C. The second line throws an exception at runtime.
  • D. The code does not compile.
Show Answer C · runtime exception

delete(1, 3) removes indexes 1–2, leaving "adef" (4 characters, indexes 0–3). deleteCharAt(5) then references index 5, which no longer exists, throwing StringIndexOutOfBoundsException. Note delete tolerates a too-large end index, but deleteCharAt is strict.

Q9StringBuilder

What is the value of sb after this runs?

var sb = new StringBuilder("pigeon dirty");
sb.replace(3, 6, "sty");
  • A. pigsty dirty
  • B. pigstydirty
  • C. stydirty
  • D. pigsty
Show Answer A · pigsty dirty

StringBuilder.replace(start, end, str) deletes [3, 6) — the characters eon — then inserts "sty" in that spot, producing pig + sty + " dirty" = "pigsty dirty". This is different from String.replace, which swaps matching characters/sequences.

Q10Equality

What is printed?

var one = new StringBuilder();
var two = new StringBuilder();
var three = one.append("a");
System.out.println(one == two);
System.out.println(one == three);
  • A. true then true
  • B. false then false
  • C. false then true
  • D. true then false
Show Answer C · false then true

one and two are two separate objects, so == is false. append() mutates and returns the same object, so three is just another reference to one → one == three is true.

Q11Equality

What happens with this code?

var name = "a";
var builder = new StringBuilder("a");
System.out.println(name == builder);
  • A. Prints true
  • B. Prints false
  • C. Does not compile
  • D. Throws at runtime
Show Answer C · does not compile

String and StringBuilder are unrelated types. The compiler knows references of these two types can never point to the same object, so comparing them with == is a compile-time error.

Q12Equality & Pool

Which print true? (Choose all that apply.)

var x = "Hello World";
var y = "Hello World";
var z = " Hello World".trim();
var w = new String("Hello World").intern();
  • A. x == y
  • B. x == z
  • C. x == w
  • D. x.equals(z)
Show Answer A, C, and D

A: both are the same pooled literal → true. B: trim() runs at runtime and builds a new object outside the pool → false. C: intern() returns the pooled reference for "Hello World" → true. D: equals compares characters, which match → true.

Q13Equality & Pool

Given the assignments below, what does first == third print, and what does first == third.intern() print?

var first  = "rat" + 1;
var third  = "r" + "a" + "t" + new String("1");
  • A. true, then true
  • B. false, then true
  • C. false, then false
  • D. true, then false
Show Answer B · false, then true

first is a compile-time constant ("rat" + 1) and is pooled as "rat1". third includes new String("1"), so it is computed at runtime → a separate object → first == third is false. Calling intern() on third returns the pooled "rat1", which equals first → true.

Q14Arrays

After int ids[], types;, which is true?

  • A. Both ids and types are int[].
  • B. ids is int[]; types is a plain int.
  • C. Both are plain int.
  • D. The line does not compile.
Show Answer B · ids is int[], types is int

When the brackets follow the variable name (ids[]), they apply only to that one variable. So ids is an int[] but types is just an int. Contrast with int[] ids, types; where the brackets follow the type and both become int[].

Q15Arrays

What is the result?

String[] strings = {"stringValue"};
Object[] objects = strings;
objects[0] = new StringBuilder();
  • A. Compiles and runs with no output.
  • B. Does not compile.
  • C. Throws ArrayStoreException at runtime.
  • D. Throws ClassCastException at runtime.
Show Answer C · ArrayStoreException

Assigning String[] to Object[] is allowed (widening, no cast). The compiler permits objects[0] = new StringBuilder() because the declared type is Object[]. But the actual array is still a String[], so storing a StringBuilder throws ArrayStoreException at runtime.

Q16Arrays

Which of these cause a compile error? (Choose all that apply.)

String[] mammals = {"monkey", "chimp", "donkey"};
// A int x = mammals.length;
// B int y = mammals.length();
// C int z = new String[6].length;
// D mammals[3] = "cat";
  • A. Line A
  • B. Line B
  • C. Line C
  • D. Line D
Show Answer B only

length is a field on arrays, not a method, so B's length() does not compile. A and C use the field correctly (C is 6). D compiles fine — it's only at runtime that index 3 on a length-3 array throws ArrayIndexOutOfBoundsException, which is not a compile error.

Q17Arrays

What does this print?

String[] arr = {"10", "9", "100"};
Arrays.sort(arr);
System.out.println(Arrays.toString(arr));
  • A. [9, 10, 100]
  • B. [10, 100, 9]
  • C. [100, 10, 9]
  • D. [10, 9, 100]
Show Answer B · [10, 100, 9]

These are Strings, so they sort alphabetically, not numerically. Comparing character by character: "100" comes before "10"? No — "10" is a prefix of "100", so the shorter "10" sorts first, then "100", then "9" (since '9' > '1'). Result: [10, 100, 9].

Q18Arrays

Given int[] n = {2, 4, 6, 8};, what does Arrays.binarySearch(n, 3) return?

  • A. 1
  • B. -1
  • C. -2
  • D. 2
Show Answer C · -2

3 is not present. It would be inserted at index 1 (between 2 and 4). For a not-found value, the result is -(insertion point) - 1 = -(1) - 1 = -2. (The array is sorted, so the result is well-defined.)

Q19Arrays

What does Arrays.mismatch(new int[]{1, 2}, new int[]{1}) return?

  • A. -1
  • B. 0
  • C. 1
  • D. 2
Show Answer C · 1

The arrays agree at index 0 (both 1). They first differ at index 1, where the first array has 2 and the second has no element. mismatch returns that first differing index, 1. (It returns -1 only when the arrays are completely equal.)

Q20Arrays

Which declaration does not compile?

  • A. int[][] a = new int[2][];
  • B. int[][] b = {{1, 4}, {3}, {9, 8, 7}};
  • C. int[][] c = new int[][];
  • D. int[] d[], e[][];
Show Answer C · new int[][]

When using new for a multidimensional array, the first dimension's size is required; later dimensions may be omitted. So new int[2][] (A) is legal but new int[][] (C) is not. B is a valid jagged array, and D declares d as 2D and e as 3D.

Q21Math API

Which lines do not compile? (Choose all that apply.)

// A double one   = Math.pow(1, 2);
// B int    two   = Math.round(1.0);
// C float  three = Math.random();
// D int    four  = Math.round(1.0f);
  • A. Line A
  • B. Line B
  • C. Line C
  • D. Line D
Show Answer B and C

B fails: round(double) returns a long, which can't be assigned to int without a cast. C fails: random() returns a double, not a float. A is fine (pow returns double). D is fine: round(float) returns an int.

Q22Math API

What is the value and type of Math.ceil(3.14)?

  • A. 3 as an int
  • B. 4 as an int
  • C. 4.0 as a double
  • D. 3.0 as a double
Show Answer C · 4.0 (double)

ceil rounds up to the next whole number but still returns a double, so the result is 4.0. (Math.floor(3.14) would give 3.0, also a double.)

Q23Math API

Which statement is true about this line?

System.out.println(12345123451234512345L);
  • A. It prints the number.
  • B. It does not compile.
  • C. It throws an exception at runtime.
  • D. It prints a negative number due to overflow.
Show Answer B · does not compile

The literal is larger than Long.MAX_VALUE, so even with the L suffix it doesn't fit in a long — that's a compile-time error. To hold a value this large you'd use new BigInteger("12345123451234512345").

Q24Date-Time

Which line does not compile or throws at runtime, and why?

// A var a = new LocalDate();
// B var b = LocalDate.of(2025, Month.JANUARY, 20);
  • A. A does not compile; B is fine.
  • B. B does not compile; A is fine.
  • C. Both compile; A throws at runtime.
  • D. Both are fine.
Show Answer A · A does not compile

The date-time classes use the factory pattern — their constructors are private. You can't call new LocalDate(), so line A is a compile error. Line B uses the static of() factory correctly. (Passing day 32 would throw DateTimeException, but 20 is valid.)

Q25Date-Time

What is printed?

var date = LocalDate.of(2025, Month.JANUARY, 20);
date.plusDays(10);
System.out.println(date);
  • A. 2025-01-30
  • B. 2025-01-20
  • C. It does not compile.
  • D. It throws an exception.
Show Answer B · 2025-01-20

LocalDate is immutable. plusDays(10) returns a new date but the return value is ignored, so date is unchanged. This "ignored return value" pattern is one of the most common date-time traps.

Q26Date-Time

Starting from LocalDate.of(2025, Month.JANUARY, 31), what is the result of plusMonths(1)?

  • A. 2025-02-31
  • B. 2025-03-03
  • C. 2025-02-28
  • D. It throws DateTimeException.
Show Answer C · 2025-02-28

February has no 31st, and 2025 is not a leap year (it's not a multiple of 4), so there's no Feb 29 either. The API adjusts to the last valid day of the target month — Feb 28 — rather than overflowing or throwing.

Q27Date-Time

Which combinations throw UnsupportedTemporalTypeException at runtime? (Choose all that apply.)

  • A. LocalDate.now().plus(Period.ofDays(1))
  • B. LocalDate.now().plus(Duration.ofHours(6))
  • C. LocalTime.now().plus(Period.ofMonths(1))
  • D. LocalTime.now().plus(Duration.ofMinutes(30))
Show Answer B and C

Period holds date units and Duration holds time units. B applies a time-based Duration to a date-only LocalDate → throws. C applies a date-based Period to a time-only LocalTime → throws. A (Period + LocalDate) and D (Duration + LocalTime) are the correct pairings.

Q28Date-Time

What does ChronoUnit.HOURS.between(LocalTime.of(5, 15), LocalTime.of(6, 55)) return?

  • A. 2
  • B. 1
  • C. 100
  • D. It throws an exception.
Show Answer B · 1

The gap is 1 hour 40 minutes. ChronoUnit.between truncates toward the whole unit rather than rounding, so in HOURS it returns 1, not 2. (Asked in MINUTES, the same gap would return 100.)

Quick Revision

  • Concatenation: both numeric → add; either a String → concatenate; always left to right. "c" + null → "cnull".
  • substring(b, e): b inclusive, e exclusive; e == length is OK; reversed or past-length throws. indexOf returns -1 when absent.
  • StringBuilder: append/insert/delete/replace mutate and return this; substring returns a String and does not mutate. delete tolerates a too-large end; deleteCharAt does not.
  • Equality: == is reference identity; literals & compile-time constants are pooled; runtime-built strings and new String() are not; intern() pulls from the pool. String == StringBuilder won't compile.
  • Arrays: length is a field; length() won't compile. Covariant stores throw ArrayStoreException. binarySearch on a sorted array → index or -(insertion point)-1; unsorted is undefined. String sort is alphabetic.
  • Math: watch return types — round(double)→long, round(float)→int, ceil/floor/pow/random→double. A literal too large for long won't compile.
  • Dates: private constructors (use of()); immutable (don't ignore the return); months count from 1; Period = date units (not LocalTime), Duration = time units (not LocalDate); ChronoUnit.between truncates.
