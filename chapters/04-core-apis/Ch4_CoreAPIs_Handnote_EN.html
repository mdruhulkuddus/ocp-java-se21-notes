SECTION 01

Creating & Manipulating Strings

A String is a sequence of characters and a reference type. It is special: you do not need new. All three of these create a String "Fluffy":

String name = "Fluffy";
String name = new String("Fluffy");
String name = """
Fluffy""";  // text block

String implements CharSequence (so does StringBuilder). Strings are immutable — every "modifying" method returns a new String.

Concatenating

The three concatenation rules — memorize them

  1. If both operands are numeric, + means numeric addition.
  2. If either operand is a String, + means concatenation.
  3. The expression is evaluated left to right.
System.out.println(1 + 2);            // 3
System.out.println("a" + "b");        // ab
System.out.println("a" + "b" + 3);    // ab3
System.out.println(1 + 2 + "c");      // 3c
System.out.println("c" + 1 + 2);      // c12
System.out.println("c" + null);        // cnull
⚠ EXAM TRAP

Watch variable types in mixed expressions. With int three = 3; and String four = "4";:

  • 1 + 2 + three + four
  • 1 + 2 → both int → 3
  • 3 + three → both int → 6
  • 6 + four → four is String → switch to concat → "64"

+= follows the same rules: s += "2" equals s = s + "2". Starting from var s = "1", then s += "2" → "12", then s += 3 → "123" (a String "plus" anything → concatenation).

Important String methods

Indexes are zero-based, but length() uses normal counting. For "animals": length is 7, valid indexes are 0–6.

length() & charAt(int)

var name = "animals";
name.length();    // 7
name.charAt(0);   // a
name.charAt(6);   // s
name.charAt(7);   // StringIndexOutOfBoundsException

indexOf(...) — returns -1 when not found (no exception)

Start index is inclusive; end index is exclusive. Accepts a char (as int ch) or a String.

var name = "animals";
name.indexOf('a');       // 0
name.indexOf("al");      // 4
name.indexOf('a', 4);    // 4
name.indexOf("al", 5);   // -1 (search starts after the match)
name.indexOf('a', 2, 4); // -1 (only looks at indexes 2,3)
name.indexOf("al", 2, 6);// 4

substring(begin[, end]) — the trickiest method on the exam

✓ KEY RULE

Returns characters from begin (inclusive) up to but not including end (exclusive). The end index may be one past the end (= length). It must not be reversed and must not be more than one past the end.

var name = "animals";  // length 7
name.substring(3);     // mals
name.substring(3, 4);  // m
name.substring(3, 7);  // mals  (7 == end, allowed)
name.substring(3, 3);  // "" empty string
name.substring(3, 2);  // exception (reversed)
name.substring(3, 8);  // exception (8 > length)

Case, search, replace, whitespace

  • text.trim().length() → 3
  • text.strip().length() → 3
  • text.stripLeading().length() → 5 (keeps trailing tab + space)
  • text.stripTrailing().length() → 4 (keeps leading space)
★ REMEMBER

For the exam, whitespace = spaces plus \t (tab), \n (newline), and \r (carriage return). \t is a single character.

⚠ EXAM TRAP — the "extra" character

Because indent() normalizes line termination, a text block with no trailing newline gains one. A 6-char block after indent(1) becomes 10 (3 added spaces + 1 normalized newline), not 9.

Empty vs blank

" ".isEmpty();  // false (has a space)
"".isEmpty();   // true
" ".isBlank();  // true (only whitespace)
"".isBlank();   // true

Formatting

String.format(...) is static; formatted(...) is an instance method. Symbols you must know:

SymbolApplies to
%sAny type (commonly String)
%dInteger values (int, long)
%fFloating-point (float, double) — defaults to 6 decimals, rounds
%nSystem-dependent line break
⚠ EXAM TRAP

Mismatched types throw at runtime: "Food: %d tons".formatted(2.0) throws IllegalFormatConversionException (a double given to %d). With %.1f, 90.250000 prints 90.3 (rounded, not truncated).

Method chaining

Read left to right; call the next method on the previous return value. Each call on an immutable String makes a new String.

  • "AniMaL ".trim().toLowerCase().replace('a','A')
  • trim() → "AniMaL"
  • toLowerCase() → "animal"
  • replace('a','A') → "AnimAl"

Because String is immutable, the original variable never changes:

String a = "abc";
String b = a.toUpperCase();          // b = "ABC", a still "abc"
b = b.replace("B", "2").replace('C', '3');
// "ABC" -> "A2C" -> "A23"; a="abc", b="A23"
SECTION 02

Using the StringBuilder Class

Appending to a String in a loop creates many throwaway objects ("" + 26 letters → 27 String objects). StringBuilder is mutable and reuses one object.

String

  • Immutable
  • Methods return a new String
  • Chained calls produce new objects
  • Pooled (literals/constants)

StringBuilder

  • Mutable
  • Most methods mutate & return this
  • Chained calls all act on one object
  • Not pooled; StringBuffer is the thread-safe (slower) cousin — not on the exam

Mutability & chaining

⚠ EXAM TRAP

Chaining a StringBuilder does not create new objects — it mutates the same object and returns a reference to it. Assignments back to it "do nothing."

StringBuilder a = new StringBuilder("abc");
StringBuilder b = a.append("de");       // a and b -> "abcde"
b = b.append("f").append("g");        // same object -> "abcdefg"
System.out.println("a=" + a);          // a=abcdefg
System.out.println("b=" + b);          // b=abcdefg

Creating a StringBuilder

new StringBuilder();        // empty
new StringBuilder("animal"); // initial value
new StringBuilder(10);      // reserve capacity 10 (not contents!)

Important methods

Common methods (same as String): charAt, indexOf, length, substring

✓ KEY RULE

substring() returns a String and does not modify the StringBuilder — it only inquires about state. Compare with append(), insert(), delete() which do mutate.

var sb = new StringBuilder("animals");
String sub = sb.substring(sb.indexOf("a"), sb.indexOf("al")); // substring(0,4)="anim"
int len = sb.length();   // 7
char ch = sb.charAt(6);  // s
System.out.println(sub + " " + len + " " + ch); // anim 7 s

append — 10+ overloads, returns this

var sb = new StringBuilder().append(1).append('c');
sb.append("-").append(true);
System.out.println(sb); // 1c-true

insert(int offset, ...) — indexes shift as you go

var sb = new StringBuilder("animals");
sb.insert(7, "-");  // animals-
sb.insert(0, "-");  // -animals-
sb.insert(4, "-");  // -ani-mals-

delete(start,end) & deleteCharAt(index)

delete() is lenient: an end past the length is treated as the end. deleteCharAt() is strict.

var sb = new StringBuilder("abcdef");
sb.delete(1, 3);       // adef
sb.deleteCharAt(5);   // exception (only 4 chars left)

var sb2 = new StringBuilder("abcdef");
sb2.delete(1, 100);    // a  (end past length is OK)

replace(start,end,str) — different from String's replace!

It logically deletes [start,end) first, then inserts the new text (lengths need not match; end past length = end).

var b = new StringBuilder("pigeon dirty");
b.replace(3, 6, "sty"); // pigsty dirty

var c = new StringBuilder("pigeon dirty");
c.replace(3, 100, ""); // pig

reverse() & toString()

var sb = new StringBuilder("ABC");
sb.reverse();           // CBA
String s = sb.toString(); // convert back to String
⚠ EXAM TRAP — chained inserts
  • new StringBuilder().append("aaa").insert(1,"bb").insert(4,"ccc")
  • append → aaa
  • insert(1,"bb") → abbaa
  • insert(4,"ccc") → abbaccca
SECTION 03

Understanding Equality

== (reference)

  • Primitives: compares values
  • Objects: same object in memory?
  • On StringBuilder: reference check

equals() (logical)

  • Depends on the class's implementation
  • String.equals: same characters?
  • StringBuilder does not override equals → reference check
var one = new StringBuilder();
var two = new StringBuilder();
var three = one.append("a");   // append returns 'one'
System.out.println(one == two);   // false (different objects)
System.out.println(one == three); // true  (same object)
⚠ EXAM TRAP — type mismatch won't compile 
var name = "a";
var builder = new StringBuilder("a");
System.out.println(name == builder); // DOES NOT COMPILE

The compiler knows two completely different types can never reference the same object.

The String pool

✓ KEY RULE

The string pool holds literals and compile-time constants. Anything computed at runtime (a method call like trim(), +=, or new String(...)) creates a fresh object outside the pool.

var x = "Hello World";
var y = "Hello World";
System.out.println(x == y);             // true  (same pooled literal)

var z = " Hello World".trim();
System.out.println(x == z);             // false (computed at runtime)

var concat = "hello "; concat += "world";
System.out.println("hello world" == concat); // false (+= -> new String)

var n  = "Hello World";
var n2 = new String("Hello World").intern();
System.out.println(n == n2);            // true  (intern() pulls from pool)

Compile-time constants vs runtime

  • first = "rat" + 1 → compile-time constant "rat1" (pooled)
  • second = "r"+"a"+"t"+"1" → also compile-time constant → same pool ref
  • third = "r"+"a"+"t"+new String("1") → not constant → new object
  • first == second → true
  • first == second.intern() → true
  • first == third → false
  • first == third.intern() → true (intern finds "rat1" in pool)
★ REMEMBER

In real code, always use equals() to compare Strings — never == or intern(). Those only matter on the exam.

SECTION 04

Understanding Arrays

An array is a fixed-size, ordered list on the heap; it can hold duplicates and any Java type. char[] is a reference variable; the elements are char primitives.

Creating arrays of primitives

int[] numbers = new int[3];          // all default 0
int[] more    = new int[]{42, 55, 99};
int[] shorter = {42, 55, 99};         // anonymous array

All five of these declare the same thing — brackets can go before or after, with or without a space:

int[] a;  int [] b;  int []c;  int d[];  int e [];
⚠ EXAM TRAP — bracket placement on a multi-variable line 
int[] ids, types;  // BOTH are int[]
int ids[], types;  // ids is int[]; types is a plain int!

Arrays of reference types

Arrays are objects; equals() is not overridden, so it uses reference equality (it never compares elements).

String[] bugs = {"cricket", "beetle", "ladybug"};
String[] alias = bugs;
String[] other = {"cricket", "beetle", "ladybug"};
bugs.equals(alias);  // true  (same reference)
bugs.equals(other);  // false (different object, elements ignored)
bugs.toString();     // [Ljava.lang.String;@... (use Arrays.toString instead)

Arrays.toString(bugs) → [cricket, beetle, ladybug]. An uninitialized array field is null; new String[2] gives two null slots.

Array casting & ArrayStoreException

⚠ EXAM TRAP — compiles but throws at runtime 
String[] strings = {"stringValue"};
Object[] objects = strings;           // widening, no cast needed
String[] again = (String[]) objects;   // narrowing, cast needed
again[0] = new StringBuilder();        // DOES NOT COMPILE (String[] needs String)
objects[0] = new StringBuilder();      // compiles, but throws ArrayStoreException at runtime

Using an array — length is a field

String[] mammals = {"monkey", "chimp", "donkey"};
mammals.length;     // 3 (no parentheses!)
mammals.length();   // DOES NOT COMPILE
new String[6].length; // 6 (slots, even if all null)
⚠ EXAM TRAP — ArrayIndexOutOfBoundsException

For an array of size 10, valid indexes are 0–9. All three of these throw at runtime:

numbers[10] = 3;               // index 10 invalid
numbers[numbers.length] = 5;   // length is one past max index
for (int i = 0; i <= numbers.length; i++) ... // <= reaches index 10

Sorting

Arrays.sort(array) sorts ascending. Strings sort alphabetically, so digit strings can surprise you.

Arrays.sort(new int[]{6,9,1});         // 1 6 9
Arrays.sort(new String[]{"10","9","100"}); // 10 100 9 (alphabetic!)
★ REMEMBER — "7Up" sort order

Numbers (7) sort first, then Uppercase (U), then lowercase (p).

Searching with Arrays.binarySearch

Binary search rules (array MUST be sorted)

  • Found: returns the index of the match.
  • Not found: returns -(insertion point) - 1 (a negative).
  • Unsorted array: result is undefined — pick "unpredictable" as the answer.
int[] n = {2,4,6,8};
Arrays.binarySearch(n, 2);  // 0
Arrays.binarySearch(n, 4);  // 1
Arrays.binarySearch(n, 1);  // -1  (insert at 0 -> -(0)-1)
Arrays.binarySearch(n, 3);  // -2  (insert at 1 -> -(1)-1)
Arrays.binarySearch(n, 9);  // -5  (insert at 4 -> -(4)-1)

Comparing arrays: equals / compare / mismatch

MethodArrays contain the same dataArrays are different
Arrays.equals()truefalse
Arrays.compare()0positive or negative number
Arrays.mismatch()-1index of first difference (≥0)

compare() ordering rules

  • Negative = first array is "smaller"; 0 = equal; positive = first is "larger".
  • If all leading elements match, the shorter array is smaller.
  • null < any value; numbers < letters; uppercase < lowercase; a prefix is smaller.
  • Comparing arrays of different types → DOES NOT COMPILE.
Arrays.mismatch(new int[]{1}, new int[]{1});      // -1 (equal)
Arrays.mismatch(new String[]{"a"}, new String[]{"A"}); // 0  (differ at index 0)
Arrays.mismatch(new int[]{1,2}, new int[]{1});  // 1  (extra element at index 1)

Varargs & multidimensional arrays

A varargs parameter (String... args) is used as a normal array inside the method (args.length, args[0]). All three main signatures are valid: String[] args, String args[], String... args.

int[][] vars1;          // 2D
int vars2[][];          // 2D
int[] vars3[];          // 2D
int[] vars4[], space[][];// vars4 is 2D, space is 3D

String[][] rect = new String[3][2];        // rectangular
int[][] jagged = {{1,4}, {3}, {9,8,7}};   // asymmetric
int[][] partial = new int[2][];          // first dim only; fill later
✓ KEY RULE

For an array of arrays you may omit the size of later dimensions, but the first dimension's size is required when you use new (e.g. new int[2][] is legal, new int[][] is not).

Calculating with Math APIs

⚠ EXAM TRAP — return types are the trap

Always check the return type of a Math method against the variable type. Mismatches cause DOES NOT COMPILE.

Math.round(123.50); // 124  (long)

⚠ EXAM TRAP — classic compile check
  • double one = Math.pow(1, 2); → pow returns double → OK
  • int two = Math.round(1.0); → round(double) returns long → DOES NOT COMPILE
  • float three = Math.random(); → random returns double → DOES NOT COMPILE

BigInteger & BigDecimal

Use these when values don't fit in primitives or precision matters (money). Both are immutable — chain methods. Prefer valueOf(); you may pass a long to either, but a double only to BigDecimal. Constants include BigInteger.ZERO, BigDecimal.ONE.

var bigInt = BigInteger.valueOf(199)
    .add(BigInteger.valueOf(1))   // 200
    .divide(BigInteger.TEN)        // 20
    .max(BigInteger.valueOf(6)); // 20

System.out.println(new BigInteger("12345123451234512345"));
System.out.println(12345123451234512345L); // DOES NOT COMPILE (too big for long)

double cents = 64.1 * 100; // 6409.999999999999 (floating-point error)
BigDecimal.valueOf(64.1).multiply(BigDecimal.valueOf(100)); // 6410.0
SECTION 06

Working with Dates & Times

Import with import java.time.*;. The old java.util.Date is not on the exam. Pick the class by how much information you need:

ClassContainsExample
LocalDatedate onlyyour birthday this year
LocalTimetime onlymidnight
LocalDateTimedate + time, no zonestroke of midnight on New Year's Eve
ZonedDateTimedate + time + zonea call at 9 a.m. EST

Creating dates & times

✓ KEY RULE — months count from 1

Unlike indexes, the Date-Time API counts months from 1 (just like humans). Month.JANUARY == 1.

LocalDate.of(2025, Month.JANUARY, 20);  // same as below
LocalDate.of(2025, 1, 20);
LocalTime.of(6, 15);            // hour, minute
LocalTime.of(6, 15, 30);        // + seconds
LocalTime.of(6, 15, 30, 200);   // + nanoseconds
LocalDateTime.of(date, time);     // combine
ZonedDateTime.of(dateTime, ZoneId.of("US/Eastern"));
⚠ EXAM TRAP — no public constructors

These classes use the factory pattern (private constructors + static methods):

var d = new LocalDate();                       // DOES NOT COMPILE
var d = LocalDate.of(2025, Month.JANUARY, 32); // DateTimeException (no Jan 32)

Also note: ZonedDateTime.of has no overload that takes a Month enum.

Manipulating — immutable, so use the return value

⚠ EXAM TRAP — ignored return value 
var date = LocalDate.of(2025, Month.JANUARY, 20);
date.plusDays(10);            // result thrown away!
System.out.println(date);    // still 2025-01-20
  • start 2025-01-20
  • plusDays(2) → 2025-01-22
  • plusWeeks(1) → 2025-01-29
  • plusMonths(1) → 2025-02-28 (no Feb 29 in 2025)
  • plusYears(5) → 2030-02-28
★ REMEMBER — leap years

Leap years are multiples of 4 or 400, but not other multiples of 100. So 2000 & 2028 are leap years; 2100 is not.

⚠ EXAM TRAP — wrong unit for the type 
var date = LocalDate.of(2025, Month.JANUARY, 20);
date = date.plusMinutes(1); // DOES NOT COMPILE (LocalDate has no time)

Use with...() methods (e.g. withYear, withDayOfMonth) to copy with one field changed, and at...() methods (atTime, atStartOfDay, atDate, atZone) to convert between types.

Period vs Duration

Period (date-based)

  • Days, months, years
  • Output PyYmMdD (zeros omitted)
  • ofYears/ofMonths/ofWeeks/ofDays/of(y,m,d)
  • Use with LocalDate, LocalDateTime, ZonedDateTime
  • Not LocalTime

Duration (time-based)

  • Days, hours, minutes, seconds, nanos
  • Output starts with PT
  • ofDays/ofHours/ofMinutes/ofSeconds/ofMillis/ofNanos, or of(n, ChronoUnit)
  • Use with LocalTime, LocalDateTime, ZonedDateTime
  • Not LocalDate
⚠ EXAM TRAP — Period can't chain factories 
var wrong = Period.ofYears(1).ofWeeks(1); // only ofWeeks(1) is used!

These are static methods, so chaining just overwrites. of() takes only years/months/days. Period.ofMonths(3) prints P3M; Period.between(xmas, newYears) → P7D, reversed → P-7D.

TypeUse with Period?Use with Duration?
LocalDateYesNo
LocalTimeNoYes
LocalDateTimeYesYes
ZonedDateTimeYesYes

Applying the wrong one throws UnsupportedTemporalTypeException at runtime, e.g. LocalTime.plus(Period.ofMonths(-1)) or LocalDate.plus(Duration.ofHours(6)).

ChronoUnit & truncation

var one = LocalTime.of(5, 15);
var two = LocalTime.of(6, 55);
ChronoUnit.HOURS.between(one, two);   // 1  (truncates, not rounds)
ChronoUnit.MINUTES.between(one, two); // 100
ChronoUnit.MINUTES.between(one, date);// DateTimeException (mixed types)

LocalTime.of(3,12,45).truncatedTo(ChronoUnit.MINUTES); // 03:12

Instant & daylight saving time

Instant is a moment in GMT. You can convert a ZonedDateTime to an Instant with toInstant(), but you cannot convert a LocalDateTime (no time zone → no fixed moment).

U.S. daylight saving time (the only one tested)

  • Spring forward (March, ~2 a.m.): 1:59 a.m. jumps to 3:00 a.m. — there is no 2:30 a.m. Offset changes -05:00 → -04:00.
  • Fall back (November, ~2 a.m.): the 1:00–1:59 a.m. hour happens twice. Offset changes -04:00 → -05:00.
  • Creating a time that doesn't exist (e.g. 2:30 on spring-forward day) simply rolls forward to 3:30.
  • March 9, 2025, 1:30 EST → offset -05:00, hour 1
  • plusHours(1) → 3:30, offset -04:00, hour 3
  • GMT check: 1:30 −(−5:00)=6:30 GMT → 3:30 −(−4:00)=7:30 GMT (exactly one hour)
★ SHORT RECAP

Short Recap

Strings

  • String is immutable; + = addition if both numeric, otherwise concatenation, evaluated left→right.
  • substring(b,e): b inclusive, e exclusive; e may equal length but not exceed it or reverse; indexOf returns -1 when absent.

StringBuilder

  • Mutable; append/insert/delete mutate & return this; substring returns a String without mutating.
  • delete/replace tolerate an end index past the length; deleteCharAt does not.

Equality

  • == is reference equality; String.equals compares characters; StringBuilder does not override equals.
  • Literals & compile-time constants are pooled; runtime-built strings and new String() are not; intern() pulls from the pool.

Arrays

  • length is a field (no parentheses); indexes 0..length-1; equals on arrays is reference-based.
  • binarySearch on a sorted array returns the index or -(insertion point)-1; unsorted → undefined.

Math

  • Watch return types: round(double)→long, round(float)→int, pow→double, random→double, ceil/floor→double.
  • A literal too large for long doesn't compile; use BigInteger/BigDecimal (immutable, valueOf).

Dates & Times

  • Months count from 1; classes are immutable with private constructors; ignored return values are a classic trap.
  • Period = date units (not LocalTime); Duration = time units (not LocalDate); wrong type → UnsupportedTemporalTypeException.
