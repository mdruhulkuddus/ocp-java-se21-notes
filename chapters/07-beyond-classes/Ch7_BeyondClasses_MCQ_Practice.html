Q01INTERFACES

Which of the following are valid interface-related declarations? (Choose all that apply.)

1: public abstract interface WalksOnTwoLegs {}
2: final interface WalksOnEightLegs {}
3: public interface Climb { Number getSpeed(int age); }
4: public interface Dance { protected void step(); }
  • A. Line 1
  • B. Line 2
  • C. Line 3
  • D. Line 4
Show Answer
ANSWER: A, C

Line 1 compiles — abstract is optional on an interface. Line 3 compiles — a bodyless abstract method is fine. Line 2 fails: an interface cannot be final. Line 4 fails: protected conflicts with the implicit public.

Q02INTERFACES

Given the types below, which declarations compile? (Choose all that apply.)

interface CanRun {}
class Hyena {}
interface Nocturnal {}

A: class Cheetah extends CanRun {}
B: interface HasFur extends Hyena {}
C: interface HasBigEyes extends Nocturnal {}
D: class Owl implements Nocturnal {}
  • A. Cheetah
  • B. HasFur
  • C. HasBigEyes
  • D. Owl
Show Answer
ANSWER: C, D

An interface may extends another interface (C), and a class may implements an interface (D). A class cannot extends an interface (A), and an interface cannot extends a class (B).

Q03INTERFACES

What is the result of compiling the following?

interface Herbivore { void eatPlants(int p); }
interface Omnivore  { int  eatPlants(int p); }
public class Tiger implements Herbivore, Omnivore {}
  • A. Everything compiles
  • B. Tiger does not compile
  • C. Herbivore does not compile
  • D. Omnivore does not compile
Show Answer
ANSWER: B

The two inherited methods are incompatible — one returns void, the other int. No single override can satisfy both, so Tiger does not compile regardless of its body. The interfaces themselves are fine.

Q04INTERFACES

Which lines do not compile? (Choose all that apply.)

public interface Dance {
   1: int MAX = 4;
   2: private int count = 4;
   3: protected void step();
   4: default void move() {}
}
  • A. Line 1
  • B. Line 2
  • C. Line 3
  • D. Line 4
Show Answer
ANSWER: B, C

Line 1 is a valid constant (implicitly public static final) and Line 4 is a valid default method. Line 2 fails: interface variables are implicitly public, so private conflicts. Line 3 fails: protected conflicts with the implicit public (interfaces have no protected members).

Q05this/super

Cat inherits two conflicting default methods and overrides them. Which expression correctly calls Walk's hidden version?

interface Walk { default int getSpeed() { return 5; } }
interface Run  { default int getSpeed() { return 10; } }
public class Cat implements Walk, Run {
   public int getSpeed() { return 1; }
   public int getWalkSpeed() { return ________; }
}
  • A. Walk.getSpeed()
  • B. Walk.this.getSpeed()
  • C. Walk.super.getSpeed()
  • D. super.Walk.getSpeed()
Show Answer
ANSWER: C

The special syntax is InterfaceName.super.method(). Walk.getSpeed() would call it as a static method (it isn't one), and Walk.this.getSpeed() does not work for default methods.

Q06INTERFACES

What is the result?

public interface Hop { static int getJumpHeight() { return 8; } }
public class Bunny implements Hop {
   public int test() { return getJumpHeight(); }
}
  • A. Compiles; test() returns 8
  • B. Does not compile — must be Hop.getJumpHeight()
  • C. Compiles; test() returns 0
  • D. Hop does not compile
Show Answer
ANSWER: B

Static interface methods are not inherited. Even though Bunny implements Hop, the call must be qualified with the interface name: Hop.getJumpHeight().

Q07INTERFACES

Which method does not compile?

public interface ZooTrainTour {
   abstract int getTrainName();
   private static void ride() {}
   default void playHorn() { getTrainName(); ride(); }
   public static void slowDown() { playHorn(); }
   static void speedUp() { ride(); }
}
  • A. playHorn()
  • B. slowDown()
  • C. speedUp()
  • D. All compile
Show Answer
ANSWER: B

slowDown() is static and tries to call the default method playHorn() without an instance reference, so it fails. playHorn() (a default method) may call the abstract method and the private static ride(); speedUp() (static) may call the private static ride().

Q08ENUMS

Given enum Season { WINTER, SPRING, SUMMER, FALL }, what is the result of:

Season t = Season.valueOf("summer");
  • A. Returns SUMMER
  • B. Returns null
  • C. Throws IllegalArgumentException at runtime
  • D. Does not compile
Show Answer
ANSWER: C

valueOf() requires an exact match. There is no value named summer (only SUMMER), so it throws IllegalArgumentException.

Q09ENUMS

Which lines do not compile? (Choose all that apply.)

// Season s is a Season enum value
1: if (Season.SUMMER == 2) {}
2: if (Season.SUMMER == Season.FALL) {}
3: switch(s) { case 0 -> "x"; default -> "y"; }
4: System.out.println(Season.SUMMER.ordinal());
  • A. Line 1
  • B. Line 2
  • C. Line 3
  • D. Line 4
Show Answer
ANSWER: A, C

Line 1 compares an enum to an int — illegal. Line 3 uses an ordinal literal in a case — illegal. Line 2 (enum-to-enum ==) and Line 4 (ordinal()) are both fine.

Q10ENUMS

Which statements about enums are valid? (Choose all that apply.)

  • A. public enum Color extends Base {}
  • B. public final enum Color {}
  • C. public enum Color implements Paintable {}
  • D. Two enum values can be compared with ==
Show Answer
ANSWER: C, D

An enum may implement an interface (C) and its values compare safely with == (D). An enum cannot extends anything (A) and cannot be marked final (B).

Q11ENUMS

What is the result?

public class Favorites {
   enum Flavors {
      VANILLA, CHOCOLATE, STRAWBERRY;
      public Flavors() {}
   }
}
  • A. Compiles fine
  • B. Does not compile — an enum constructor cannot be public
  • C. Does not compile — missing semicolon after the values
  • D. Compiles but throws at runtime
Show Answer
ANSWER: B

Enum constructors are implicitly private; marking one public (or protected) does not compile. The semicolon after the values is present, so C is wrong.

Q12ENUMS

The constructor prints "c,". What is the output of main?

enum SeasonV {
   WINTER("Low"), SPRING("Med"), SUMMER("High"), FALL("Med");
   SeasonV(String v) { System.out.print("c,"); }
}
// main:
System.out.print("begin,");
var a = SeasonV.SUMMER;
System.out.print("middle,");
var b = SeasonV.SUMMER;
System.out.print("end");
  • A. begin,c,middle,c,end
  • B. begin,c,c,c,c,middle,end
  • C. begin,middle,end
  • D. begin,c,middle,end
Show Answer
ANSWER: B

The first time any value is referenced, all four values are constructed (4 × c,). The second reference to SUMMER constructs nothing.

Q13ENUMS

What is the result?

public enum SeasonWithTimes {
   WINTER { public String getHours() { return "10am-3pm"; } },
   SUMMER { public String getHours() { return "9am-7pm"; } },
   SPRING, FALL;
   public abstract String getHours();
}
  • A. Compiles
  • B. Does not compile — SPRING and FALL must implement getHours()
  • C. Does not compile — an enum cannot declare an abstract method
  • D. Compiles but throws at runtime
Show Answer
ANSWER: B

When an enum declares an abstract method, every value must implement it. WINTER and SUMMER do, but SPRING and FALL don't — so it fails. (Giving getHours() a concrete body instead would fix it.)

Q14SEALED

What is the result?

public class sealed Frog permits GlassFrog {}
public final class GlassFrog extends Frog {}
  • A. Compiles
  • B. Does not compile — should be public sealed class Frog
  • C. Does not compile — GlassFrog must be non-sealed
  • D. Does not compile — permits isn't allowed here
Show Answer
ANSWER: B

The modifier order is wrong — sealed must come before class. A final subclass is a valid choice, so C is wrong.

Q15SEALED

What is the result, assuming both types are in the same file?

public sealed class Bird permits Eagle {}
class Eagle extends Bird {}
  • A. Compiles
  • B. Does not compile — Eagle must be final, sealed, or non-sealed
  • C. Does not compile — Eagle must be in a different package
  • D. Compiles, but Eagle cannot be extended
Show Answer
ANSWER: B

Every direct subclass of a sealed class must declare exactly one of final, sealed, or non-sealed. Plain class Eagle has none, so it fails.

Q16SEALED

What is the result (all in one file)?

abstract sealed class Mammal permits Feline {}
non-sealed class Feline extends Mammal {}
class Tiger extends Feline {}
class BengalTiger extends Tiger {}
  • A. All four compile
  • B. Tiger does not compile
  • C. BengalTiger does not compile
  • D. Feline does not compile
Show Answer
ANSWER: A

Feline is non-sealed, which reopens the hierarchy. Tiger and BengalTiger need no special modifier and may extend freely. Mammal is still effectively sealed to Feline and its subclasses via polymorphism.

Q17SEALED

This does not compile. What is the fix?

public sealed class Snake permits Cobra {
   final class Cobra extends Snake {}
}
  • A. It already compiles
  • B. Use permits Snake.Cobra
  • C. Make Cobra non-sealed
  • D. A nested class cannot be final
Show Answer
ANSWER: B

A nested permitted subclass must be referenced through the enclosing namespace: permits Snake.Cobra. (When all subclasses are nested, omitting permits entirely is the cleaner option.)

Q18RECORDS

What is printed?

public record Crane(int numberEggs, String name) {}
var c = new Crane(0, "Craig");
System.out.println(c.name());
System.out.println(c);
  • A. getName() form; then Crane(0, Craig)
  • B. Craig; then Crane[numberEggs=0, name=Craig]
  • C. name; then {numberEggs=0, name=Craig}
  • D. Does not compile — accessor must be getName()
Show Answer
ANSWER: B

Record accessors are the bare field name plus () — name(), not getName(). The generated toString() prints Crane[numberEggs=0, name=Craig].

Q19RECORDS

Given record Crane(int numberEggs, String name) {}, which lines do not compile? (Choose all that apply.)

1: new Crane(4, "Cammy");
2: new Crane("Cammy", 4);
3: new Crane("Cammy");
4: new Crane(0, "Craig");
  • A. Line 1
  • B. Line 2
  • C. Line 3
  • D. Line 4
Show Answer
ANSWER: B, C

The generated constructor expects (int, String) in that order. Line 2 has the arguments reversed; Line 3 is missing one. Lines 1 and 4 match the declaration.

Q20RECORDS

What is the result?

public record Crane(int numberEggs, String name) {
   public Crane {
      this.numberEggs = 10;
   }
}
  • A. Compiles; numberEggs is always 10
  • B. Does not compile — a compact constructor cannot assign fields via this
  • C. Does not compile — a compact constructor needs parentheses
  • D. Compiles but throws at runtime
Show Answer
ANSWER: B

A compact constructor can transform the constructor parameters but cannot set the record's fields with this.field. Note that a compact constructor is correctly written without parentheses, so C is wrong.

Q21RECORDS

Which lines do not compile? (Choose all that apply.)

public record Crane(int numberEggs, String name) {
   1: private static int TYPE = 10;
   2: public int size;
   3: static { System.out.print("Hi"); }
   4: { System.out.print("Bye"); }
}
  • A. Line 1
  • B. Line 2
  • C. Line 3
  • D. Line 4
Show Answer
ANSWER: B, D

Records forbid instance fields outside the header (Line 2) and instance initializers (Line 4). Static fields (Line 1) and static initializers (Line 3) are allowed.

Q22RECORDS

Given record Monkey(String name, int age) {} and an Object animal, which lines do not compile? (Choose all that apply.)

1: if (animal instanceof Monkey m) {}
2: if (animal instanceof Monkey(String n, int a) m) {}
3: if (animal instanceof Monkey(String n, long a)) {}
4: if (animal instanceof Monkey(Object n, int a)) {}
  • A. Line 1
  • B. Line 2
  • C. Line 3
  • D. Line 4
Show Answer
ANSWER: B, C

Line 2 names both the record and its fields — not allowed. Line 3 uses long where the field is int — numeric promotion isn't supported in record patterns. Line 1 is simple binding; Line 4 works because String is compatible with Object.

Q23RECORDS

What does the program print, in order?

record Snake(Object data) {}
long showData(Snake snake) {
   return switch(snake) {
      case Snake(Long hiss)   -> hiss + 1;
      case Snake(Integer nag) -> nag + 10;
      case Snake(Number crow) -> crow.intValue() + 100;
      case Snake(Object kaa)  -> -1;
   };
}
// showData(new Snake(1)), showData(new Snake(2L)), showData(new Snake(3.0))
  • A. 1, 2, 3
  • B. 11, 3, 103
  • C. 11, 3, 100
  • D. -1, -1, -1
Show Answer
ANSWER: B

new Snake(1) autoboxes to Integer → 1 + 10 = 11. 2L is a Long → 2 + 1 = 3. 3.0 is a Double (not Long/Integer) but is a Number → 3 + 100 = 103.

Q24NESTED

Which lines do not compile? (Choose all that apply.)

public class Fox {
   private class Den {}
   1: public void goHome() { new Den(); }
   2: public static void visitFriend() { new Den(); }
}
public class Squirrel {
   3: public void visitFox() { new Den(); }
}
  • A. Line 1
  • B. Line 2
  • C. Line 3
  • D. None of them
Show Answer
ANSWER: B, C

An inner class needs an instance of the outer class. Line 1 has one (instance method). Line 2 is in a static method with no Fox instance. Line 3 is outside Fox entirely, and Den is private — two reasons to fail.

Q25NESTED

Which is true about lines A and B?

class Emu1 {
   String name = "Emmy";
   A: static Feathers createFeathers() { return new Feathers("grey"); }
   record Feathers(String color) {
      B: void fly() { System.out.print(name + " is flying"); }
   }
}
  • A. Both A and B compile
  • B. Line A compiles; line B does not
  • C. Line A does not compile; line B does
  • D. Neither compiles
Show Answer
ANSWER: B

A nested record is implicitly static, so createFeathers() can build one with no outer instance (A compiles). But a static nested type cannot reach the outer instance variable name, so line B does not compile.

Q26NESTED

What is the result?

public void processData() {
   final int length = 5;
   int width = 10;
   class VolumeCalculator {
      public int multiply() { return length * width; }
   }
   width = 2;
}
  • A. Compiles
  • B. Does not compile — width is not effectively final
  • C. Does not compile — length cannot be final
  • D. Does not compile — a local class needs an access modifier
Show Answer
ANSWER: B

A local class may capture only final or effectively final locals. Because width is reassigned (width = 2), it is not effectively final, so the local class does not compile. Local classes have no access modifier — that is normal, not an error.

Q27NESTED

Which statement about anonymous classes is correct?

  • A. It may extend one class and implement one interface at the same time
  • B. It must extend exactly one class or implement exactly one interface
  • C. It can be declared abstract
  • D. It requires an access modifier
Show Answer
ANSWER: B

An anonymous class extends a single class or implements a single interface — never both (unless the class is java.lang.Object). It cannot be abstract or final and takes no access modifier.

Q28POLYMORPHISM

Lemur extends Primate implements HasTail and declares int age. Which lines do not compile? (Choose all that apply.)

Lemur lemur = new Lemur();
Primate primate = lemur;
HasTail hasTail = lemur;
1: System.out.println(hasTail.age);
2: System.out.println(primate.hasHair());
3: Lemur l3 = primate;
4: System.out.println(hasTail.isTailStriped());
  • A. Line 1
  • B. Line 2
  • C. Line 3
  • D. Line 4
Show Answer
ANSWER: A, C

Line 1 fails because age isn't visible through a HasTail reference. Line 3 fails because assigning a supertype to a subtype needs an explicit cast. Lines 2 and 4 call members visible through their reference types.

Q29CASTING

What is the result?

interface Canine {}
interface Dog {}
class Wolf implements Canine {}

Wolf wolfy = new Wolf();
Dog badWolf = (Dog) wolfy;
  • A. Does not compile
  • B. Compiles and runs without error
  • C. Compiles but throws ClassCastException at runtime
  • D. Compiles only because Wolf is final
Show Answer
ANSWER: C

Because some subclass of Wolf could implement Dog, the compiler allows the cast — but at runtime the actual object isn't a Dog, so it throws ClassCastException. If Wolf were final, the cast would fail to compile instead.

Q30HIDING

What is the output?

class Penguin {
   public static int getHeight() { return 3; }
   public void printInfo() { System.out.println(this.getHeight()); }
}
public class CrestedPenguin extends Penguin {
   public static int getHeight() { return 8; }
   public static void main(String[] args) {
      new CrestedPenguin().printInfo();
   }
}
  • A. 8
  • B. 3
  • C. Does not compile
  • D. Throws at runtime
Show Answer
ANSWER: B

getHeight() is static, so it is hidden, not overridden. printInfo() is defined in Penguin, so its call to getHeight() resolves to Penguin's version → 3. (Had it been an instance method, the answer would be 8.)

Quick Revision

  • Interfaces: implicit public; constants are public static final; static methods need the interface-name prefix; InterfaceName.super.m() reaches a hidden default.
  • Enums: values first + semicolon; constructors implicitly private and run once; valueOf is exact-match; an abstract enum method forces every value to implement it.
  • Sealed: direct subclasses are final/sealed/non-sealed; same package; permits optional in the same file or when nested (then use Outer.Inner).
  • Records: implicitly final; accessor is name(); compact constructors transform parameters only; no instance fields/initializers outside the header.
  • Nested: inner needs an outer instance; nested records are implicitly static; local classes capture only effectively-final locals; anonymous classes extend one class or one interface.
  • Polymorphism: object type owns the data, reference type controls access; instance methods override (object), statics & variables hide (reference); unrelated-interface casts compile but may throw ClassCastException.
