অধ্যায় ৭ — Beyond Classes

Interface, enum, sealed class, record, nested class এবং polymorphism — সাধারণ class-এর বাইরে Java-র টাইপ সিস্টেম।

এই অধ্যায়ে যা শিখবেন

  • Interface তৈরি ও ব্যবহার; functional interface চেনা; private, static এবং default interface method ব্যবহার।
  • field, method এবং constructor সহ enum টাইপ তৈরি।
  • abstract ও sealed টাইপ এবং record দিয়ে inheritance প্রয়োগ।
  • Encapsulation প্রয়োগ ও immutable object তৈরি; nested class object তৈরি।
  • Polymorphism প্রয়োগ; object type বনাম reference type; casting এবং pattern matching সহ instanceof ব্যবহার।

এই পেজে যা আছে

  2. Enum নিয়ে কাজ

SECTION 01

Interface বাস্তবায়ন

একটি interface হলো এমন একটি abstract data type, যা কিছু abstract method ঘোষণা করে — implementing class-কে সেগুলো অবশ্যই দিতে হয়। একটি class কেবল একটিমাত্র class extends করতে পারে, কিন্তু comma দিয়ে আলাদা করে যত খুশি interface implements করতে পারে। এভাবেই Java টাইপের multiple inheritance পায়।

Interface ঘোষণা ও ব্যবহার

interface keyword দিয়ে interface ঘোষণা হয়। Interface-এর variable গুলো constant — implicitly public static final — এবং ঘোষণার সময়ই constant value দিয়ে initialize করতে হয়। Abstract method গুলো implicitly public।

public abstract interface WalksOnTwoLegs {}   // compiles; abstract is optional

final interface WalksOnEightLegs {}            // DOES NOT COMPILE — interface can't be final

public class Biped {
   public static void main(String[] args) {
      var e = new WalksOnTwoLegs();           // DOES NOT COMPILE — can't instantiate interface
   }
}
Exam Trap

Abstract class-কে যেমন final করা যায় না, interface-কেও তেমনি যায় না — final করলে এর মানে দাঁড়ায় কোনো class কখনোই এটি implement করতে পারবে না। আর new SomeInterface() কখনোই লেখা যাবে না।

Interface implement করা concrete class-কে প্রতিটি inherited abstract method override করতে হবে এবং স্পষ্টভাবে public লিখতে হবে (interface-এর implicit public এখানে এমনি এমনি পাওয়া যায় না — override-এ আবার লিখতে হয়)। Return type covariant হতে পারে।

public interface Climb {
   Number getSpeed(int age);
}
public class FieldMouse implements Climb {
   @Override
   public Float getSpeed(int age) { return 2.0f; }  // Float is covariant with Number; must be public
}

Interface extend ও keyword মেশানো

একটি interface একসাথে একাধিক interface extends করতে পারে (class পারে না)। পরীক্ষায় প্রায়ই class ও interface-এর মধ্যে extends/implements অদলবদল করে ফাঁদ পাতা হয়।

অনুমোদিত

  • class implements interface(s)
  • class extends একটি class
  • interface extends interface(s)

DOES NOT COMPILE

  • class extends interface
  • interface implements interface
  • interface extends একটি class
public interface CanRun {}
public class Cheetah extends CanRun {}      // DOES NOT COMPILE — class can't extend interface

public class Hyena {}
public interface HasFur extends Hyena {}     // DOES NOT COMPILE — interface can't extend class

Duplicate abstract method inherit করা

দুটি inherited abstract method ঠিক আছে যদি একটিমাত্র override দুটোকেই satisfy করতে পারে (compatible signature, covariant return)। Parameter-এর নাম মিলতে হবে না — শুধু type মিলতে হবে।

public interface Herbivore { int eatPlants(int plantsLeft); }
public interface Omnivore  { int eatPlants(int foodRemaining); }
public class Bear implements Herbivore, Omnivore {
   public int eatPlants(int plants) { return plants - 1; }  // satisfies both
}

// Incompatible — one returns void, the other int:
public interface Herbivore { void eatPlants(int p); }
public interface Omnivore  { int  eatPlants(int p); }
public class Tiger implements Herbivore, Omnivore {}  // DOES NOT COMPILE — impossible to override both

Implicit modifier ও conflict

Compiler যে implicit modifier বসায়

  • Interface implicitly abstract।
  • Interface variable implicitly public static final (constant)।
  • Body ছাড়া interface method implicitly abstract।
  • private ছাড়া interface method implicitly public।

যেহেতু member গুলো implicitly public, তাই এর সাথে সাংঘর্ষিকভাবে protected বা private লিখলে compile হয় না। Interface protected বা package member সমর্থন করে না।

public interface Dance {
   private int count = 4;       // DOES NOT COMPILE — conflicts with implicit public
   protected void step();        // DOES NOT COMPILE — conflicts with implicit public
}
Exam Trap — visibility কমানো

Interface method public। Implementing class যদি override-এ দুর্বল (package) access দেয়, তাহলে compile হয় না: class Georgette implements Poodle { void play() {} } ব্যর্থ হয়, কারণ Poodle-এ play() public। তবে abstract class-এর package-access method-কে package access দিয়ে override করা ঠিক আছে।

default method

default method-এর নিয়ম

  1. কেবল interface-এর ভেতরে ঘোষণা করা যায়।
  2. default keyword ও body থাকতে হবে।
  3. Implicitly public।
  4. abstract, final বা static করা যায় না।
  5. Implementing class override করতে পারে।
  6. একই signature-এর দুটি default inherit করলে class-কে অবশ্যই override করতে হবে।
public interface Walk { default int getSpeed() { return 5; } }
public interface Run  { default int getSpeed() { return 10; } }
public class Cat implements Walk, Run {}   // DOES NOT COMPILE — duplicate default, ambiguous

// Fix by overriding, and reach a "hidden" version with InterfaceName.super:
public class Cat implements Walk, Run {
   public int getSpeed() { return 1; }
   public int getWalkSpeed() { return Walk.super.getSpeed(); }  // not Walk.this
}
Key Rule

default method একটি instance-এর সাথে যুক্ত। Interface-এর উপর সরাসরি কল করা যায় না: Dance.getRhythm() compile হয় না। কোনো inherited (hidden) default-এ পৌঁছাতে বিশেষ syntax InterfaceName.super.method() ব্যবহার করুন।

static ও private interface method

static interface method-এর নিয়ম

  1. static ও body থাকতে হবে।
  2. Access modifier না থাকলে implicitly public।
  3. abstract বা final করা যায় না।
  4. Inherit হয় না — interface-এর নাম দিয়ে কল করতে হবে।
public interface Hop { static int getJumpHeight() { return 8; } }

public class Bunny implements Hop {
   public void printDetails() {
      System.out.println(getJumpHeight());        // DOES NOT COMPILE — needs Hop. prefix
      System.out.println(Hop.getJumpHeight());    // OK
   }
}

private ও private static method কেবল interface-এর ভেতরে code-duplication কমাতে ব্যবহৃত হয়। একটি private static method interface-এর যেকোনো method থেকে কল করা যায়; non-static private method কেবল default ও অন্য non-static private method থেকে কল করা যায়। কোনোটিই implementing class থেকে কল করা যায় না। মনে রাখুন: static method object reference ছাড়া default/private instance method কল করতে পারে না।

SECTION 02

Enum নিয়ে কাজ

একটি enum হলো constant-এর একটি নির্দিষ্ট সেট, যা compile-time টাইপ-নিরাপত্তা দেয়। Value গুলো সাধারণত SNAKE_CASE-এ লেখা হয়। == বা equals() দিয়ে তুলনা করা যায়, কারণ প্রতিটি value JVM-এ একবারই তৈরি হয়।

public enum Season { WINTER, SPRING, SUMMER, FALL }

var s = Season.SUMMER;
System.out.println(Season.SUMMER);        // SUMMER (name via toString)
System.out.println(s == Season.SUMMER);   // true

সাধারণ method

Methodরিটার্ন করেনোট
values()সব value-র arrayfor-each loop-এ ব্যবহারযোগ্য
name()Stringঘোষিত নাম
ordinal()int0-based ঘোষণার ক্রম
valueOf(String)মিলে যাওয়া valueহুবহু মিলতে হবে, নইলে IllegalArgumentException
 
Season.valueOf("SUMMER");   // SUMMER
Season.valueOf("summer");   // IllegalArgumentException at runtime
Exam Trap

enum-কে int-এর সাথে তুলনা করা বা ordinal ব্যবহার করা যায় না যেখানে value চাওয়া হয়:

if (Season.SUMMER == 2) {} → DOES NOT COMPILE। switch-এ enum-এর জন্য case 0 -> → DOES NOT COMPILE।

enum extend করা বা final করা যায় না, কিন্তু এটি interface implement করতে পারে।

public enum ExtendedSeason extends Season {}  // DOES NOT COMPILE — can't extend an enum

switch-এ enum

সব value handle করা হলে (exhaustive) enum switch-এ default branch লাগে না। Java 21-এ case label-এ enum-এর নাম optional।

String getWeather(Season value) {
   return switch (value) {
      case SUMMER        -> "Too hot";
      case Season.WINTER -> "Too cold";   // name optional but allowed
      case SPRING, FALL  -> "Just right";
   };  // no default needed — all values covered
}

Complex enum: field, constructor, method

Key Rule

value list-এর বাইরে কিছু থাকলেই value-র তালিকা আগে আসবে এবং semicolon দিয়ে শেষ হবে। Enum constructor implicitly private; public/protected দিলে compile হয় না। প্রতিটি constructor enum প্রথমবার ব্যবহারের সময় একবারই চলে।

enum SeasonWithVisitors implements Visitors {
   WINTER("Low"), SPRING("Medium"), SUMMER("High"), FALL("Medium");  // ; required
   private final String visitors;
   public static final String DESCRIPTION = "Weather enum";
   private SeasonWithVisitors(String visitors) {     // implicitly private
      System.out.print("constructing,");
      this.visitors = visitors;
   }
   @Override public void printVisitors() { System.out.println(visitors); }
}

CONSTRUCTOR প্রতি VALUE-তে একবার চলে

  • SeasonWithVisitors.SUMMER দু'বার কল করলে প্রিন্ট হয় begin,constructing,constructing,constructing,constructing,middle,end
  • প্রথম ব্যবহারে চারটি value-ই construct হয়; দ্বিতীয় reference কিছুই construct করে না।

প্রতিটি value-র নিজস্ব method body

enum-এ abstract method ঘোষণা করলে প্রতিটি value-কে তা implement করতে হবে। অথবা একটি default body দিয়ে কেবল বিশেষ ক্ষেত্রগুলো override করুন।

public enum SeasonWithTimes {
   WINTER { public String getHours() { return "10am-3pm"; } },
   SUMMER { public String getHours() { return "9am-7pm"; } },
   SPRING, FALL;
   public String getHours() { return "9am-5pm"; }   // default for the rest
}
SECTION 03

Sealed Class

একটি sealed class নিয়ন্ত্রণ করে কোন কোন class এটি extend করতে পারবে। তিনটি keyword গুরুত্বপূর্ণ: sealed (নির্দিষ্ট type-এ extension সীমিত করে), permits (সেই type-গুলোর তালিকা দেয়) এবং non-sealed (sealed parent-কে অজানা subclass-এর জন্য আবার খুলে দেয়)।

Sealed class-এর নিয়ম

  • sealed ও (optionally) permits দিয়ে ঘোষণা।
  • sealed class ও তার direct subclass একই package-এ (বা named module-এ) থাকতে হবে।
  • প্রতিটি direct subclass অবশ্যই final, sealed, বা non-sealed-এর ঠিক একটি হবে।
  • subclass একই file-এ বা sealed class-এর ভেতরে nested হলে permits optional।
  • Interface-ও sealed হতে পারে; extending interface কেবল sealed বা non-sealed হতে পারে (কখনো final নয়)।
public class sealed Frog permits GlassFrog {}   // DOES NOT COMPILE — modifier order (sealed before class)

public abstract sealed class Mammal permits Wolf {}
public final class Wolf extends Mammal {}
public final class Tiger extends Mammal {}     // DOES NOT COMPILE — Tiger not in permits
Exam Trap — subclass-এর modifier

সবচেয়ে বেশি যাচাই হওয়া নিয়ম: sealed class-এর direct subclass-কে অবশ্যই final, sealed, বা non-sealed ঘোষণা করতে হবে। এর কোনোটি ছাড়া সাধারণ class Sub extends SealedParent {} compile হয় না।

final subclass

  • নির্দিষ্ট — আর extend করা যায় না।
  • final class Gazelle extends Antelope {}
  • class DamaGazelle extends Gazelle {} → DNC

sealed subclass

  • নিজের permits তালিকা দিতে হবে।
  • Indirect subclass শীর্ষ parent-এ থাকতে হয় না।
  • type-এর পুরো সেট তবুও compile-time-এ নির্দিষ্ট।
abstract sealed class Mammal permits Feline {}
non-sealed class Feline extends Mammal {}
class Tiger extends Feline {}            // OK — Feline is non-sealed
class BengalTiger extends Tiger {}       // OK — Tiger not final

permits বাদ দেওয়া ও nested subclass

subclass গুলো sealed class-এর একই file-এ থাকলে, বা এর ভেতরে nested হলে permits বাদ দেওয়া যায়। তবে nested subclass নাম দিয়ে উল্লেখ করলে enclosing namespace দিয়ে qualify করতে হবে।

public sealed class Snake permits Cobra {
   final class Cobra extends Snake {}     // DOES NOT COMPILE — needs Snake.Cobra
}
public sealed class Snake permits Snake.Cobra {
   final class Cobra extends Snake {}     // OK
}

Sealed type ও pattern-matching switch

enum-এর মতোই, sealed type একটি switch-কে default ছাড়া exhaustive হতে দেয় — কিন্তু কেবল তখনই যখন compiler প্রতিটি সম্ভাবনা দেখতে পায়। sealed abstract parent তা নিশ্চিত করে।

abstract sealed class Fish permits Trout, Bass {}
final class Trout extends Fish {}
final class Bass  extends Fish {}

String getType(Fish fish) {
   return switch (fish) {
      case Trout t -> "Trout!";
      case Bass  b -> "Bass!";
   };  // no default needed — but removing 'abstract' from Fish breaks this
}
SECTION 04

Encapsulation ও Record

Encapsulation instance variable-গুলো private করে এবং method-এর মাধ্যমে access বাধ্যতামূলক করে class-এর member রক্ষা করে। যতক্ষণ instance variable গুলো private, ততক্ষণ class টি well-encapsulated — getter/setter optional। Immutable করতে setter বাদ দিন।

Record boilerplate কমিয়ে দেয়

একটি record হলো compact, immutable, encapsulated data carrier। Compiler একটি constructor, প্রতি field-এ একটি accessor, এবং equals(), hashCode(), toString() তৈরি করে দেয়।

public record Crane(int numberEggs, String name) {}

var mommy = new Crane(4, "Cammy");
System.out.println(mommy.numberEggs());   // 4   (accessor is name(), NOT getName())
System.out.println(mommy.name());         // Cammy
System.out.println(mommy);                // Crane[numberEggs=4, name=Cammy]
Key Rule — generated member

Constructor-এর parameter ঘোষণার ক্রম অনুসরণ করে। Accessor হলো শুধু field-এর নাম + () — কোনো get/is prefix নেই। সমান field-এর দুটি record equals() এবং একই hashCode() দেয়। field-হীন record বৈধ: public record Crane() {}।

var m1 = new Crane("Cammy", 4);  // DOES NOT COMPILE — wrong order
var m2 = new Crane("Cammy");     // DOES NOT COMPILE — missing argument

Constructor: long, compact, overloaded

Compact constructor

  • বন্ধনী ছাড়া ঘোষণা: public Crane {}
  • parameter-কে validate/transform করে।
  • Java পরে আসল constructor চালায়।
  • this.x দিয়ে field assign করা যায় না।

Long ও overloaded

  • Long: ঘোষণার মতো একই param; প্রতিটি field set করতে হবে।
  • Overloaded: ভিন্ন param; প্রথম লাইন অবশ্যই this(...)।
  • কেবল long form this.x দিয়ে field set করতে পারে।
// Compact — transform the parameter, not the field:
public record Crane(int numberEggs, String name) {
   public Crane {                          // no parentheses!
      if (name == null || name.length() < 1)
         throw new IllegalArgumentException();
      name = name.substring(0,1).toUpperCase() + name.substring(1).toLowerCase();
      this.numberEggs = 10;                // DOES NOT COMPILE — can't set field in compact ctor
   }
}

// Empty long constructor leaves fields unset:
public record Crane(int numberEggs, String name) {
   public Crane(int numberEggs, String name) {}   // DOES NOT COMPILE — fields not assigned
}
Exam Trap — record member

Record implicitly final, কিছু extend করতে পারে না, এবং header-এর বাইরে instance field ও instance initializer নিষিদ্ধ। Static field ও static initializer অনুমোদিত।

public record Crane(int numberEggs, String name) {
   private static int TYPE = 10;            // OK — static field
   public int size;                          // DOES NOT COMPILE — instance field
   static { System.out.print("Hi"); }        // OK — static initializer
   { System.out.print("Bye"); }              // DOES NOT COMPILE — instance initializer
}
public record BlueCrane() extends Crane {}  // DOES NOT COMPILE — records are final

Record pattern matching (Java 21-এ নতুন)

Record deconstruction-এর নিয়ম

  • কোনো field দিলে, ক্রম অনুযায়ী সব field দিতে হবে।
  • field-এর নাম record-এর থেকে ভিন্ন হতে পারে।
  • প্রতিটি pattern type ঘোষিত type-এর সাথে compatible হতে হবে (numeric promotion নয়)।
  • record-এর নাম অথবা এর field — দুটোই একসাথে নয়।
record Monkey(String name, int age) {}

if (animal instanceof Monkey m) {}                          // OK — simple binding
if (animal instanceof Monkey(String n, int a) m) {}        // DOES NOT COMPILE — named record AND fields
if (animal instanceof Monkey(String n, long a)) {}         // DOES NOT COMPILE — numeric promotion
if (animal instanceof Monkey(Object n, int a)) {}         // OK — String is compatible with Object

RECORD-এ SWITCH — showData(Snake)

  • record Snake(Object data) {} — Long, Integer, Number, Object case দিয়ে match।
  • new Snake(1) → 11  ·  new Snake(2L) → 3  ·  new Snake(3.0) → 103
SECTION 05

Nested Class তৈরি

nested class হলো আরেকটি class-এর ভেতরে সংজ্ঞায়িত class। চার ধরনের হয়:

ধরনকোথায়outer instance দরকার?access modifier
Inner classmember level, non-staticহ্যাঁচারটিই
static nested classmember level, staticনাচারটিই
Local classmethod/block-এর ভেতরেn/aকোনোটিই নয়
Anonymous classinline ঘোষণা ও তৈরিn/aকোনোটিই নয়

Inner class-এর instance দরকার

একটি inner class যেকোনো access level ব্যবহার করতে পারে, outer-এর সব member (এমনকি private) access করতে পারে, এবং outer class-এর instance থেকে তৈরি করতে হয় — static context-এ অদ্ভুত outer.new Inner() syntax দিয়ে।

public class Fox {
   private class Den {}
   public void goHome() { new Den(); }            // OK — instance context
   public static void visitFriend() { new Den(); }  // DOES NOT COMPILE — static, no Fox instance
}
public class Squirrel {
   public void visitFox() { new Den(); }         // DOES NOT COMPILE — not in Fox, and Den is private
}

// Creating an inner instance from outside / static:
var home = new Home();
Room room = home.new Room();        // or: new Home().new Room();

একই নামের field বিভিন্ন nesting level-এ আলাদা করতে EnclosingClass.this.field ব্যবহার করুন: A.this.x, B.this.x।

static nested class ও nested record

static nested class outer instance ছাড়াই তৈরি হয় কিন্তু outer instance member-এ পৌঁছাতে পারে না। Nested record implicitly static, তাই একইভাবে আচরণ করে।

class Emu1 {
   String name = "Emmy";
   static Feathers createFeathers() { return new Feathers("grey"); }  // OK — record is static
   record Feathers(String color) {
      void fly() { System.out.print(name + " is flying"); }       // DOES NOT COMPILE — outer instance var
   }
}

Local ও anonymous class

Key Rule

একটি local class কেবল final বা effectively final local variable capture করতে পারে। একটি anonymous class ঠিক একটি class extend বা একটি interface implement করতে পারে — কখনো দুটোই নয় — এবং এর ঘোষণা semicolon দিয়ে শেষ হয়।

public void processData() {
   final int length = 5;
   int width = 10;
   class VolumeCalculator {
      public int multiply() { return length * width; }  // DOES NOT COMPILE — width not effectively final
   }
   width = 2;                                          // this reassignment breaks it
}

// Anonymous class — note the trailing semicolon:
SaleTodayOnly sale = new SaleTodayOnly() {
   int dollarsOff() { return 3; }
};
SECTION 06

Polymorphism বোঝা

Polymorphism একটি object-কে তার নিজের type, supertype, বা implemented interface-এর reference দিয়ে access করতে দেয় — supertype-এ উপরে ওঠার সময় কোনো cast লাগে না।

public class Lemur extends Primate implements HasTail {
   public boolean isTailStriped() { return false; }
   public int age = 10;
   public static void main(String[] args) {
      Lemur lemur = new Lemur();
      System.out.println(lemur.age);            // 10
      HasTail hasTail = lemur;
      System.out.println(hasTail.isTailStriped()); // false
      Primate primate = lemur;
      System.out.println(primate.hasHair());    // true
   }
}
Key Rule — object বনাম reference

object type ঠিক করে memory-তে কোন property আছে; reference type ঠিক করে কোন member কল করা যাবে। Supertype-এ reassign করলে member লুকায়, কিন্তু কখনো মুছে যায় না। hasTail.age ও primate.isTailStriped() compile হয় না, কারণ সেই reference দিয়ে ঐ member দৃশ্যমান নয়।

Casting-এর নিয়ম

চারটি casting নিয়ম

  1. Subtype → supertype: explicit cast লাগে না।
  2. Supertype → subtype: explicit cast লাগে।
  3. Incompatible type-এ invalid cast runtime-এ ClassCastException ছুঁড়ে।
  4. Compiler unrelated class type-এর মধ্যে cast নিষিদ্ধ করে।
Lemur lemur3 = primate;            // DOES NOT COMPILE — missing explicit cast
Bird bird = (Bird) fish;           // DOES NOT COMPILE — Fish and Bird unrelated
Exam Trap — interface cast

Compiler একটি unrelated interface-এ cast অনুমোদন করে (কোনো subclass হয়তো এটি implement করতে পারে), তাই compile হয় কিন্তু runtime-এ ClassCastException ছুঁড়ে। ব্যতিক্রম: source class যদি final হয়, compiler জানে কোনো subclass interface-টি implement করতে পারবে না, তাই compile হয় না। unrelated type-এ instanceof-ও compile হয় না।

Overriding বনাম hiding

একটি instance method override করলে তা সর্বত্র প্রতিস্থাপিত হয় — call object-এর প্রকৃত type-এ dispatch হয়। Hiding (static method ও সব variable) reference type দিয়ে নির্ধারিত হয়।

Overriding (instance method)

  • EmperorPenguin.getHeight() = 8; parent-এর printInfo() this.getHeight() কল করে → 8
  • object type-এ dispatch।

Hiding (static / variable)

  • static getHeight() hidden → parent-এর printInfo() প্রিন্ট করে 3
  • reference type দিয়ে নির্ধারিত, object দিয়ে নয়।
class Marsupial {
   protected int age = 2;
   public static boolean isBiped() { return false; }
}
public class Kangaroo extends Marsupial {
   protected int age = 6;
   public static boolean isBiped() { return true; }
   public static void main(String[] args) {
      Kangaroo joey = new Kangaroo();
      Marsupial moey = joey;
      System.out.println(joey.isBiped());  // true   (reference type Kangaroo)
      System.out.println(moey.isBiped());  // false  (reference type Marsupial)
      System.out.println(joey.age);        // 6      (variable hidden — ref type)
      System.out.println(moey.age);        // 2      (variable hidden — ref type)
   }
}
Remember

super.super দিয়ে grandparent-এর method-এ পৌঁছানো যায় না। Penguin.printInfo()-এর ভেতরে super.getHeight() Object-কে নির্দেশ করে এবং compile হয় না — child-এ printInfo() override করে সেখানে super.getHeight() কল করলে parent-এর value প্রিন্ট হবে।

সংক্ষিপ্ত রিভিউ

  • একটি class অনেক interface implements করে কিন্তু একটি class extends করে; একটি interface অনেক interface extends করে। class/interface-এর মধ্যে keyword মেশালে ব্যর্থ হয়।
  • Interface member implicitly public; variable public static final; body-হীন method abstract। default/static/private method-এ body লাগে; static method inherit হয় না (interface-এর নাম দিন)।
  • Enum: value আগে + semicolon; constructor implicitly private ও একবার চলে; interface implement করতে পারে কিন্তু extend করা যায় না; valueOf হুবহু-মিল।
  • Sealed: direct subclass final, sealed, বা non-sealed; একই package; same file বা nested হলে permits optional।
  • Record: implicitly final; compact constructor parameter transform করে (this.field নয়); header-এর বাইরে instance field/initializer নেই; accessor name()।
  • Nested: inner-এর outer instance দরকার; static nested ও nested record-এর নয়; local class কেবল effectively-final local capture করে; anonymous class একটি class বা একটি interface।
  • Polymorphism: object type data-র মালিক, reference type access নিয়ন্ত্রণ করে; instance method override হয় (object), static ও variable hide হয় (reference)।
