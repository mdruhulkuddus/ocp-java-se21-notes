Q01LAMBDA SYNTAX

Which lambdas are valid (each should return a boolean)? (Choose all that apply.)

  • A. () -> true
  • B. x, y -> x.startsWith("test")
  • C. (String x) -> x.startsWith("test")
  • D. (x, y) -> { return x.startsWith("test"); }
Show Answer
ANSWER: A, C, D

B fails: with more than one parameter, parentheses are required. A, C, and D are all valid forms (you don't have to use every parameter).

Q02LAMBDA SYNTAX

What is wrong with this lambda?

x -> { return x.startsWith("giraffe") }
  • A. Nothing — it is valid
  • B. Missing semicolon inside the braces
  • C. Missing parentheses around x
  • D. Missing return
Show Answer
ANSWER: B

When you use braces, the body is a normal block: the return statement needs a terminating semicolon.

Q03var

What is the result?

var invalid = (Animal a) -> a.canHop();
  • A. Compiles; type is the lambda's interface
  • B. Does not compile — not enough context for var + lambda
  • C. Compiles; invalid has type Object
  • D. Does not compile — the lambda needs braces
Show Answer
ANSWER: B

A lambda takes its type from context, and var also needs context to infer a type. With both on the same line, there isn't enough information to determine the functional interface.

Q04SAM RULE

Sprint has one abstract method sprint(). Which of these are functional interfaces? (Choose all that apply.)

A: interface Dash extends Sprint {}
B: interface Skip extends Sprint { void skip(); }
C: interface Sleep { private void snore(){} default int getZzz(){return 1;} }
D: interface Climb { void reach(); default void fall(){} static int up(){return 1;} }
  • A. Dash
  • B. Skip
  • C. Sleep
  • D. Climb
Show Answer
ANSWER: A, D

Dash inherits the single abstract sprint(). Climb has exactly one abstract method, reach() (default/static methods don't count). Skip has two abstract methods; Sleep has none.

Q05FUNCTIONAL INTERFACE

What is the result?

@FunctionalInterface
public interface Dance {
   void move();
   void rest();
}
  • A. Compiles
  • B. Does not compile — it has two abstract methods
  • C. Compiles but emits a warning
  • D. Does not compile — methods need bodies
Show Answer
ANSWER: B

@FunctionalInterface makes the compiler enforce the single-abstract-method rule. With two abstract methods, the annotation causes a compile error.

Q06FUNCTIONAL INTERFACE

Is Dive a functional interface?

public interface Dive {
   String toString();
   public boolean equals(Object o);
   public abstract int hashCode();
   public void dive();
}
  • A. Yes — dive() is the single abstract method
  • B. No — it has four abstract methods
  • C. No — toString() can't be declared in an interface
  • D. No — equals() makes it non-functional
Show Answer
ANSWER: A

toString(), equals(Object), and hashCode() match public methods of Object, so they don't count toward the SAM total. That leaves dive() as the only abstract method.

Q07FUNCTIONAL INTERFACE

Is Hibernate a functional interface?

public interface Hibernate {
   String toString();
   public boolean equals(Hibernate o);
   public abstract int hashCode();
   public void rest();
}
  • A. Yes — rest() is the only abstract method
  • B. No — equals(Hibernate) doesn't match Object, leaving two abstract methods
  • C. No — hashCode() is not allowed
  • D. No — toString() is not allowed
Show Answer
ANSWER: B

equals(Hibernate) has a different signature from Object.equals(Object), so it counts. Combined with rest(), that's two abstract methods — not functional.

Q08METHOD REF

Which method reference is equivalent to the lambda (s, p) -> s.startsWith(p)?

  • A. str::startsWith
  • B. String::startsWith
  • C. String::new
  • D. Math::startsWith
Show Answer
ANSWER: B

This is an unbound instance reference: String::startsWith. The first parameter s is the instance, and p is the argument passed to startsWith().

Q09METHOD REF

For an unbound reference like String::isEmpty (assigned to an interface whose method is boolean check(String text)), what is the supplied parameter used as?

  • A. A method argument
  • B. The instance the method is called on
  • C. A static class reference
  • D. It is ignored
Show Answer
ANSWER: B

It looks like a static call, but isEmpty() is an instance method. Java uses the runtime parameter as the instance on which the method is invoked.

Q10METHOD REF

Given var str = ""; StringChecker lambda = () -> str.startsWith("Zoo");, how can this be written as a method reference?

  • A. str::startsWith
  • B. str::startsWith("Zoo")
  • C. Both A and B work
  • D. Neither — there's no way to pass the "Zoo" argument
Show Answer
ANSWER: D

A method reference can bind the instance (str) but cannot also supply the literal argument "Zoo". Both attempts fail to compile, so this lambda cannot become a method reference.

Q11METHOD REF

What is printed?

interface Converter { long round(double num); }
Converter methodRef = Math::round;
System.out.println(methodRef.round(100.1));
  • A. 100
  • B. 100.1
  • C. Does not compile
  • D. 101
Show Answer
ANSWER: A

The Converter context (one double parameter, long return) tells Java to use Math.round(double). Math.round(100.1) is 100.

Q12BUILT-IN

What is the method name and parameter count of Supplier<T>?

  • A. get(), 0 parameters
  • B. accept(T), 1 parameter
  • C. apply(T), 1 parameter
  • D. test(T), 1 parameter
Show Answer
ANSWER: A

A Supplier generates a value from nothing: T get() with zero parameters.

Q13BUILT-IN

Which functional interface fills the blank?

____________<List> ex = x -> "".equals(x.get(0));
  • A. Function<List>
  • B. Consumer<List>
  • C. Predicate<List>
  • D. Supplier<List>
Show Answer
ANSWER: C

One parameter in, a boolean out, and a single generic type — that's a Predicate. (A Function would need a second generic for the return type.)

Q14BUILT-IN

You need something that returns a Boolean and takes a String. Which interface?

  • A. Predicate<String>
  • B. Function<String, Boolean>
  • C. Supplier<Boolean>
  • D. Consumer<String>
Show Answer
ANSWER: B

A Predicate returns the boolean primitive, not a Boolean object. Returning a Boolean while taking a String is Function<String, Boolean>.

Q15BUILT-IN

What is the result?

Function<List<String>> ex1 = x -> x.get(0);
  • A. Compiles
  • B. Does not compile — Function needs two type arguments
  • C. Does not compile — x.get(0) is invalid
  • D. Compiles; returns a String
Show Answer
ANSWER: B

Function<T, R> requires both the input type and the return type. Only one type argument is supplied, so it does not compile.

Q16BUILT-IN

What is the result?

UnaryOperator<Long> ex2 = (Long l) -> 3.14;
  • A. Compiles
  • B. Does not compile — it must return a Long, not a double
  • C. Does not compile — UnaryOperator needs two generics
  • D. Compiles; the double is autoboxed
Show Answer
ANSWER: B

A UnaryOperator<Long> takes a Long and must return a Long. Returning 3.14 (a double) breaks that contract.

Q17BUILT-IN

Which statement about UnaryOperator<T> and BinaryOperator<T> is correct?

  • A. They may use any types for inputs and output
  • B. They force all type parameters to be the same type; UnaryOperator extends Function, BinaryOperator extends BiFunction
  • C. They return void
  • D. UnaryOperator takes two parameters
Show Answer
ANSWER: B

UnaryOperator<T> extends Function<T, T> and BinaryOperator<T> extends BiFunction<T, T, T>, so input(s) and output share one type.

Q18CONVENIENCE

What is printed?

Function<Integer,Integer> before = x -> x + 1;
Function<Integer,Integer> after  = x -> x * 2;
System.out.println(after.compose(before).apply(3));
  • A. 7
  • B. 8
  • C. 6
  • D. 9
Show Answer
ANSWER: B

compose runs the argument first: before turns 3 into 4, then after doubles it to 8. (With andThen the order would reverse.)

Q19CONVENIENCE

What is printed?

Consumer<String> c1 = x -> System.out.print("1: " + x);
Consumer<String> c2 = x -> System.out.print(",2: " + x);
c1.andThen(c2).accept("Annie");
  • A. 1: Annie
  • B. 1: Annie,2: Annie
  • C. ,2: Annie1: Annie
  • D. 2: Annie,1: Annie
Show Answer
ANSWER: B

andThen runs the two consumers in sequence, passing the same input "Annie" to each: 1: Annie then ,2: Annie.

Q20CONVENIENCE

egg tests contains("egg") and brown tests contains("brown"). What does egg.and(brown.negate()) match?

  • A. Strings containing both egg and brown
  • B. Strings containing egg but not brown
  • C. Strings containing neither
  • D. Strings containing brown but not egg
Show Answer
ANSWER: B

negate() inverts brown, and and() requires both — so the result matches strings that contain egg and do not contain brown.

Q21PRIMITIVE

What is the single abstract method of BooleanSupplier?

  • A. get()
  • B. getAsBoolean()
  • C. test()
  • D. apply()
Show Answer
ANSWER: B

BooleanSupplier — the only boolean functional interface — defines boolean getAsBoolean().

Q22PRIMITIVE

What is the single abstract method name of IntUnaryOperator?

  • A. apply
  • B. applyAsInt
  • C. getAsInt
  • D. accept
Show Answer
ANSWER: B

When a primitive is returned, the method is renamed. The operator families use applyAs… — here applyAsInt.

Q23PRIMITIVE

Which functional interfaces can fill the blank? (Choose all that apply.)

var d = 1.0;
______________ f1 = x -> 1;
f1.applyAsInt(d);
  • A. DoubleToIntFunction
  • B. ToIntFunction<Double>
  • C. IntToDoubleFunction
  • D. IntFunction<Integer>
Show Answer
ANSWER: A, B

The blank needs a method named applyAsInt returning int. DoubleToIntFunction takes a double; ToIntFunction<Double> takes an object that the double autoboxes into. IntToDoubleFunction returns a double, and IntFunction's method is apply.

Q24PRIMITIVE

How does IntFunction<R> differ from Function<T, R>?

  • A. It returns void
  • B. It takes an int primitive, so only the return type generic <R> remains
  • C. It takes two parameters
  • D. Its method is named getAsInt
Show Answer
ANSWER: B

Because the input is a primitive int, the input generic is gone — only <R> (the object return type) is needed. Its method is still apply.

Q25VARIABLES

Which lambda parameter lists do not compile? (Choose all that apply.)

  • A. (var x, y) -> "Hello"
  • B. (var x, Integer y) -> true
  • C. (var x, var y) -> true
  • D. (Integer x, y) -> "goodbye"
Show Answer
ANSWER: A, B, D

All parameters must use one consistent format. A mixes var with none, B mixes var with a type, and D mixes a type with none. C is valid — all var.

Q26VARIABLES

What is the result?

(a, b) -> { int a = 0; return 5; }
  • A. Compiles
  • B. Does not compile — it redeclares the parameter a
  • C. Does not compile — missing semicolon
  • D. Compiles; the local a shadows the parameter
Show Answer
ANSWER: B

A lambda body can't declare a local variable with the same name as a lambda parameter already in scope.

Q27VARIABLES

Which variables can a lambda body access regardless of whether they're effectively final? (Choose all that apply.)

  • A. Instance variables
  • B. Static variables
  • C. Local variables
  • D. Lambda parameters
Show Answer
ANSWER: A, B, D

Instance variables, static variables, and lambda parameters are always accessible. Local variables (and method parameters) are accessible only when final or effectively final.

Q28VARIABLES

A method parameter is reassigned after a lambda references it. Where does the compiler report the error?

  • A. At the reassignment line
  • B. At the lambda's use of the variable
  • C. At the method declaration
  • D. There is no error
Show Answer
ANSWER: B

Reassigning a variable is fine in itself; the problem only exists from the lambda's perspective, so the error is reported at the line where the lambda uses the variable — even though the reassignment comes later.

Quick Revision

  • Syntax: drop parentheses only for a single untyped parameter; drop braces/return/semicolon only for a single expression. var x = lambda; never compiles.
  • SAM rule: exactly one abstract method. Object methods (toString, equals(Object), hashCode) don't count — but equals(SomeType) does.
  • Method refs: static (Math::round), bound (str::startsWith), unbound (String::startsWith — first param is the instance), constructor (String::new).
  • Built-ins: Supplier get / Consumer accept / Predicate test / Function apply; UnaryOperator & BinaryOperator force one shared type.
  • Convenience: compose runs its argument first; andThen runs the caller first; Predicate has and/or/negate.
  • Primitives & variables: method renamed to getAsInt/applyAsDouble; parameters share one format; captured locals must be effectively final.
