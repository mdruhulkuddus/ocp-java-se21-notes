অধ্যায় ৮ — Lambdas and Functional Interfaces

পঠনযোগ্য deferred execution: lambda syntax, single-abstract-method নিয়ম, method reference, এবং built-in functional interface।

এই অধ্যায়ে যা শিখবেন

  • java.util.function-এর built-in functional interface ও তাদের convenience method।

SECTION 01

Lambda লেখা

একটি lambda expression হলো এমন একটি code block যা চারপাশে পাস করা যায় — parameter ও body সহ একটি নামহীন method, যা একটি anonymous class-এর ভেতরে থাকে। Lambda deferred execution সম্ভব করে: code এখন লেখা হয় কিন্তু পরে চলে।

// Instead of a whole CheckIfHopper class, just pass the logic:
print(animals, a -> a.canHop());
print(animals, a -> a.canSwim());
print(animals, a -> !a.canSwim());

Syntax (এবং এর optional অংশ)

নিচের দুটি লাইন হুবহু একই কাজ করে। arrow operator -> parameter list থেকে body আলাদা করে।

a -> a.canHop()                          // shortest form
(Animal a) -> { return a.canHop(); }     // most verbose form

যা বাদ দেওয়া যায়

  • Parentheses — কেবল তখন, যখন একটিমাত্র parameter থাকে এবং এর type উল্লেখ করা হয় না।
  • Braces, return, ও semicolon — কেবল single-expression body-এর জন্য (brace ছাড়া)।
  • প্রতিটি declared parameter ব্যবহার করতে হবে না। আর s -> {} একটি বৈধ lambda।

বৈধ (boolean রিটার্ন)

  • () -> true
  • x -> x.startsWith("test")
  • (String x) -> x.startsWith("test")
  • (x, y) -> { return x.startsWith("test"); }

অবৈধ

  • x, y -> ... — parentheses নেই
  • x -> { x.startsWith("c"); } — return নেই
  • x -> { return x.startsWith("g") } — semicolon নেই
  • String x -> ... — type দিলে parentheses লাগে
Exam Trap — lambda + var

var invalid = (Animal a) -> a.canHop(); compile হয় না। একটি lambda context থেকে তার type পায়, আর var-এরও context লাগে — একসাথে কোন functional interface ব্যবহার হবে তা বোঝার মতো যথেষ্ট তথ্য নেই।

SECTION 02

Functional Interface

একটি functional interface হলো ঠিক একটিমাত্র abstract method-যুক্ত interface — single abstract method (SAM) নিয়ম। Lambda ও method reference কেবল functional interface-এ assign করা যায়।

@FunctionalInterface
public interface Sprint {
   public void sprint(int speed);
}
Key Rule — @FunctionalInterface

এই annotation compiler-কে SAM নিয়ম প্রয়োগ করতে বলে (লঙ্ঘন হলে error), কিন্তু এটি optional — একটিমাত্র abstract method-যুক্ত interface annotation থাকুক বা না থাকুক functional। interface Dance { void move(); void rest(); }-এ @FunctionalInterface দিলে compile হয় না (দুটি abstract method)।

কোনগুলো functional interface?

Interface (Sprint-এ sprint() আছে)Functional?কেন
interface Dash extends Sprint {}হ্যাঁএকমাত্র abstract sprint() inherit করে
interface Skip extends Sprint { void skip(); }নাদুটি abstract: sprint() + skip()
interface Sleep { private void snore(){} default int getZzz(){return 1;} }নাকোনো abstract method নেই
interface Climb { void reach(); default…; static…; private static…; }হ্যাঁএকমাত্র abstract: reach()

Object-method ব্যতিক্রম

যে abstract method-এর signature Object-এর কোনো public method-এর সাথে মেলে — toString(), equals(Object), hashCode() — তা SAM গণনায় ধরা হয় না, কারণ প্রতিটি implementing class এমনিতেই Object থেকে তা inherit করে।

public interface Soar { abstract String toString(); }   // NOT functional — toString doesn't count

public interface Dive {                // IS functional — dive() is the only SAM
   String toString();
   public boolean equals(Object o);
   public abstract int hashCode();
   public void dive();
}
Exam Trap — দেখতে এক, signature আলাদা

interface Hibernate { String toString(); boolean equals(Hibernate o); int hashCode(); void rest(); } functional নয়। equals(Hibernate) ≠ equals(Object), তাই এটি গণনায় ধরা হয় — ফলে দুটি abstract method থেকে যায় (equals(Hibernate) ও rest())। এছাড়া interface Object-এর সাথে অসঙ্গত method ঘোষণা করতে পারে না (যেমন int toString() compile হয় না)।

SECTION 03

Method Reference

একটি method reference (:: operator) deferred execution-এর জন্য lambda-র সংক্ষিপ্ত রূপ। একটি method reference ও তার সমতুল্য lambda runtime-এ একই আচরণ করে। চার ধরন আছে।

ধরন::-এর আগেউদাহরণসমতুল্য lambda
static methodClass nameMath::roundx -> Math.round(x)
instance method, bound objectvariable namestr::startsWiths -> str.startsWith(s)
instance method, unbound parameterClass nameString::isEmptys -> s.isEmpty()
constructorClass nameString::new() -> new String()
Key Rule — bound বনাম unbound

Unbound reference যেমন String::startsWith ↔ (s, p) -> s.startsWith(p)-এ, প্রথম parameter-টি হলো instance যার উপর method কল হয়; বাকিগুলো argument। দেখতে static মনে হলেও static নয়।

Exam Trap — সব lambda reference হয় না

সব method reference lambda-তে রূপান্তরিত হয়, কিন্তু উল্টোটা নয়। () -> str.startsWith("Zoo") method reference হতে পারে না — "Zoo" argument-টি পাস করার কোনো উপায় নেই। এখানে str::startsWith ও str::startsWith("Zoo") দুটোই compile হয় না।

যেহেতু overloaded method থাকে (যেমন Math.round double বা float নেয়), Java functional-interface context দিয়ে সঠিকটি বেছে নেয়। যদি কোনোটি না পায় বা একাধিক পায়, তাহলে compiler error দেয় (ambiguous-type error)।

SECTION 04

Built-in Functional Interface

মূল interface গুলো java.util.function-এ থাকে। <T> হলো input, <U> দ্বিতীয় input, <R> return। এই টেবিল মুখস্থ করুন — এটি সরাসরি যাচাই হয় এবং অধ্যায় ৯ ও ১০-এর ভিত্তি।

InterfaceMethodReturn# param
Supplier<T>get()T0
Consumer<T>accept(T)void1 (T)
BiConsumer<T,U>accept(T,U)void2 (T, U)
Predicate<T>test(T)boolean1 (T)
BiPredicate<T,U>test(T,U)boolean2 (T, U)
Function<T,R>apply(T)R1 (T)
BiFunction<T,U,R>apply(T,U)R2 (T, U)
UnaryOperator<T>apply(T)T1 (T)
BinaryOperator<T>apply(T,T)T2 (T, T)
 
Supplier<LocalDate> s   = LocalDate::now;       // get()
Consumer<String> c      = System.out::println;   // accept("Annie")
Predicate<String> p     = String::isEmpty;       // test("")  -> true
Function<String,Integer> f = String::length;    // apply("cluck") -> 5
BiFunction<String,String,String> b = String::concat;
UnaryOperator<String> u = String::toUpperCase;   // apply("chirp") -> CHIRP
Exam Trap — Predicate বনাম Function

"একটি Boolean রিটার্ন করে ও একটি String নেয়" — এটি Function<String,Boolean>, Predicate নয় — কারণ Predicate boolean primitive রিটার্ন করে। UnaryOperator/BinaryOperator সব type parameter একই type-এ বাধ্য করে, তাই BinaryOperator<Integer> সমতুল্য BiFunction<Integer,Integer,Integer>-এর চেয়ে ভালো।

Function<List<String>> ex1 = x -> x.get(0);   // DOES NOT COMPILE — Function needs 2 type args
UnaryOperator<Long> ex2 = (Long l) -> 3.14;     // DOES NOT COMPILE — must return Long, not double

Convenience (default) method

Functional interface একত্র করা

  • Predicate: and(), or(), negate() — যেমন egg.and(brown.negate())।
  • Consumer: andThen() — একই input-এ পরপর দুটি consumer চালায়।
  • Function: andThen() (এটি, তারপর arg) ও compose() (arg আগে, তারপর এটি)।

andThen বনাম compose

  • c1.andThen(c2) তারপর accept("Annie") → 1: Annie,2: Annie (একই input দুটিতে)।
  • after.compose(before) তারপর apply(3) → before: 3→4, after দ্বিগুণ করে 4→8।
SECTION 05

Primitive Functional Interface

Boxing এড়াতে double, int, ও long-এর জন্য primitive variant আছে — সাথে boolean-এর জন্য একটি। দুটি জিনিস বদলায়: generics প্রায়ই উধাও হয় (type নামের মধ্যেই থাকে), এবং primitive রিটার্ন করলে single abstract method-এর নাম বদলায়।

Remember — BooleanSupplier

একমাত্র boolean functional interface হলো BooleanSupplier, যার method getAsBoolean()। BooleanSupplier b = () -> true; তারপর b.getAsBoolean()।

SECTION 06

Lambda-তে Variable

Lambda-র চারপাশে variable তিন জায়গায় আসে: parameter list, body-তে declared local variable, এবং body থেকে referenced variable। প্রতিটিই পরীক্ষার প্রিয় ফাঁদ।

Parameter list — সর্বত্র একই format

নিচের তিনটি বিনিময়যোগ্য, এবং x-এর inferred type String:

Predicate<String> p = x -> true;
Predicate<String> p = (var x) -> true;
Predicate<String> p = (String x) -> true;
Key Rule — সামঞ্জস্যপূর্ণ format

প্রতিটি parameter একই format ব্যবহার করতে হবে: সব untyped, সব typed, বা সব var। Parameter-এ final বা annotation যোগ করা যায়: (final var x, @Deprecated var y) -> x.compareTo(y)।

(var x, y) -> "Hello"                  // DOES NOT COMPILE — mix of var and none
(var x, Integer y) -> true             // DOES NOT COMPILE — mix of var and type
(String x, var y, Integer z) -> true   // DOES NOT COMPILE — three formats mixed
(Integer x, y) -> "goodbye"            // DOES NOT COMPILE — mix of type and none

Body-তে local variable

(a, b) -> { int c = 0; return 5; }   // OK
(a, b) -> { int a = 0; return 5; }   // DOES NOT COMPILE — redeclares parameter a

চারপাশের variable reference করা

Lambda body যা access করতে পারে

  • Instance variable — সবসময় অনুমোদিত।
  • Static variable — সবসময় অনুমোদিত।
  • Local variable ও method parameter — কেবল final বা effectively final হলে।
  • Lambda parameter — সবসময় অনুমোদিত।
Exam Trap — error কোথায় পড়ে

captured local/parameter reassign হলে compiler error lambda-র ব্যবহারের জায়গায় দেখায়, reassignment-এর লাইনে নয় — এমনকি reassignment lambda-র পরে হলেও। মান assign করা শুধু lambda-র দৃষ্টিকোণ থেকেই সমস্যা।

সংক্ষিপ্ত রিভিউ

  • Lambda parentheses কেবল একটি untyped parameter-এর জন্য optional; braces/return/semicolon কেবল single-expression body-এর জন্য optional। var x = lambda; কখনো compile হয় না।
  • Functional interface-এ ঠিক একটি abstract method (SAM)। @FunctionalInterface প্রয়োগ করে কিন্তু বাধ্যতামূলক নয়। Object-signature method (toString, equals(Object), hashCode) গণনায় ধরা হয় না।
  • চার ধরনের method reference: static (Math::round), bound instance (str::startsWith), unbound instance (String::isEmpty, প্রথম param-ই instance), constructor (String::new)।
  • মূল interface: Supplier (0→T), Consumer/BiConsumer (→void), Predicate/BiPredicate (→boolean), Function/BiFunction (→R), UnaryOperator/BinaryOperator (একই type)।
  • Convenience: Predicate.and/or/negate, Consumer.andThen, Function.andThen/compose (compose-এ argument আগে চলে)।
  • Primitive-এ generics বাদ পড়ে ও method-নাম বদলায় (getAsInt, applyAsDouble); BooleanSupplier একমাত্র boolean।
  • Lambda parameter একই format; body parameter redeclare করতে পারে না; captured local/param effectively final হতে হবে, error lambda-তে দেখায়।
