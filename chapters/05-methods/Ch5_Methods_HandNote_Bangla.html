SECTION 01

Method ডিজাইন করা

একটা method declaration-এর অংশগুলোর একটা নির্দিষ্ট ক্রম আছে। Access modifier আর optional specifier গুলো return type-এর আগে যেকোনো ক্রমে আসতে পারে; কিন্তু return type-এর পরে ক্রম ফিক্সড — name → parameter list → exception list → body।

অংশ (Element)উদাহরণআবশ্যক?
Access modifierpublicনা
Optional specifierfinal / staticনা
Return typevoidহ্যাঁ
Method namenapহ্যাঁ
Parameter list(int minutes)হ্যাঁ (খালি হতে পারে)
Exception listthrows InterruptedExceptionনা
Method body{ ... }হ্যাঁ (abstract ছাড়া)

Access modifier ও optional specifier

public class ParkTrip {
  public void skip1() {}
  default void skip2() {} // DOES NOT COMPILE — default কোনো access modifier নয়
  void public skip3() {} // DOES NOT COMPILE — modifier return type-এর পরে
  void skip4() {}          // OK — package (default) access
}
⚠ EXAM TRAP

default একটা keyword (switch / interface-এ ব্যবহার হয়), কিন্তু এটা কখনোই access modifier নয়। Package access-এর জন্য কোনো keyword নেই — শুধু modifier বাদ দিলেই হয়।

public static final void bike3() {}  // OK
public final static void bike4() {}  // OK — specifier-এর ক্রম যেকোনো হতে পারে
public modifier void bike5() {}     // DOES NOT COMPILE — modifier আসল specifier নয়
public void final bike6() {}        // DOES NOT COMPILE — specifier return type-এর পরে
final public void bike7() {}        // OK (অদ্ভুত হলেও বৈধ: specifier আগে, access পরে)

Return type ও return statement

✓ KEY RULE

প্রতিটা method-এর একটা return type থাকতেই হবে — কিছু রিটার্ন না করলে void। Non-void method-এ অবশ্যই একটা reachable return থাকতে হবে যা assignable একটা value রিটার্ন করে।

public String hike4() {}            // DOES NOT COMPILE — return নেই
public hike5() {}                   // DOES NOT COMPILE — return type নেই
public String int hike6() {}        // DOES NOT COMPILE — দুটো return type
String hike7(int a) {              // DOES NOT COMPILE — return শুধু if-এর ভেতরে
  if (1 < 2) return "orange";
}

int getHeight2() { int t = 9L; return t; }  // DOES NOT COMPILE — long→int assign
int getHeight3() { long t = 9L; return t; } // DOES NOT COMPILE — long value int হিসেবে return
★ REMEMBER

Compiler if(1<2)-কে "সবসময় true" হিসেবে ধরে না; তাই গ্যারান্টিড return লাগবেই। আর reachable return-এর পরে dead code থাকলে সেটা শুধু warning দেয়, error নয়।

Method name ও parameter list

public void jog1() {}    // OK
public void 2jog() {}    // DOES NOT COMPILE — digit দিয়ে শুরু
public void Jog_$() {}   // OK — খারাপ স্টাইল, তবু বৈধ
public _() {}            // DOES NOT COMPILE — শুধু একটা underscore নিষিদ্ধ

public void run4(int a; int b) {} // DOES NOT COMPILE — ';' নয়, ',' লাগবে

Method signature

Signature = method name + parameter list। Parameter-এর নাম signature-এর অংশ নয়; শুধু type আর তাদের ক্রম গণনা হয়।

// DOES NOT COMPILE — একই signature (নাম গণনা হয় না)
public void visitZoo(String name, int waitTime) {}
public void visitZoo(String attraction, int rainFall) {}

// OK — type-এর ক্রম আলাদা
public void visitZoo(String name, int waitTime) {}
public void visitZoo(int rainFall, String attraction) {}
SECTION 02

Local ও Instance Variables

Local variables

  • Method/block-এর ভেতরে declare হয়
  • একমাত্র modifier: final
  • কোনো default value নেই — ব্যবহারের আগে assign করতেই হবে
  • Block শেষ হলে reference ধ্বংস হয়

Instance variables

  • Class-এর member
  • Modifiers: final, volatile, transient (+ access)
  • Default value পায় (0, null, false...)
  • final instance var কোনো default পায় না
⚠ EXAM TRAP — final reference বনাম contents

final শুধু reference লক করে, object নয়। তাই giraffe.setName("George") বা friends[2]=2 ঠিক আছে; কিন্তু giraffe বা friends-কে আবার reassign করা যাবে না।

final int rest;
if(isWeekend) rest = 5;
System.out.print(rest); // DOES NOT COMPILE — হয়তো assign-ই হয়নি

Effectively final

✓ KEY RULE

যে local variable assign হওয়ার পরে আর reassign হয় না, সেটা effectively final। দ্রুত টেস্ট: final যোগ করে দেখুন — তাও compile হলে এটা effectively final ছিল।

String name = "Harry the Hippo"; // effectively final
var size = 10;                  // নয় — নিচে increment হয়
boolean wet;
if(size > 100) size++;          // size reassign করছে
name.substring(0);              // result বাদ; name অপরিবর্তিত (immutable)
wet = true;                     // effectively final (একবার assign)
★ REMEMBER

Method/constructor-এর parameterগুলো আসলে pre-initialized local variable। Final / effectively-final ব্যাপারটা lambda ও local class-এ (Ch. 7–8) গুরুত্বপূর্ণ — সেগুলো শুধু final বা effectively-final local capture করতে পারে।

Final instance variables

final int age = 10;        // declaration-এ
final int fishEaten;        // instance initializer-এ
final String name;          // constructor-এ
{ fishEaten = 10; }
public PolarBear() { name = "Robert"; }

একটা final instance variable-কে ঠিক একবার assign করতে হবে: declaration-এ, instance initializer-এ, অথবা প্রতিটা constructor-এ। কোনো জায়গায় assign না হলে compiler error।

SECTION 03

Varargs নিয়ে কাজ

Varargs parameter তৈরির নিয়ম

  1. একটা method-এ সর্বোচ্চ একটি varargs parameter থাকতে পারে।
  2. Varargs parameter অবশ্যই parameter list-এর সবশেষে থাকতে হবে।
public void walk1(int... steps) {}            // OK
public void walk2(int start, int... steps) {} // OK
public void walk3(int... steps, int start) {} // DOES NOT COMPILE — শেষে নেই
public void walk4(int... a, int... b) {}      // DOES NOT COMPILE — দুটো varargs

Varargs কল করা

আপনি হয় একটা array পাঠাতে পারেন, নয় আলাদা আলাদা value (Java নিজে array বানিয়ে নেয়)। কিছু না পাঠালে length-0 array তৈরি হয়, null নয়।

public static void walkDog(int start, int... steps) {
  System.out.println(steps.length);
}
walkDog(1);                  // 0
walkDog(1, 2);               // 1
walkDog(1, 2, 3);            // 2
walkDog(1, new int[]{4,5});  // 2
walkDog(1, null);            // compile হয়, পরে NullPointerException
⚠ EXAM TRAP — null পাঠানো

null তো int নয়, তাই Java একে array reference ধরে নেয় (যেটা null)। পরে steps.length কল করতে গিয়ে runtime-এ NullPointerException হয় — কিন্তু compile ঠিকই হয়ে যায়।

SECTION 04

Access Modifiers প্রয়োগ

সবচেয়ে restrictive থেকে কম restrictive ক্রমে: private → package (keyword নেই) → protected → public।

যে class থেকে অ্যাক্সেসprivatepackageprotectedpublic
একই classহ্যাঁহ্যাঁহ্যাঁহ্যাঁ
একই package, অন্য classনাহ্যাঁহ্যাঁহ্যাঁ
অন্য package, subclassনানাহ্যাঁহ্যাঁ
অন্য package, unrelated classনানানাহ্যাঁ

protected-এর "two-phase" ফাঁদ

⚠ EXAM TRAP — এই অধ্যায়ের সবচেয়ে কঠিন পয়েন্ট

ভিন্ন package-এর একটা subclass থেকে protected অ্যাক্সেস নির্ভর করে member-টা কীভাবে ছোঁয়া হচ্ছে তার উপর:

  • Variable ছাড়া (inherited / this দিয়ে): allowed।
  • Variable দিয়ে: তখনই allowed যখন reference type একই class বা তার subclass — parent type হলে নয়।
package pond.swan;
public class Swan extends Bird {     // Bird আছে pond.shore-এ
  public void swim() {
    floatInWater();              // OK — inherited, variable নেই
    Swan other = new Swan();
    other.floatInWater();        // OK — reference type Swan
    Bird b = new Bird();
    b.floatInWater();            // DOES NOT COMPILE — Bird ref, ভিন্ন package
  }
}

একইভাবে Bird other = new Goose(); তারপর other.floatInWater(); fail করে — কারণ চেক হয় reference type Bird, runtime-এর Goose নয়।

SECTION 05

Static Data অ্যাক্সেস

static member কোনো নির্দিষ্ট instance-এর নয়, পুরো class-এর — সব instance-এর মধ্যে শেয়ার্ড। main() static বলেই JVM object ছাড়াই একে কল করতে পারে।

Reference দিয়ে static অ্যাক্সেস (null ফাঁদ)

⚠ EXAM TRAP — কোনো NullPointerException হয় না

Instance দিয়ে static member অ্যাক্সেস করলে object নয়, reference type দেখা হয়। তাই null reference হলেও কাজ করে — Java কখনো null-কে dereference করে না।

Snake s = new Snake();
System.out.println(s.hiss);  // 2
s = null;
System.out.println(s.hiss);  // 2 — তাও ঠিক, hiss static

// একটাই শেয়ার্ড hiss: 4 → 6 → 5
Snake.hiss = 4;
Snake a = new Snake(); a.hiss = 6;
Snake b = new Snake(); b.hiss = 5;
System.out.println(Snake.hiss); // 5

Class বনাম instance membership

✓ KEY RULE

একটা static member কোনো object reference ছাড়া instance member-কে ছুঁতে পারে না। কিন্তু instance member অনায়াসে static member ব্যবহার করতে পারে।

public static int count;
public static void addGorilla() { count++; }        // OK
public void babyGorilla() { count++; }              // OK
public static void announceToEveryone() {
  addGorilla();
  babyGorilla();   // DOES NOT COMPILE — static → instance method
}
public int total;
public static double average = total / count; // DOES NOT COMPILE — static-এ instance var

Constants ও static final

private static final int NUM_BUCKETS = 45;
NUM_BUCKETS = 5;  // DOES NOT COMPILE — final reassign করা যায় না

private static final String[] treats = new String[10];
treats[0] = "popcorn";  // OK — contents বদলানো যায়, reference নয়

Static initializers

14: private static int one;
15: private static final int two;
16: private static final int three = 3;
17: private static final int four;  // DOES NOT COMPILE — কখনো initialize হয়নি
18: static {
19:   one = 1;
20:   two = 2;
21:   three = 3;   // DOES NOT COMPILE — line 16-এ আগেই initialize
22:   two = 4;     // DOES NOT COMPILE — দ্বিতীয়বার assign
23: }
★ REMEMBER

একটা final static var-কে static initializer-এ একবার assign করা যায় (ওটাই সেই assignment)। Static initializer গুলো class প্রথম লোড হওয়ার সময়, declaration-এর ক্রমে চলে।

Static imports

import static java.util.Arrays;        // DOES NOT COMPILE — static import member-এর জন্য
import static java.util.Arrays.asList; // OK
static import java.util.Arrays.*;      // DOES NOT COMPILE — এটা "import static"
...
Arrays.asList("one"); // DOES NOT COMPILE — Arrays class import করা হয়নি
⚠ EXAM TRAP

সঠিক ক্রম import static (static import নয়)। Static import দিয়ে member আনা যায়, class কখনোই নয়। দুটো আলাদা class থেকে একই নামের member static-import করলে error।

SECTION 06

Method-এর মধ্যে ডেটা পাঠানো

Java পুরোপুরি pass-by-value। Method variable-এর একটা কপি পায়। Parameter-কে reassign করলে caller-এর উপর কোনো প্রভাব পড়ে না — কিন্তু শেয়ার্ড object-এর উপর mutating method কল করলে পড়ে।

Parameter reassign

int num = 4;
newNumber(num);
// num এখনো 4
String name = "Webby";
speak(name); // ভেতরে reassign
// name এখনো "Webby"

শেয়ার্ড object mutate

var name = new StringBuilder("Webby");
speak(name); // s.append("Georgette")
// প্রিন্ট হবে WebbyGeorgette
⚠ EXAM TRAP — ignored return value

Return করা value assign না করলে কিছুই বদলায় না। addTickets(tickets) বাদ পড়ে; guests = addGuests(guests) রাখা হয়।

int tickets = 2;
String guests = "abc";
addTickets(tickets);            // return বাদ → tickets = 2
guests = addGuests(guests);     // guests = "abcd"
System.out.println(tickets + guests); // 2abcd

Autoboxing ও unboxing

✓ KEY RULE — একবারে একটাই রূপান্তর

Java হয় primitive widen করবে, নয় autobox করবে — কিন্তু এক ধাপে দুটো একসাথে নয়।

long ears = e;             // OK — Integer→int unbox, তারপর int→long widen
Long badGorilla = 8;     // DOES NOT COMPILE — একসাথে widen+box

Character elephant = null;
char bad = elephant;       // NullPointerException — null unbox

c.jump(123L);  // DOES NOT COMPILE — long→int narrowing
g.rest(8);     // DOES NOT COMPILE — rest(Long): int→Long মানে widen+box

Integer[] winterHours = { 10.5, 17.0 }; // DOES NOT COMPILE — double≠Integer
SECTION 07

Method Overloading

Overloading = একই নাম, কিন্তু আলাদা parameter list। Return type, access modifier, specifier আর exception list overloading-এর জন্য অপ্রাসঙ্গিক।

public void fly(int n) {}
public int fly(int n) { return 1; } // DOES NOT COMPILE — শুধু return type আলাদা

public void fly(int n) {}
public static void fly(int n) {}        // DOES NOT COMPILE — একই signature
public void fly(int km) {}              // DOES NOT COMPILE — একই signature

public void fly(int[] lengths) {}
public void fly(int... lengths) {}        // DOES NOT COMPILE — varargs = array signature

Resolution-এর ক্রম

Java কীভাবে overload বাছে (বেশি → কম specific)

  1. Type অনুযায়ী exact match
  2. বড় primitive type (widening)
  3. Autoboxed type
  4. Varargs

Backward compatibility-র জন্য autoboxing ও varargs সবশেষে বিবেচিত হয়।

// Pelican: fly(String) vs fly(Object) → "string-object"
// Parrot: print(List<Integer>), print(CharSequence), print(Object) → "CIO"
// Ostrich: fly(int) vs fly(long) ; fly(123), fly(123L) → "int-long"

// Glider: প্রতি কলে most-specific জেতে → প্রিন্ট হয় "142"
glide("a");          // 1  → glide(String)
glide("a","b");      // 4  → glide(String,String), varargs নয়
glide("a","b","c");  // 2  → এখন শুধু glide(String...) ম্যাচ করে
★ REMEMBER

যখন একটা primitive overload (fly(int)) আর একটা wrapper overload (fly(Integer)) দুটোই থাকে, fly(3) primitive-টা বাছে — Java অযথা autoboxing-এর ঝামেলা এড়ায়।
