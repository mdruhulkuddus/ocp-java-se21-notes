Q1main()

Which are legal entry-point methods runnable from the command line? (Choose all that apply.)

  • A. private static void main(String[] args)
  • B. public static final main(String[] args)
  • C. public void main(String[] args)
  • D. public static final void main(String[] args)
  • E. public static void main(String[] args)
Show Answer D and E

E is the canonical signature you must memorize. D is the same with a redundant but legal final. A fails — main must be public. C fails — it must be static. B fails — it has no return type (main must return void).

Q2Packages

In which orders can these be assembled into a program that compiles? (Choose all that apply.)

X: class Rabbit {}
Y: import java.util.*;
Z: package animals;
  • A. X, Y, Z
  • B. Y, Z, X
  • C. Z, Y, X
  • D. Y, X
  • E. Z, X
Show Answer C, D, and E

package and import are both optional, but when present the order must be package → import → class. C (Z,Y,X) is the full correct order; D (Y,X) skips the optional package; E (Z,X) skips the optional import. A and B put things out of order; F (X,Z) puts the class first.

Q3Class Structure

Which are true? (Choose all that apply.)

public class Bunny {
   public static void main(String[] x) {
      Bunny bun = new Bunny();
} }
  • A. Bunny is a class.
  • B. bun is a class.
  • C. main is a class.
  • D. bun is a reference to an object.
  • E. The main() method doesn't run because the parameter name is x.
Show Answer A and D

Bunny is declared with public class Bunny (A). bun is a reference variable pointing to a Bunny object (D). main is a method, not a class. E is wrong: the parameter type (String[]) matters, not its name — x is perfectly fine.

Q4Identifiers

Which are valid Java identifiers? (Choose all that apply.)

  • A. _
  • B. _helloWorld$
  • C. true
  • D. java.lang
  • E. Public
  • F. 1980_s
  • G. _Q2_
Show Answer B, E, and G

Identifiers begin with a letter, currency symbol, or underscore, and may contain digits — but: a single underscore _ is illegal (A); reserved words like true are illegal (C); a period is not allowed (D); they can't start with a digit (F). Public is fine because it's capitalized — only the lowercase public is reserved (E). B and G use only legal characters.

Q5Garbage Collection

Which are correct about this program? (Choose all that apply.)

2: public class Bear {
3:    private Bear pandaBear;
4:    private void roar(Bear b) {
5:       System.out.println("Roar!");
6:       pandaBear = b;
7:    }
8:    public static void main(String[] args) {
9:       Bear brownBear = new Bear();
10:      Bear polarBear = new Bear();
11:      brownBear.roar(polarBear);
12:      polarBear = null;
13:      brownBear = null;
14:      System.gc(); } }
  • A. The object from line 9 is first eligible for GC after line 13.
  • B. The object from line 9 is first eligible for GC after line 14.
  • C. The object from line 10 is first eligible for GC after line 12.
  • D. The object from line 10 is first eligible for GC after line 13.
  • E. Garbage collection is guaranteed to run.
  • F. Garbage collection might or might not run.
Show Answer A, D, and F

GC is never guaranteed, so F is correct and E is wrong. The line-9 object (brownBear) is reachable until line 13 → A. The line-10 object (polarBear) is also reachable via brownBear.pandaBear (set on line 6), so nulling polarBear on line 12 isn't enough — it stays reachable until brownBear goes null on line 13 → D, not C.

Q6Scope

How many variables are in scope on line 14?

1: public class Camel {
2:    { int hairs = 3_000_0; }
3:    long water, air=2;
4:    boolean twoHumps = true;
5:    public void spit(float distance) {
6:       var path = "";
7:       { double teeth = 32 + distance++; }
8:       while(water> 0) {
9:          int age = twoHumps ? 1 : 2;
10:         short i=-1;
11:         for(i=0; i<10; i++) {
12:            var Private = 2;
13:         }
14:         // SCOPE
15:      }
16:   }
17: }
  • A. 4
  • B. 5
  • C. 6
  • D. 7
Show Answer D · 7

Trace the braces. The block variables on lines 2 and 7, and Private on line 12 (scoped to the for loop), are all out of scope at line 14. In scope: the three instance variables water, air, twoHumps (lines 3–4), plus the parameter distance wait — count the available ones: path (6), age (9), i (10) are inside the still-open method/while, plus the 3 instance vars and distance… the book's count is 7: water, air, twoHumps, distance, path, age, i.

Q7Text Blocks

Which are true about this code? (Choose all that apply.)

public class KitchenSink {
   private int numForks;
   public static void main(String[] args) {
      int numKnives;
      System.out.print("""
         "# forks = " + numForks +
          " # knives = " + numKnives +
         # cups = 0""");
   }
}
  • A. The output includes # forks = 0.
  • B. The output includes # knives = 0.
  • C. The output includes # cups = 0.
  • D. The output includes a blank line.
  • E. The output includes one or more lines that begin with whitespace.
  • F. The code does not compile.
Show Answer C and E

Everything inside the """ is just text — the variables are never evaluated (and conveniently, the uninitialized numKnives is never referenced, so it doesn't break compilation). So A and B are wrong. C matches the literal text. E is correct because " # knives is indented (essential whitespace). D is wrong — there's no trailing blank line.

Q8var

Which var snippets compile without issue inside a method? (Choose all that apply.)

  • A. var spring = null;
  • B. var fall = "leaves";
  • C. var evening = 2; evening = null;
  • D. var night = Integer.valueOf(3);
  • E. var day = 1/0;
  • F. var winter = 12, cold;
  • G. var morning = ""; morning = null;
Show Answer B, D, E, and G

A var can't be initialized to null with no type (A fails), but a non-primitive var can be reassigned null later — so G (String) compiles. B (String) and D (Integer) are clear. E compiles — 1/0 is a valid int expression; it throws only at runtime, but the question is about compiling. C fails — evening is an int (primitive), which can't take null. F fails — var can't be used in multiple-variable declarations.

Q9Default Values

Which is correct?

  • A. An instance variable of type float defaults to 0.
  • B. An instance variable of type char defaults to null.
  • C. A local variable of type double defaults to 0.0.
  • D. A local variable of type int defaults to null.
  • E. A class variable of type String defaults to null.
Show Answer E

Local variables (C, D) have no default at all — they must be assigned before use. Primitives never default to null (B, D wrong). A float default is written 0.0 with a decimal, not 0 (A wrong). Reference types like String in class/instance variables default to null → E.

Q10Numeric Literals

Which expressions, inserted into var magic = ___;, let the code compile? (Choose all that apply.)

  • A. 3_1
  • B. 1_329_.0
  • C. 3_13.0_
  • D. 5_291._2
  • E. 2_234.0_0
  • F. 9___6
  • G. _1_3_5_0
Show Answer A, E, and F

An underscore is allowed anywhere in a numeric literal except at the start, at the end, or adjacent to the decimal point — and multiple underscores in a row are fine. So A, E, and F (consecutive underscores) are valid. B and D put _ next to the .; C ends with _; G starts with _.

Q11Imports

What is the maximum number of imports that can be removed and still compile?

// Tank.java
package aquarium;
import java.lang.*;
import java.lang.System;
import aquarium.Water;
import aquarium.*;
public class Tank {
   public void print(Water water) {
      System.out.println(water); } }

  • A. 2
  • B. 3
  • C. 4
  • D. Does not compile

All four imports are removable. The two java.lang imports are redundant because java.lang is imported automatically. The two aquarium imports are redundant because Tank and Water are in the same package. (If they were in different packages, only one of the two could go.)

Q12Class Structure

Which are correct about this class? (Choose all that apply.)

1: public class ClownFish {

3:    { int fins = gills; }
4:    void print(int length = 3) {

Q13Imports

There are two Water classes — aquarium.Water and aquarium.jellies.Water. WaterFiller (in package employee) references Water. Which single import lets it compile? (Choose all that apply.)

  • A. import aquarium.*;
  • B. import aquarium.Water;
  • C. import aquarium.jellies.Water;
  • D. import aquarium.*; and import aquarium.jellies.*;
Show Answer A, B, and C

A wildcard-imports aquarium, which includes aquarium.Water. B and C import a specific Water by class name — both unambiguous. D fails: two wildcard imports each containing a Water create an ambiguous reference, so Java can't tell which to use. (Importing the same class name twice explicitly would also fail.)

Q14Data Types

Which lines generate a compiler error? (Choose all that apply.)

3: short numPets = 5L;
4: int numGrains = 2.0;
5: String name = "Scruffy";
6: int d = numPets.length();

  • A. Line 3

Q15Garbage Collection

Which statements about garbage collection are correct? (Choose all that apply.)

  • A. Calling System.gc() is guaranteed to free memory by destroying eligible objects.
  • B. Garbage collection runs on a set schedule.
  • C. Garbage collection lets the JVM reclaim memory for other objects.
  • D. An object may be eligible for GC but never removed from the heap.
  • E. An object is eligible for GC once no references to it are accessible.
  • F. Marking a variable final means its object is never garbage collected.
Show Answer C, D, and E

There are no guarantees about when GC runs, and the JVM may ignore System.gc() — so A and B are wrong. C states GC's purpose. D is true (e.g. the program ends before GC runs). E is the core eligibility rule. F is wrong — final means the variable can't be reassigned within its scope; the object is still collectible once it's unreachable.

Q16Text Blocks

Which are true? (Choose all that apply.)

var blocky = """
   squirrel \s
   pigeon   \
   termite""";
System.out.print(blocky);
  • A. It outputs two lines.
  • B. It outputs three lines.
  • C. There is one line with trailing whitespace.
  • D. There are two lines with trailing whitespace.
Show Answer A and C

The \ at the end of the pigeon line means "skip this line break," joining pigeon and termite into one line — so there are two lines total (A). Q18var

Which statements about var are true? (Choose all that apply.)

  • A. A var can be used as a constructor parameter.
  • B. The type of a var is known at compile time.
  • C. A var cannot be used as an instance variable.
  • D. A var can be used in a multiple-variable assignment.
  • E. The type of a var cannot change at runtime.
  • F. The word var is a reserved word in Java.
Show Answer B, C, and E

var is only for local variables — not constructor/method parameters or instance/class variables, so A is wrong and C is right. Its type is fixed at compile time (B) and can't change at runtime (E), though its value can. D is wrong — no multiple-variable declarations. F is wrong — var is a reserved type name, not a reserved word (you can still name a variable var).

What is the output?

1: public class Salmon {
2:    int count;
3:    { System.out.print(count+"-"); }
4:    { count++; }
5:    public Salmon() {
6:       count = 4;
7:       System.out.print(2+"-");

Line 10 prints 7-. Creating the object runs the two instance initializers in order: line 3 prints the current count (default 0) → 0-, then line 4 does count++ → 1. Then the constructor sets count = 4 and prints 2-. Finally line 12 prints s.count = 4-. Result: 7-0-2-4-.

Q24Class Structure

How many public top-level classes may a single .java file contain?

  • A. Zero or one.
  • B. Exactly one, always.
  • C. Up to two.
  • D. Any number.
Show Answer A · zero or one

A file may have at most one public top-level class, and if present it must match the file name. A file can legally contain multiple non-public top-level classes — or even no public class at all — so the answer is "zero or one," not "exactly one."

Q25main() Args

Running java Zoo.java Bronx Zoo on a class whose main body is System.out.println(args[1]); prints what?

  • A. Bronx
  • B. Zoo
  • C. Zoo.java
  • D. An ArrayIndexOutOfBoundsException.

Program arguments are zero-indexed and exclude the program name. So args[0] is "Bronx" and args[1] is "Zoo" → prints Zoo. (If only one argument were passed, args[1] would throw ArrayIndexOutOfBoundsException — but here two were given.)

Q26Initialization Order

In what order do these run when an instance is created?

  • A. Constructor, then instance initializer blocks, then field initializers.
  • B. Field initializers and instance initializer blocks in the order they appear, then the constructor.
  • C. The constructor first, then everything else top to bottom.
  • D. Instance initializers always run before all field initializers.
Show Answer B

Fields and instance initializer blocks run in the order they appear in the source, top to bottom; the constructor body runs last. (Within that, a field can't be read by an initializer that appears above its declaration.) The constructor never runs first.

Q28Data Types

Which statement about primitives and references is correct?

  • A. String is one of the eight primitive types.
  • B. A char can be assigned a small non-negative integer literal like 65.
  • C. A boolean can be assigned 0 or 1.
  • D. A long literal does not need an L suffix even when it exceeds int range.
Show Answer B

char is an integral type, so char c = 65; is valid (it becomes 'A') → B. String is a reference type, not one of the eight primitives (A wrong). boolean holds only true/false, never numbers (C wrong). A literal beyond int range needs the L suffix or it won't compile (D wrong).

Quick Revision

  • main(): must be public static void main(String[] args) — public, static, void, and a String[]/varargs parameter (any name). args[0] is the first program argument.
  • File order: package → import → class; both package and import are optional. At most one public top-level class per file (matching the file name).
  • Imports: java.lang.* and same-package classes are automatic. Class-name imports beat wildcards; two wildcards offering the same class name are ambiguous.
  • Identifiers: start with letter, $, or _; not a lone _; no reserved words; no leading digit. Case matters (Public ≠ public).
  • Literals: 0x hex, 0b binary; underscores allowed except at start/end or next to the decimal point (consecutive ones are OK). long beyond int range needs L; float needs f.
  • Defaults: instance/class fields default (objects → null, numeric → 0/0.0, boolean → false); local variables have no default. You can't call methods on primitives.
  • var: local variables only; single declaration; must be initialized on the same line; type fixed at compile time; can be reassigned (and set to null if non-primitive). Not a reserved word.
  • Init order: fields & instance initializer blocks run top-to-bottom, then the constructor. A void-returning "constructor" is really a method and won't run on new.
  • Scope & GC: a loop/block variable dies at its closing brace. An object is GC-eligible once unreachable; System.gc() is only a suggestion, and GC may never run.
