SECTION 01

Environment ও Class Structure

Java-র প্রধান component

JDK (Java Development Kit) হলো Java development-এর জন্য ন্যূনতম software। গুরুত্বপূর্ণ command:

Commandকাজ
javac.java source file-কে .class bytecode-এ রূপান্তর করে।
javaপ্রোগ্রাম চালায়।
jarfile-গুলো একসাথে package করে।
javadocdocumentation তৈরি করে।

javac bytecode তৈরি করে; java JVM চালু করে, যেটা সেই bytecode আসল machine-এ কীভাবে চালাতে হয় জানে। Java 11 থেকে আলাদা JRE নেই — প্রোগ্রাম চালাতেও পূর্ণ JDK ব্যবহার হয়।

★ REMEMBER

Java class file JVM-এ চলে, তাই Java আছে এমন যেকোনো machine-এ চলে — শুধু যেখানে compile হয়েছিল সেখানে নয়। Java 21 বের হয়েছিল সেপ্টেম্বর ২০২৩-এ।

Field ও method

একটা class-এর দুটো প্রধান উপাদান: method (function/procedure) এবং field (variable)। দুটো মিলে class-এর member। সবচেয়ে সহজ class হলো শুধু public class Animal { }।

public class Animal {
   String name;                       // a field
   public String getName() {           // method with String return type
      return name;
   }
   public void setName(String newName) { // void = returns nothing
      name = newName;                  // newName is a parameter
   }
}
✓ KEY RULE — method signature

method signature হলো method-এর নাম + parameter-এর type (নাম নয়, return type নয়)। public int numberVisitors(int month)-এর signature হলো numberVisitors(int)।

Comment

তিন রকম comment: single-line //, multiline /* ... */, এবং Javadoc /** ... */। Comment কোডের যেকোনো জায়গায় বসতে পারে।

⚠ EXAM TRAP — nested multiline comment

প্রথম /* থেকে প্রথম */ পর্যন্ত সবটা একটাই comment, ভেতরে // বা আরেকটা /* থাকলেও। নিচের line টা compile হয় না, কারণ একটা extra */ থেকে যায়:

/*
 * /* ferret */
 */

compiler ভেতরের */-এ comment শেষ ধরে, ফলে একটা বাড়তি */ থেকে যায় — invalid syntax।

Class ও source file

Top-level type-এর নিয়ম

  • top-level type-কে public হতে হবে না; class Animal { } ঠিক আছে।
  • এক file-এ একাধিক top-level type থাকতে পারে, কিন্তু সর্বোচ্চ একটা public হতে পারে।
  • type public হলে তার নাম filename-এর সাথে মিলতে হবে। Animal.java file-এ public class Animal2 → DOES NOT COMPILE।
SECTION 02

main() Method ও কোড চালানো

execution শুরু হয় main() দিয়ে — প্রোগ্রামের entry point, যেটা JVM খুঁজে নেয়।

public class Zoo {
   public static void main(String[] args) {
      System.out.println("Hello World");
   }
}

Compile ও run (খেয়াল: run করার সময় .class বাদ দিতে হয়):

javac Zoo.java   // produces Zoo.class
java Zoo         // runs it (no .class extension)

Signature-এর শব্দগুলো

Keywordঅর্থ
publicaccess modifier — যেকোনো জায়গা থেকে পূর্ণ access।
staticmethod-কে class-এর সাথে বাঁধে, তাই object ছাড়া Zoo.main() হিসেবে call করা যায়।
voidreturn type — কোনো data ফেরায় না।
String[] argsparameter list — String argument-এর একটা array।
✓ KEY RULE — তিনটা valid parameter form

compiler main-এর parameter-এ নিচের যেকোনোটা মেনে নেয় (এবং যেকোনো valid variable নাম):

String[] args
String options[]
String... friends   // varargs
★ REMEMBER — optional modifier

public ও static required, কিন্তু কয়েকটা optional modifier অনুমোদিত। নিচের দুটো final-ই optional এবং এটা এখনো valid entry point:

public final static void main(final String[] args) {}

Parameter পাঠানো

Argument class নামের পরে আসে এবং index শুরু হয় 0 থেকে। space দিয়ে argument আলাদা হয়; একটার ভেতরে space চাইলে quote ব্যবহার করুন।

java Zoo Bronx Zoo        // args[0]=Bronx, args[1]=Zoo
java Zoo "San Diego" Zoo   // args[0]=San Diego
⚠ EXAM TRAP — missing argument

যে index পাঠানো হয়নি সেটা পড়লে runtime-এ throw হয়। java Zoo Zoo দিয়ে তারপর args[1] পড়লে → ArrayIndexOutOfBoundsException (Index 1 out of bounds for length 1)।

★ REMEMBER — single-file source-code launch

আলাদা compile ধাপ বাদ দেওয়া যায়: java Zoo.java Bronx Zoo। মোচড়টা হলো — এখানে .java extension রাখতে হয় (compiled java Zoo form-এর উল্টো)।

SECTION 03

Package, Import ও Class Order

Java class-গুলো package-এ ভাগ করে। একটা import statement compiler-কে বলে কোন package-এ class খুঁজতে হবে। না দিলে error: cannot find symbol আসে।

Wildcard

import java.util.*; java.util-এর সরাসরি থাকা প্রতিটা class import করে।

✓ KEY RULE

wildcard শুধু সেই package-এর সরাসরি class import করে — child package, field বা method নয়। তাই import java.util.*; java.util.concurrent.atomic.AtomicInteger আনে না; তার জন্য import java.util.concurrent.atomic.*; দরকার।

⚠ EXAM TRAP — invalid import form 
import java.nio.*;            // NO GOOD — wildcard matches class names, not "file.Files"
import java.nio.*.*;          // NO GOOD — only one wildcard, and it must be last
import java.nio.file.Paths.*; // NO GOOD — cannot import methods, only class names

Redundant import

java.lang বিশেষ — এটা স্বয়ংক্রিয়ভাবে import হয়। এছাড়া current class-এর একই package-এর class-এর জন্য কোনো import লাগে না।

1: import java.lang.System;  // redundant (java.lang auto-imported)
2: import java.lang.*;       // redundant (java.lang auto-imported)
3: import java.util.Random;
4: import java.util.*;        // redundant here (Random already imported on line 3)

চারটার মধ্যে তিনটা redundant। line 3 না থাকলে java.util.* redundant হতো না — তখন সেটাই Random import করত।

Naming conflict

java.util.Date ও java.sql.Date দুটোই আছে। দুটো wildcard-এ যদি দুই জায়গায় Date থাকে, ব্যবহার ambiguous হয়ে যায় ও compile হয় না।

Conflict সমাধানের নিয়ম

  • explicit class import wildcard-এর চেয়ে অগ্রাধিকার পায়: import java.util.Date; + import java.sql.*; compile হয়, java.util.Date ব্যবহার করে।
  • একই নামের দুটো explicit import → ambiguous, compile হয় না।
  • দুটোই দরকার হলে একটা import-এ নিন আর অন্যটার fully qualified name ব্যবহার করুন (যেমন java.sql.Date sqlDate;)।

Package দিয়ে compile ও run

javac packagea/ClassA.java packageb/ClassB.java
java packageb.ClassB             // prints "Got it"

javac packagea/*.java            // wildcard for files in one dir
// javac *.java would NOT pick up files inside packages

javac -d classes packagea/ClassA.java // -> classes/packagea/ClassA.class
java -cp classes packageb.ClassB
java -classpath classes packageb.ClassB
java --class-path classes packageb.ClassB // note the TWO dashes
⚠ EXAM TRAP — case ও dash

Java option case sensitive: -d চলে, -D নয়। --class-path-এ দুটো dash লাগে আর -cp, -classpath-এ একটা। dash-এর সংখ্যা ভুল হলে প্রোগ্রাম চলবে না। -d option target directory-র নিচে package directory structure ধরে রাখে।

Element-এর order — PIC নিয়ম

✓ KEY RULE — PIC (picture)

Package, Import, Class — এই ক্রমে। package প্রথম non-comment line (optional); import package-এর পরে (optional); type import-এর পরে। field ও method class-এর ভেতরে যেকোনো জায়গায়। comment যেকোনো জায়গায়।

import java.util.*;
package structure;        // DOES NOT COMPILE (package must come before import)
String name;               // DOES NOT COMPILE (field outside a class)
public class Meerkat { }
SECTION 04

Object তৈরি ও Initialization

Constructor call করা

instance তৈরি হয় new + class নাম + parenthesis দিয়ে: Park p = new Park();।

⚠ EXAM TRAP — constructor বনাম method

Constructor-এর নাম class-এর সাথে মেলে এবং return type থাকে না। constructor-এর মতো দেখতে কিন্তু return type আছে এমনটা আসলে সাধারণ method — compile হয় কিন্তু new দিয়ে call হয় না:

public class Chick {
   public void Chick() { }   // NOT A CONSTRUCTOR (has void return type)
}

কোনো constructor না লিখলে compiler একটা default "do nothing" constructor দেয়।

Field পড়া ও লেখা

instance variable সরাসরি পড়া/লেখা যায়: mother.numberEggs = 1;। একই object-এ আরেকটা field initialize করার সময় আগে-initialize হওয়া field পড়া যায়:

2: String first = "Theodore";
3: String last  = "Moose";
4: String full  = first + last;  // reads first & last, writes full

Instance initializer block

method-এর বাইরে থাকা code block হলো instance initializer। brace জোড়া গুনে code block গোনা যায়; instance initializer method-এর ভেতরে থাকতে পারে না।

1: public class Bird {
2:    public static void main(String[] args) {
3:       { System.out.println("Feathers"); }  // inner block, NOT an initializer
4:    }
5:    { System.out.println("Snowy"); }        // instance initializer (outside method)
6: }

এখানে চারটা code block: class, method, inner block, instance initializer। শুধু line 5 instance initializer।

Initialization-এর order

দুটো নিয়ম

  1. field ও instance initializer block file-এ যে ক্রমে আছে সেই ক্রমে চলে।
  2. constructor চলে সব field ও instance initializer-এর পরে।
⚠ EXAM TRAP — forward reference

define হওয়ার আগে কোনো field পড়া যায় না:

{ System.out.println(name); } // DOES NOT COMPILE
private String name = "Fluffy";
  • private int number = 3; → number = 3
  • { number = 4; } → number = 4
  • constructor number = 5; → number = 5
  • তাই egg.number প্রিন্ট করে 5

default-এ একটা numeric literal হলো int। long-এর জন্য L, float-এর জন্য f/F যোগ করুন।

Literal-এ underscore

✓ KEY RULE

underscore অনুমোদিত সর্বত্র, কেবল নয়: শুরুতে, শেষে, decimal point-এর ঠিক আগে, বা decimal point-এর ঠিক পরে। পাশাপাশি একাধিক underscore-ও চলে।

double notAtStart   = _1000.00;   // DOES NOT COMPILE
double notAtEnd     = 1000.00_;   // DOES NOT COMPILE
double notByDecimal = 1000_.00;   // DOES NOT COMPILE
double annoyingOK   = 1_00_0.0_0; // ugly, but compiles

Reference type

একটা reference heap-এ থাকা object-এর memory address রেখে তাকে "point" করে; address দেখা যায় না। reference method call করতে পারে (null না হলে); primitive পারে না।

4: String reference = "hello";
5: int len = reference.length();
6: int bad = len.length();  // DOES NOT COMPILE (int has no methods)
⚠ EXAM TRAP — null

reference-কে null দেওয়া যায়; primitive-কে যায় না:

int value = null;   // DOES NOT COMPILE
String name = null; // fine

Wrapper class

Boolean.valueOf case-insensitive "true"-র জন্য true দেয় ("TrUe"→true), নয়তো false — "kangaroo" ও null সহ। intValue()/byteValue()-র মতো helper precision হারাতে পারে: Double.valueOf("200.99") থেকে byteValue() → -56, intValue() → 200।

Text block

text block শুরু ও শেষ হয় """ দিয়ে; ভেতরের content escape করতে হয় না, এবং এর type এখনো String।

SECTION 06

Variable Declaration ও var

Identifier

Legal identifier-এর চারটা নিয়ম

  1. শুরু হতে হবে একটা letter, currency symbol ($, ¥, €…), বা _ দিয়ে।
  2. number থাকতে পারে, কিন্তু number দিয়ে শুরু হতে পারে না।
  3. শুধু একটা underscore _ অনুমোদিত নয়।
  4. reserved word হতে পারে না। (true, false, null literal, এগুলোও নাম হিসেবে চলে না।)
// legal
long okidentifier; float $OK2Identifier; boolean _alsoOK1d3ntifi3r;

// illegal
int 3DPointClass;    // starts with a number
byte hollywood@vine; // @ is not a letter, digit, $ or _
double public;       // reserved word
short _;             // single underscore not allowed

একাধিক variable declare করা

এক statement-এ একাধিক variable-কে একটা type declaration share করতে হবে। comma দিয়ে আলাদা প্রতিটা অংশ নিজের একটা declaration।

String s1, s2;                 // two declared, none initialized
String s3 = "yes", s4 = "no";  // two declared & initialized
int i1, i2, i3 = 0;           // only i3 is initialized!
int num, String value;        // DOES NOT COMPILE (different types)
double d1, double d2;          // DOES NOT COMPILE (type repeated)

Variable initialize করা

Local variable

  • default value নেই
  • ব্যবহারের আগে initialize করতেই হবে (নয়তো compiler error)
  • declare করে না-ব্যবহার করা ঠিক আছে

Instance / class variable

  • স্বয়ংক্রিয়ভাবে default পায়
  • object-এ null, number-এ 0, boolean-এ false, char-এ \u0000
⚠ EXAM TRAP — uninitialized local পড়া 
int x;
int reply = x + 10;  // DOES NOT COMPILE (x never initialized)

if/else-এর শুধু একটা branch-এ মান সেট হলে পরে পড়লে compile হয় না। দুই branch-এই সেট হলে ঠিক আছে।

★ REMEMBER — final local

local variable-এ final দিলে সেটা constant — reassign করলে compile হয় না। final array reference-এ এখনো content বদলানো যায়, শুধু reference reassign নয়: favoriteNumbers = null; → compile হয় না।

Local variable type inference: var

var-এর নিয়ম

  • শুধু local variable-এর জন্য — instance/class variable নয়, method/constructor parameter নয়।
  • type infer হয় declaration-এর একই line থেকে।
  • type fix হয় compile time-এ; value runtime-এ বদলাতে পারে কিন্তু type নয়।
  • var x = null; একা → compile হয় না (তবে reference-typed var পরে null reassign করা যায়)।
  • multiple-variable declaration চলে না: var a, b = 3; → compile হয় না।
  • var একটা reserved type name, reserved word নয় — variable বা method নাম হতে পারে, কিন্তু class/interface/enum নয়।
var number = 7;
number = 4;
number = "five";  // DOES NOT COMPILE (inferred int)

var question;    // DOES NOT COMPILE (no value on the declaration line)
var n = null;    // DOES NOT COMPILE
SECTION 07

Scope ও Garbage Collection

Variable scope

Variable ভেদে scope

  • Local variable: declaration থেকে block-এর শেষ পর্যন্ত।
  • Method parameter: পুরো method।
  • Instance variable: declaration থেকে object garbage collection-এর জন্য eligible হওয়া পর্যন্ত।
  • Class (static) variable: declaration থেকে প্রোগ্রাম শেষ হওয়া পর্যন্ত।
⚠ EXAM TRAP — out of scope 
4: if (hungry) {
5:    int bitesOfCheese = 1;
6: } // bitesOfCheese goes out of scope here
7: System.out.println(bitesOfCheese); // DOES NOT COMPILE

ভেতরের block বাইরের block-এর variable পড়তে পারে, কিন্তু উল্টোটা নয়। scope ট্র্যাক করতে ভেতর থেকে বাইরে brace মেলান।

Garbage collection

সব object heap-এ ("free store") থাকে। garbage collector আর reachable নয় এমন object মুছে memory খালি করে।

একটা object GC-এর জন্য eligible হয় যখন…

  • এর দিকে আর কোনো reference থাকে না, অথবা
  • এর সব reference scope-এর বাইরে চলে গেছে।
★ REMEMBER — System.gc() শুধু একটা suggestion

System.gc() শুধু collection-এর পরামর্শ দেয়; JVM সেটা উপেক্ষা করতে পারে। GC-eligible ≠ সাথে সাথে collect, আর একটা object eligible হয়েও কখনো collect না-ও হতে পারে (যেমন আগে প্রোগ্রাম শেষ হয়ে গেলে)। একটা variable-কে final করলে তার object-এর garbage collection আটকায় না।

⚠ EXAM TRAP — eligibility ট্রেস করা 
4: one = new String("a");
5: two = new String("b");
6: one = two;          // "a" now has no references -> eligible for GC
7: String three = one;
8: one = null;          // "b" still reachable via two & three
9: }                    // "b" eligible only when scope ends here

"a" line 6-এ eligible হয়; "b" line 9-এ method শেষ হওয়া পর্যন্ত reachable থাকে।

★ SHORT RECAP

সংক্ষিপ্ত রিভিউ

Environment ও structure

  • javac .java → .class bytecode compile করে; java JVM-এ চালায়। run করার সময় .class extension বাদ।
  • member = field + method; method signature = নাম + parameter type। public top-level type-এর নাম filename-এর সাথে মিলতে হবে।

main() ও package

  • public static void main(String[] args); তিন parameter form ([] দুই পাশে, বা ...); args index 0 থেকে।
  • wildcard শুধু সরাসরি class import করে (child package/field/method নয়); java.lang স্বয়ংক্রিয়; explicit import wildcard-কে হারায়। order হলো PIC।

Object ও initialization

  • constructor = class নাম, return type নেই; return type থাকা "constructor" শুধু একটা method। field ও instance initializer source order-এ চলে, তারপর constructor।

Data type

  • আট primitive; literal default int (L/f যোগ); underscore প্রান্তে বা decimal-এর পাশে নয়; String primitive নয়।
  • reference null হতে পারে ও method call করতে পারে; primitive কোনোটাই পারে না। numeric wrapper Number extend করে; খারাপ input-এ valueOf NumberFormatException throw করে।

Variable, scope ও GC

  • local-এর default নেই, ব্যবহারের আগে initialize করতে হবে; instance/class variable default পায়। var শুধু local, একই line থেকে infer, type compile time-এ fix।
  • object unreachable হলে GC-eligible; System.gc() শুধু একটা hint; final collection আটকায় না।
