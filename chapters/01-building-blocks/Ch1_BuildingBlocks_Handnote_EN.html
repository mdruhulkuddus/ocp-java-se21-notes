SECTION 01

The Environment & Class Structure

Major components of Java

The JDK (Java Development Kit) is the minimum software for Java development. Key commands:

CommandPurpose
javacConverts .java source files into .class bytecode.
javaExecutes the program.
jarPackages files together.
javadocGenerates documentation.

javac produces bytecode; java launches the JVM, which knows how to run that bytecode on the actual machine. Since Java 11 there is no separate JRE — you use the full JDK to run programs.

★ REMEMBER

Java class files run on the JVM and therefore run on any machine with Java, not just the one they were compiled on. Java 21 was released September 2023.

Fields and methods

A class has two primary elements: methods (functions/procedures) and fields (variables). Together they are the members of the class. The simplest class is just public class Animal { }.

public class Animal {
   String name;                       // a field
   public String getName() {           // method with String return type
      return name;
   }
   public void setName(String newName) { // void = returns nothing
      name = newName;                  // newName is a parameter
   }
}
✓ KEY RULE — method signature

The method signature is the method name plus parameter types (not names, not return type). For public int numberVisitors(int month) the signature is numberVisitors(int).

Comments

Three comment types: single-line //, multiline /* ... */, and Javadoc /** ... */. Comments can go anywhere in code.

⚠ EXAM TRAP — nested multiline comments

Everything from the first /* to the first */ is one comment, even if it contains // or another /* inside. This line does not compile because of a leftover */:

/*
 * /* ferret */
 */

The compiler sees /* ... */ ending at the inner */, leaving a stray */ — invalid syntax.

Classes and source files

Top-level type rules

  • A top-level type need not be public; class Animal { } is fine.
  • A file may contain multiple top-level types, but at most one can be public.
  • If a type is public, its name must match the filename. public class Animal2 in Animal.java → DOES NOT COMPILE.
SECTION 02

The main() Method & Running Code

Execution begins with main() — the program's entry point that the JVM looks for.

public class Zoo {
   public static void main(String[] args) {
      System.out.println("Hello World");
   }
}

Compile and run (note: omit .class when running):

javac Zoo.java   // produces Zoo.class
java Zoo         // runs it (no .class extension)

Words in the signature

KeywordMeaning
publicAccess modifier — full access from anywhere.
staticBinds the method to the class, so it can be called as Zoo.main() with no object.
voidThe return type — returns no data.
String[] argsThe parameter list — an array of String arguments.
✓ KEY RULE — three valid parameter forms

The compiler accepts any of these for main's parameter (and any valid variable name):

String[] args
String options[]
String... friends   // varargs
★ REMEMBER — optional modifiers

public and static are required, but a few optional modifiers are allowed. Both final keywords below are optional and still make a valid entry point:

public final static void main(final String[] args) {}

Passing parameters

Arguments come after the class name and are indexed from 0. Spaces separate arguments; use quotes for spaces inside one.

java Zoo Bronx Zoo        // args[0]=Bronx, args[1]=Zoo
java Zoo "San Diego" Zoo   // args[0]=San Diego
⚠ EXAM TRAP — missing argument

Reading an index that wasn't passed throws at runtime. java Zoo Zoo then reading args[1] → ArrayIndexOutOfBoundsException (Index 1 out of bounds for length 1).

★ REMEMBER — single-file source-code launch

You can skip the explicit compile step: java Zoo.java Bronx Zoo. The twist: here you include the .java extension (unlike the compiled java Zoo form).

SECTION 03

Packages, Imports & Class Order

Java groups classes into packages. An import statement tells the compiler which package to look in for a class. Without it, you get error: cannot find symbol.

Wildcards

import java.util.*; imports every class directly in java.util.

✓ KEY RULE

A wildcard imports only classes directly under that package — not child packages, fields, or methods. So import java.util.*; does not bring in java.util.concurrent.atomic.AtomicInteger; you need import java.util.concurrent.atomic.*;.

⚠ EXAM TRAP — invalid import forms 
import java.nio.*;            // NO GOOD — wildcard matches class names, not "file.Files"
import java.nio.*.*;          // NO GOOD — only one wildcard, and it must be last
import java.nio.file.Paths.*; // NO GOOD — cannot import methods, only class names

Redundant imports

java.lang is special — it is automatically imported. Also, a class in the same package as the current class needs no import.

1: import java.lang.System;  // redundant (java.lang auto-imported)
2: import java.lang.*;       // redundant (java.lang auto-imported)
3: import java.util.Random;
4: import java.util.*;        // redundant here (Random already imported on line 3)

Three of these four are redundant. If line 3 were absent, java.util.* would not be redundant — it would be the thing importing Random.

Naming conflicts

java.util.Date and java.sql.Date both exist. Two wildcards that both contain Date make the usage ambiguous and it won't compile.

Conflict resolution rules

  • An explicit class import takes precedence over wildcards: import java.util.Date; + import java.sql.*; compiles, using java.util.Date.
  • Two explicit imports of the same simple name → ambiguous, does not compile.
  • To use both, pick one in an import and use the other's fully qualified name (e.g. java.sql.Date sqlDate;).

Compiling & running with packages

javac packagea/ClassA.java packageb/ClassB.java
java packageb.ClassB             // prints "Got it"

javac packagea/*.java            // wildcard for files in one dir
// javac *.java would NOT pick up files inside packages

javac -d classes packagea/ClassA.java // -> classes/packagea/ClassA.class
java -cp classes packageb.ClassB
java -classpath classes packageb.ClassB
java --class-path classes packageb.ClassB // note the TWO dashes
⚠ EXAM TRAP — case & dashes

Java options are case sensitive: -d works, -D does not. --class-path needs two dashes while -cp and -classpath need one. The wrong number of dashes means the program won't run. The -d option preserves the package directory structure under the target directory.

Ordering elements — the PIC rule

✓ KEY RULE — PIC (picture)

Package, Import, Class — in that order. Package is the first non-comment line (optional); imports come after the package (optional); the type comes after imports. Fields and methods go anywhere inside the class. Comments go anywhere.

import java.util.*;
package structure;        // DOES NOT COMPILE (package must come before import)
String name;               // DOES NOT COMPILE (field outside a class)
public class Meerkat { }
SECTION 04

Creating Objects & Initialization

Calling constructors

Create an instance with new + class name + parentheses: Park p = new Park();.

⚠ EXAM TRAP — constructor vs method

A constructor's name matches the class and has no return type. A method that looks like a constructor but has a return type is just a regular method — it compiles but is not called by new:

public class Chick {
   public void Chick() { }   // NOT A CONSTRUCTOR (has void return type)
}

If you write no constructor, the compiler supplies a default "do nothing" constructor.

Reading & writing fields

Instance variables can be read/written directly: mother.numberEggs = 1;. You can read already-initialized fields when initializing another field on the same object:

2: String first = "Theodore";
3: String last  = "Moose";
4: String full  = first + last;  // reads first & last, writes full

Instance initializer blocks

A code block outside a method is an instance initializer. Count code blocks by counting brace pairs; instance initializers cannot live inside a method.

1: public class Bird {
2:    public static void main(String[] args) {
3:       { System.out.println("Feathers"); }  // inner block, NOT an initializer
4:    }
5:    { System.out.println("Snowy"); }        // instance initializer (outside method)
6: }

Four code blocks here: class, method, inner block, instance initializer. Only line 5 is an instance initializer.

Order of initialization

The two rules

  1. Fields and instance initializer blocks run in the order they appear in the file.
  2. The constructor runs after all fields and instance initializers.
⚠ EXAM TRAP — forward reference

You cannot read a field before it is defined:

{ System.out.println(name); } // DOES NOT COMPILE
private String name = "Fluffy";
  • private int number = 3; → number = 3
  • { number = 4; } → number = 4
  • constructor number = 5; → number = 5
  • So egg.number prints 5

Java has two kinds of data: primitive types and reference types. There are eight primitives.

★ REMEMBER

Each numeric type uses twice the bits of the next-smaller one; all numeric types are signed. short is signed and char is unsigned, but both are 16-bit, so they can be cast to each other. String is not a primitive — it's an object.

Writing literals

By default a numeric literal is an int. Add L for long, f/F for float.

long max = 3123456789;   // DOES NOT COMPILE (too big for int)
long max = 3123456789L;  // OK — now it's a long

Underscores in literals

✓ KEY RULE

Underscores are allowed anywhere except at the start, the end, right before a decimal point, or right after a decimal point. Multiple adjacent underscores are allowed.

double notAtStart   = _1000.00;   // DOES NOT COMPILE
double notAtEnd     = 1000.00_;   // DOES NOT COMPILE
double notByDecimal = 1000_.00;   // DOES NOT COMPILE
double annoyingOK   = 1_00_0.0_0; // ugly, but compiles

Reference types

A reference "points" to an object on the heap by storing its memory address; you can't see the address. References can call methods (if not null); primitives cannot.

4: String reference = "hello";
5: int len = reference.length();
6: int bad = len.length();  // DOES NOT COMPILE (int has no methods)
⚠ EXAM TRAP — null

References can be assigned null; primitives cannot:

int value = null;   // DOES NOT COMPILE
String name = null; // fine

Wrapper classes

Each primitive has a wrapper class. The numeric ones extend Number; Boolean and Character do not.

★ REMEMBER — Boolean.valueOf & Number helpers

Boolean.valueOf returns true for any case-insensitive "true" ("TrUe"→true), and false otherwise — including "kangaroo" and null. Helper methods like intValue()/byteValue() can lose precision: from Double.valueOf("200.99"), byteValue() → -56, intValue() → 200.

Text blocks

A text block starts and ends with """; its contents needn't be escaped, and its type is still String.

⚠ EXAM TRAP — line break required

Incidental whitespace (left of the leftmost non-whitespace character) is stripped; essential whitespace is kept. """\n doe \\\n deer""" outputs one line (doe deer) because \ skips the line break.

SECTION 06

Declaring Variables & var

Identifiers

Four rules for legal identifiers

  1. Must begin with a letter, a currency symbol ($, ¥, €…), or _.
  2. May include numbers, but cannot start with one.
  3. A single underscore _ alone is not allowed.
  4. Cannot be a reserved word. (true, false, null are literals, not usable as names either.)
// legal
long okidentifier; float $OK2Identifier; boolean _alsoOK1d3ntifi3r;

// illegal
int 3DPointClass;    // starts with a number
byte hollywood@vine; // @ is not a letter, digit, $ or _
double public;       // reserved word
short _;             // single underscore not allowed

Declaring multiple variables

Multiple variables in one statement must share one type declaration. Each comma-separated piece is its own little declaration.

String s1, s2;                 // two declared, none initialized
String s3 = "yes", s4 = "no";  // two declared & initialized
int i1, i2, i3 = 0;           // only i3 is initialized!
int num, String value;        // DOES NOT COMPILE (different types)
double d1, double d2;          // DOES NOT COMPILE (type repeated)

Initializing variables

Local variables

  • No default value
  • Must be initialized before use (else compiler error)
  • Declared-but-unused is fine

Instance / class variables

  • Get a default automatically
  • null for objects, 0 for numbers, false for boolean, \u0000 for char
⚠ EXAM TRAP — uninitialized local read 
int x;
int reply = x + 10;  // DOES NOT COMPILE (x never initialized)

If a value is set in only one branch of an if/else, reading it afterward won't compile. Set in both branches → fine.

★ REMEMBER — final locals

final on a local variable makes it a constant — reassigning won't compile. For a final array reference you can still modify the contents, just not reassign the reference: favoriteNumbers = null; → does not compile.

Local variable type inference: var

var rules

  • Only for local variables — not instance/class variables, not method/constructor parameters.
  • The type is inferred from the same line as the declaration.
  • Type is fixed at compile time; the value can change at runtime but the type cannot.
  • var x = null; alone → does not compile (but a reference-typed var can be reassigned null later).
  • No multiple-variable declarations: var a, b = 3; → does not compile.
  • var is a reserved type name, not a reserved word — it can name a variable or method, but not a class/interface/enum.
var number = 7;
number = 4;
number = "five";  // DOES NOT COMPILE (inferred int)

var question;    // DOES NOT COMPILE (no value on the declaration line)
var n = null;    // DOES NOT COMPILE
SECTION 07

Scope & Garbage Collection

Variable scope

Scope by variable kind

  • Local variables: from declaration to the end of the block.
  • Method parameters: the whole method.
  • Instance variables: from declaration until the object is eligible for garbage collection.
  • Class (static) variables: from declaration until the program ends.
⚠ EXAM TRAP — out of scope 
4: if (hungry) {
5:    int bitesOfCheese = 1;
6: } // bitesOfCheese goes out of scope here
7: System.out.println(bitesOfCheese); // DOES NOT COMPILE

Inner blocks can read variables from outer blocks, but not vice versa. Match braces from the inside out to track scope.

Garbage collection

All objects live on the heap (the "free store"). The garbage collector frees memory by deleting objects that are no longer reachable.

An object is eligible for GC when…

  • It no longer has any references pointing to it, or
  • All references to it have gone out of scope.
★ REMEMBER — System.gc() is only a suggestion

System.gc() merely suggests collection; the JVM is free to ignore it. Eligible for GC ≠ collected immediately, and an object may be eligible yet never collected (e.g. if the program ends first). Marking a variable final does not stop its object from being garbage collected.

⚠ EXAM TRAP — tracing eligibility 
4: one = new String("a");
5: two = new String("b");
6: one = two;          // "a" now has no references -> eligible for GC
7: String three = one;
8: one = null;          // "b" still reachable via two & three
9: }                    // "b" eligible only when scope ends here

"a" becomes eligible on line 6; "b" stays reachable until the method ends on line 9.

★ SHORT RECAP

Short Recap

Environment & structure

  • javac compiles .java → .class bytecode; java runs it on the JVM. Run with no .class extension.
  • Members = fields + methods; method signature = name + parameter types. A public top-level type must match its filename.

main() & packages

  • public static void main(String[] args); three parameter forms ([] either side, or ...); args indexed from 0.
  • Wildcard imports classes directly in a package only (not child packages/fields/methods); java.lang is automatic; explicit imports beat wildcards. Order is PIC.

Objects & initialization

  • Constructor = class name, no return type; a "constructor" with a return type is just a method. Fields & instance initializers run in source order, then the constructor.

Data types

  • Eight primitives; literals default to int (add L/f); underscores not at edges or beside a decimal point; String is not a primitive.
  • References can be null and call methods; primitives can do neither. Numeric wrappers extend Number; valueOf throws NumberFormatException on bad input.

Variables, scope & GC

  • Locals have no default and must be initialized before use; instance/class variables get defaults. var is local-only, inferred from the same line, type-fixed at compile time.
  • An object is GC-eligible when unreachable; System.gc() is only a hint; final doesn't prevent collection.
