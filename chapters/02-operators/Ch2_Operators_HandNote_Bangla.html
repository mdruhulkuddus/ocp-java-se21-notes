SECTION 01

Java Operator ও Precedence বোঝা

একটা operator হলো এমন symbol যা operand (value, variable বা literal)-এর উপর প্রয়োগ হয়ে একটা result দেয়। Java-তে তিন ধরনের operator: unary (এক operand), binary (দুই), এবং ternary (তিন)।

★ REMEMBER

Operator সবসময় বাঁ থেকে ডানে মূল্যায়ন হয় না। কোনটা আগে চলবে তা precedence ঠিক করে; কিছু operator (unary, cast, assignment, ternary) ডান থেকে বাঁয়ে মূল্যায়ন হয়।

int cookies = 4;
double reward = 3 + 2 * --cookies;  // decrement first -> 3, then 2*3=6, then +3 = 9 -> 9.0
// reward = 9.0, cookies = 3

★ REMEMBER — তিনটা "ডান-থেকে-বাঁ" দ্বীপ

binary/assignment group-গুলোর মধ্যে শুধু unary, cast, ternary ও assignment ডান→বাঁ মূল্যায়ন হয়। বাকি সব মাঝখানের অংশ বাঁ→ডান। Shift operator সরাসরি পরীক্ষায় আসে না, তবে এর অস্তিত্ব জেনে রাখুন।

Parentheses precedence override করে। এগুলো অবশ্যই balanced ও সঠিকভাবে nested হতে হবে, এবং Java parentheses-এর জায়গায় bracket [] মেনে নেয় না:

long pigeon = 1 + ((3 * 5) / 3;     // DOES NOT COMPILE (unbalanced)
int  blueJay = (9 + 2) + 3) / (2 * 4; // DOES NOT COMPILE (mis-nested)
short robin = 3 + [(4 * 2) + 4];      // DOES NOT COMPILE (brackets not allowed)
SECTION 02

Unary Operator প্রয়োগ

Operatorউদাহরণবর্ণনা
Logical complement!aএকটা boolean মান উল্টে দেয়।
Bitwise complement~bএকটা সংখ্যার সব bit উল্টে দেয়।

Complement ও negation — type গুরুত্বপূর্ণ

! শুধু boolean-এ চলে; - ও ~ শুধু সংখ্যায়। এগুলো মিশিয়ে ফেলা একটা ক্লাসিক compile-fail trap:

int pelican = !5;        // DOES NOT COMPILE (! needs boolean)
boolean penguin = -true;  // DOES NOT COMPILE (- needs number)
boolean parrot = ~true;   // DOES NOT COMPILE (~ needs number)
boolean peacock = !0;     // DOES NOT COMPILE (! needs boolean; 0 is not false)
⚠ EXAM TRAP — 1 ≠ true, 0 ≠ false

কিছু ভাষার বিপরীতে, Java-তে numeric মান আর boolean মান সম্পর্কহীন। 1 মানে true নয়, 0 মানে false নয়।

Bitwise complement-এর শর্টকাট: ~n = -n - 1।

int number = 70;
System.out.println(~number);   // -71
System.out.println(~(~number));// 70
⚠ EXAM TRAP — -(-x) বনাম --x

দুইবার negate করতে parentheses লাগে: -(-x)। --x লিখলে সেটা decrement operator হিসেবে পড়া হয় আর ভিন্ন ফল দেয়।

Increment ও decrement — pre বনাম post

যে একটা নিয়মে পয়েন্ট আসে

  • Pre (++w, --x): আগে পরিবর্তন, তারপর নতুন মান return।
  • Post (y++, z--): আগে আসল মান return, তারপর পরিবর্তন।
int a = 0;
System.out.println(a);    // 0
System.out.println(++a);  // 1  (pre: new value)
System.out.println(a);    // 1
System.out.println(a--);  // 1  (post: original value)
System.out.println(a);    // 0
SECTION 03

Binary Arithmetic ও Numeric Promotion

Arithmetic operator + - * / % সব numeric primitive-এ চলে (boolean-এ নয়)। শুধু + (ও +=) String-এও চলে, যা concatenation দেয়। Multiplicative (* / %) additive (+ -)-এর চেয়ে উপরে।

int price = 2 * 5 + 3 * 4 - 8;  // 10 + 12 - 8 = 14
int p2 = 2 * ((5 + 3) * 4 - 8); // 2 * (32 - 8) = 48

Division ও modulus

Integer division floor নেয় (দশমিক বাদ, round নয়)। Modulus % হলো ভাগশেষ।

System.out.println(9 / 3);  // 3
System.out.println(9 % 3);  // 0
System.out.println(11 / 3); // 3
System.out.println(11 % 3); // 2
★ REMEMBER — modulus ও negative

negative divisor-এর sign উপেক্ষিত হয়। ফলের sign dividend-কে অনুসরণ করে: -7 % 5 = -2, কিন্তু 7 % -5 = 2।

Numeric promotion-এর নিয়ম

এই চারটা মুখস্থ রাখুন

  1. Type ভিন্ন হলে → ছোটটা বড়টায় promote হয়।
  2. Integral-এর সাথে floating-point মিশলে → integral-টা floating-point type-এ promote হয়।
  3. byte, short, char binary arithmetic operator-এ variable-এর সাথে ব্যবহৃত হলেই int-এ promote হয় — কোনো operand int না হলেও।
  4. Promotion শেষে result-এর type হয় (promoted) operand-এর মতোই।
⚠ EXAM TRAP — rule 3 কামড় দেয় 
short x = 10, y = 3;
var z = x * y;   // z is int, NOT short (both promoted to int)

double a = 39.21;
float b = 2.1;     // DOES NOT COMPILE (2.1 is a double; needs 2.1f)

খেয়াল: unary ++/-- rule 3 থেকে বাদ — একটা short-এ ++ প্রয়োগে ফল short-ই থাকে।

ধাপে-ধাপে — w * x / y যেখানে short w, float x, double y: w → int (rule 3), তারপর x-এর সাথে গুণ করতে → float, তারপর y দিয়ে ভাগ করতে product → double। ফল একটা double।

SECTION 04

Value Assign ও Casting

Assignment একটা binary operator যা ডান→বাঁ মূল্যায়ন হয়। Java ছোট→বড় auto-promote করে, কিন্তু বড়→ছোট যেতে explicit cast লাগে, নয়তো compile হয় না।

Casting-এর মূল কথা

int fur = (int)5;
short tail = (short)(4 + 10);
long feathers = 10(long);  // DOES NOT COMPILE (type on wrong side)

এগুলো সব fail — cast ছাড়া বড় বা floating মান ছোট type-এ:

int fish = 1.0;     // DOES NOT COMPILE (double -> int)
short bird = 1921222;// DOES NOT COMPILE (out of short range)
int mammal = 9f;    // DOES NOT COMPILE (float -> int)
long reptile = 192_301_398_193_810_323; // DOES NOT COMPILE (int literal too big; needs L)
⚠ EXAM TRAP — cast unary, শক্তভাবে bind করে 
short mouse = 10, hamster = 3;
short c1 = (short)mouse * hamster;       // DOES NOT COMPILE
short c2 = 1 + (short)(mouse * hamster); // DOES NOT COMPILE
short ok = (short)(mouse * hamster);       // compiles

Cast শুধু তার ঠিক পরের একটা operand-এ লাগে। c1-এ শুধু mouse cast হয়, তারপর দুটোই int-এ promote হয়। c2-তে cast ঠিক আছে কিন্তু 1 + ফলকে আবার int-এ promote করে।

Value বনাম variable

Compiler আঁটা literal arithmetic মেনে নেয়, কিন্তু একই arithmetic variable-এ করলে আপত্তি করে (rule 3 int-এ promote করে):

byte gloves = 7 * 10;   // compiles (70 fits in byte)
short boots = 2 + 1;    // compiles
short bad = 2 + hat;     // DOES NOT COMPILE (hat is a variable)
byte over = 7 * 100;   // DOES NOT COMPILE (700 overflows byte)

Compound assignment auto-cast করে

Compound operator (+= -= *= /= …) ফলকে চুপচাপ বাঁ variable-এর type-এ cast করে দেয়, তাই explicit cast-এর বদলে ব্যবহার করা যায়:

long goat = 10;
int sheep = 5;
sheep = sheep * goat; // DOES NOT COMPILE (long -> int)
sheep *= goat;        // compiles (compound casts back to int)
✓ KEY RULE — assignment একটা মান ফেরায়

Assignment নিজেই একটা expression যা assign-করা মানের সমান। তাই long c = (wolf = 3); wolf-কে 3 করে এবং c-কে 3 দেয়। সাবধান if (healthy = true) — এটা assign করে, তুলনা করে না, তাই এখানে সবসময় true।

SECTION 05

Value তুলনা করা

Equality operator (==, !=)

primitive-এ == মান তুলনা করে। object-এ reference তুলনা করে — true শুধু তখনই যখন দুটো একই object নির্দেশ করে (বা দুটোই null)। সম্পর্কহীন type মেশানো যায় না।

boolean monkey = true == 3;      // DOES NOT COMPILE (boolean vs number)
boolean ape = false != "Grape";  // DOES NOT COMPILE (boolean vs String)
System.out.print(null == null);     // true
⚠ EXAM TRAP — condition-এর ভিতরে = বনাম == 
boolean bear = false;
boolean polar = (bear = true); // assigns true, so polar is true
System.out.println(polar);        // true

Relational operator ও instanceof

< <= > >= সংখ্যায় চলে (ছোট operand promote হয়)। instanceof runtime-এ পরীক্ষা করে একটা reference কোনো type-এর instance কি না।

✓ KEY RULE — instanceof ও null

instanceof একটা null literal বা null reference-এ সবসময় false ফেরায়। কিন্তু x instanceof null compile হয় না। অসম্ভব type-এর বিপরীতে test (যেমন একটা Number variable instanceof String)-ও compile হয় না।

System.out.print(null instanceof Object); // false
String s = null;
System.out.print(s instanceof String);  // false

Logical ও bitwise operator (&, |, ^)

boolean মানে এগুলো logical operator; সংখ্যায় bitwise। দুই পাশই সবসময় মূল্যায়ন হয়।

Truth-table শর্টকাট

  • & AND — true শুধু যদি দুটোই true।
  • | inclusive OR — false শুধু যদি দুটোই false।
  • ^ exclusive OR — true শুধু যদি operand দুটো আলাদা হয়।
boolean a = true, b = true;
System.out.println(a | b); // true
System.out.println(a & b); // true
System.out.println(a ^ b); // false (same -> false)

Conditional (short-circuit) operator (&&, ||)

একই truth table, কিন্তু বাঁ পাশ ফল ঠিক করে দিলে ডান পাশ বাদ পড়ে।

⚠ EXAM TRAP — unperformed side effect 
int rabbit = 6;
boolean bunny = (rabbit >= 6) || (++rabbit <= 7);
System.out.println(rabbit); // 6  (left is true, so ++rabbit never runs)

Short-circuit NullPointerException-ও ঠেকায়: if (duck != null && duck.getAge() < 5) duck null হলে কখনো getAge() ডাকে না। একক & দিয়ে ডাকত।

SECTION 06

Ternary Operator

একমাত্র তিন-operand operator: booleanExpression ? expr1 : expr2। এটা একটা সংক্ষিপ্ত if/else যা একটা মান ফেরায়।

int owl = 5;
int food = owl < 2 ? 3 : 4; // 4

Ternary nest করা যায়; parentheses readability অনেক বাড়ায়:

int f1 = owl < 4 ? owl > 2 ? 3 : 4 : 5;
int f2 = (owl < 4 ? ((owl > 2) ? 3 : 4) : 5); // equivalent
⚠ EXAM TRAP — branch type ও assignment 
int stripes = 7;
System.out.print((stripes > 5) ? 21 : "Zebra"); // compiles (print takes Object)
int animal = (stripes < 9) ? 3 : "Horse";       // DOES NOT COMPILE (String -> int)

print()-এর জন্য দুই branch একই type হওয়ার দরকার নেই (এটা Object-এ convert করে), কিন্তু ফল assign করলে type compatibility বাধ্যতামূলক হয়।

⚠ EXAM TRAP — শুধু একটা branch চলে 
int sheep = 1, zzz = 1;
int sleep = zzz < 10 ? sheep++ : zzz++;
System.out.print(sheep + "," + zzz); // 2,1  (only sheep++ evaluated)

Condition true বলে শুধু sheep++ চলে; zzz++ একটা unperformed side effect। Condition উল্টালে শুধু zzz বদলায়।

★ SHORT RECAP

সংক্ষিপ্ত রিভিউ

Precedence ও unary

  • precedence table মুখস্থ রাখুন; unary, cast, ternary ও assignment ডান→বাঁ মূল্যায়ন হয়। Parentheses balanced হতে হবে; bracket [] চলে না।
  • ! শুধু boolean-এ; -/~ শুধু সংখ্যায়; 1≠true, 0≠false। Pre নতুন মান, post আসল মান ফেরায়।

Arithmetic ও promotion

  • * / % > + -; integer division floor নেয়; modulus-এর sign dividend অনুসরণ করে।
  • byte/short/char variable-এ binary arithmetic-এ int-এ promote হয়; floating-point-এর সাথে মিশলে float/double type-এ promote হয়।

Assignment ও casting

  • বড়→ছোট cast লাগে; cast unary, শুধু এক operand-এ bind করে। আঁটা literal arithmetic compile হয়; variable-এ একই জিনিস না-ও হতে পারে।
  • Compound operator বাঁ type-এ auto-cast করে। Assignment নিজের মানের সমান একটা expression (= বনাম == trap)।

তুলনা ও ternary

  • == object-এ reference, primitive-এ value তুলনা করে; সম্পর্কহীন type মেশানো যায় না। instanceof null-এ false; instanceof null compile হয় না।
  • & | ^ সবসময় দুই পাশ মূল্যায়ন করে; && || short-circuit করে। Ternary শুধু একটা branch চালায়; ফল assign করলে type compatibility বাধ্যতামূলক।
