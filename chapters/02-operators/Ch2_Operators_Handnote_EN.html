SECTION 01

Understanding Java Operators & Precedence

An operator is a symbol applied to operands (values, variables, or literals) to produce a result. Java has three flavours: unary (one operand), binary (two), and ternary (three).

★ REMEMBER

Operators are not always evaluated left to right. Precedence decides what runs first; some operators (unary, cast, assignment, ternary) evaluate right to left.

int cookies = 4;
double reward = 3 + 2 * --cookies;  // decrement first -> 3, then 2*3=6, then +3 = 9 -> 9.0
// reward = 9.0, cookies = 3

Operator precedence (memorise the order)

★ REMEMBER — three "right-to-left" islands

Among the binary/assignment groups, only unary, cast, ternary, and assignment evaluate right-to-left. Everything else in between is left-to-right. The shift operators are not tested directly, but know they exist.

Parentheses override precedence. They must be balanced and properly nested, and Java does not accept brackets [] in place of parentheses:

long pigeon = 1 + ((3 * 5) / 3;     // DOES NOT COMPILE (unbalanced)
int  blueJay = (9 + 2) + 3) / (2 * 4; // DOES NOT COMPILE (mis-nested)
short robin = 3 + [(4 * 2) + 4];      // DOES NOT COMPILE (brackets not allowed)
SECTION 02

Applying Unary Operators

Complement & negation — type matters

! works only on boolean; - and ~ work only on numbers. Mixing them is a classic compile-fail trap:

int pelican = !5;        // DOES NOT COMPILE (! needs boolean)
boolean penguin = -true;  // DOES NOT COMPILE (- needs number)
boolean parrot = ~true;   // DOES NOT COMPILE (~ needs number)
boolean peacock = !0;     // DOES NOT COMPILE (! needs boolean; 0 is not false)
⚠ EXAM TRAP — 1 ≠ true, 0 ≠ false

Unlike some languages, in Java numeric values and boolean values are unrelated. 1 is not true and 0 is not false.

The bitwise complement shortcut: ~n equals -n - 1.

int number = 70;
System.out.println(~number);   // -71
System.out.println(~(~number));// 70
⚠ EXAM TRAP — -(-x) vs --x

Negating twice needs parentheses: -(-x). Writing --x is parsed as the decrement operator instead, giving a different result.

Increment & decrement — pre vs post

The one rule that wins points

  • Pre (++w, --x): change first, then return the new value.
  • Post (y++, z--): return the original value first, then change.
int a = 0;
System.out.println(a);    // 0
System.out.println(++a);  // 1  (pre: new value)
System.out.println(a);    // 1
System.out.println(a--);  // 1  (post: original value)
System.out.println(a);    // 0
SECTION 03

Binary Arithmetic & Numeric Promotion

Arithmetic operators + - * / % apply to all numeric primitives (not boolean). Only + (and +=) also works on String, giving concatenation. Multiplicative (* / %) outrank additive (+ -).

int price = 2 * 5 + 3 * 4 - 8;  // 10 + 12 - 8 = 14
int p2 = 2 * ((5 + 3) * 4 - 8); // 2 * (32 - 8) = 48

Division & modulus

Integer division takes the floor (drops the decimal, no rounding). Modulus % is the remainder.

System.out.println(9 / 3);  // 3
System.out.println(9 % 3);  // 0
System.out.println(11 / 3); // 3
System.out.println(11 % 3); // 2
★ REMEMBER — modulus & negatives

A negative divisor is ignored (its sign drops). The result's sign follows the dividend: -7 % 5 is -2, while 7 % -5 is 2.

Numeric promotion rules

Memorise these four

  1. Different types → the smaller is promoted to the larger.
  2. Integral mixed with floating-point → the integral is promoted to the floating-point type.
  3. byte, short, char are promoted to int whenever used with a binary arithmetic operator on a variable — even if neither operand is int.
  4. After promotion, the result has the same type as the (promoted) operands.
⚠ EXAM TRAP — rule 3 bites 
short x = 10, y = 3;
var z = x * y;   // z is int, NOT short (both promoted to int)

double a = 39.21;
float b = 2.1;     // DOES NOT COMPILE (2.1 is a double; needs 2.1f)

Note: unary ++/-- are excluded from rule 3 — ++ on a short stays a short.

Worked chain — w * x / y with short w, float x, double y: w → int (rule 3), then → float to multiply with x, then the product → double to divide by y. Result is a double.

SECTION 04

Assigning Values & Casting

Assignment is a binary operator evaluated right to left. Java auto-promotes smaller → larger, but going larger → smaller needs an explicit cast or it fails to compile.

Casting basics

int fur = (int)5;
short tail = (short)(4 + 10);
long feathers = 10(long);  // DOES NOT COMPILE (type on wrong side)

These all fail — a larger or floating value into a smaller type without a cast:

int fish = 1.0;     // DOES NOT COMPILE (double -> int)
short bird = 1921222;// DOES NOT COMPILE (out of short range)
int mammal = 9f;    // DOES NOT COMPILE (float -> int)
long reptile = 192_301_398_193_810_323; // DOES NOT COMPILE (int literal too big; needs L)
⚠ EXAM TRAP — cast is unary, binds tight 
short mouse = 10, hamster = 3;
short c1 = (short)mouse * hamster;       // DOES NOT COMPILE
short c2 = 1 + (short)(mouse * hamster); // DOES NOT COMPILE
short ok = (short)(mouse * hamster);       // compiles

The cast applies only to the single operand right after it. In c1 only mouse is cast, then both are promoted to int. In c2 the cast is fine but 1 + re-promotes the result to int.

Values vs variables

The compiler tolerates literal arithmetic that fits, but balks at the same arithmetic on variables (rule 3 promotes to int):

byte gloves = 7 * 10;   // compiles (70 fits in byte)
short boots = 2 + 1;    // compiles
short bad = 2 + hat;     // DOES NOT COMPILE (hat is a variable)
byte over = 7 * 100;   // DOES NOT COMPILE (700 overflows byte)

Compound assignment auto-casts

Compound operators (+= -= *= /= …) silently cast the result back to the left variable's type, so they can replace an explicit cast:

long goat = 10;
int sheep = 5;
sheep = sheep * goat; // DOES NOT COMPILE (long -> int)
sheep *= goat;        // compiles (compound casts back to int)
✓ KEY RULE — assignment returns a value

An assignment is itself an expression equal to the assigned value. So long c = (wolf = 3); sets wolf to 3 and gives c the value 3. Watch for if (healthy = true) — that assigns, it doesn't compare, so it's always true here.

SECTION 05

Comparing Values

Equality operators (==, !=)

For primitives, == compares values. For objects, it compares references — true only if both point to the same object (or both are null). You cannot mix incompatible types.

boolean monkey = true == 3;      // DOES NOT COMPILE (boolean vs number)
boolean ape = false != "Grape";  // DOES NOT COMPILE (boolean vs String)
System.out.print(null == null);     // true
⚠ EXAM TRAP — = vs == inside conditions 
boolean bear = false;
boolean polar = (bear = true); // assigns true, so polar is true
System.out.println(polar);        // true

Relational operators & instanceof

< <= > >= apply to numbers (smaller operand is promoted). instanceof tests whether a reference is an instance of a type at runtime.

✓ KEY RULE — instanceof & null

instanceof on a null literal or null reference always returns false. But x instanceof null does not compile. Testing against an impossible type (e.g. a Number variable instanceof String) also does not compile.

System.out.print(null instanceof Object); // false
String s = null;
System.out.print(s instanceof String);  // false

Logical & bitwise operators (&, |, ^)

On boolean values these are logical operators; on numbers they are bitwise. Both sides are always evaluated.

Truth-table shortcuts

  • & AND — true only if both are true.
  • | inclusive OR — false only if both are false.
  • ^ exclusive OR — true only if the operands differ.
boolean a = true, b = true;
System.out.println(a | b); // true
System.out.println(a & b); // true
System.out.println(a ^ b); // false (same -> false)

Conditional (short-circuit) operators (&&, ||)

Same truth tables, but the right side is skipped when the left already decides the result.

⚠ EXAM TRAP — unperformed side effect 
int rabbit = 6;
boolean bunny = (rabbit >= 6) || (++rabbit <= 7);
System.out.println(rabbit); // 6  (left is true, so ++rabbit never runs)

Short-circuiting also guards against NullPointerException: if (duck != null && duck.getAge() < 5) never calls getAge() when duck is null. With single & it would.

SECTION 06

The Ternary Operator

The only operator with three operands: booleanExpression ? expr1 : expr2. It's a condensed if/else that returns a value.

int owl = 5;
int food = owl < 2 ? 3 : 4; // 4

Ternaries can nest; parentheses greatly help readability:

int f1 = owl < 4 ? owl > 2 ? 3 : 4 : 5;
int f2 = (owl < 4 ? ((owl > 2) ? 3 : 4) : 5); // equivalent
⚠ EXAM TRAP — branch types & assignment 
int stripes = 7;
System.out.print((stripes > 5) ? 21 : "Zebra"); // compiles (print takes Object)
int animal = (stripes < 9) ? 3 : "Horse";       // DOES NOT COMPILE (String -> int)

The two branches need not be the same type for print() (it converts to Object), but assigning the result forces type compatibility.

⚠ EXAM TRAP — only one branch runs 
int sheep = 1, zzz = 1;
int sleep = zzz < 10 ? sheep++ : zzz++;
System.out.print(sheep + "," + zzz); // 2,1  (only sheep++ evaluated)

Because the condition is true, only sheep++ runs; zzz++ is an unperformed side effect. Flip the condition and only zzz changes.

★ SHORT RECAP

Short Recap

Precedence & unary

  • Memorise the precedence table; unary, cast, ternary, and assignment evaluate right-to-left. Parentheses must be balanced; brackets [] aren't allowed.
  • ! only on boolean; -/~ only on numbers; 1≠true, 0≠false. Pre returns the new value, post returns the original.

Arithmetic & promotion

  • * / % outrank + -; integer division floors; modulus sign follows the dividend.
  • byte/short/char promote to int in binary arithmetic on variables; mixing with floating-point promotes to the float/double type.

Assignment & casting

  • Larger→smaller needs a cast; the cast is unary and binds to one operand only. Literal arithmetic that fits compiles; the same on variables may not.
  • Compound operators auto-cast back to the left type. An assignment is an expression equal to its value (= vs == trap).

Comparisons & ternary

  • == compares references for objects, values for primitives; can't mix incompatible types. instanceof on null is false; instanceof null doesn't compile.
  • & | ^ always evaluate both sides; && || short-circuit. Ternary runs only one branch; assigning its result enforces type compatibility.
