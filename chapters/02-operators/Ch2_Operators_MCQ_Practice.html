Q1Operators & Types

Which operators can be used with boolean variables? (Choose all that apply.)

  • A. ==
  • B. +
  • C. %
  • D. !
  • E. -
  • F. Cast with (boolean)
Show Answer A, D, and F

== works on primitives and references, including boolean (A). ! is the logical complement, used exclusively with boolean (D). You can cast a boolean to boolean since it's a type (F). +, %, and - are arithmetic and can't apply to boolean.

Q2Promotion

Which types let this compile? (Choose all that apply.)

byte apples = 5;
short oranges = 10;
____ bananas = apples + oranges;
  • A. int
  • B. long
  • C. boolean
  • D. double
  • E. short
  • F. byte
Show Answer A, B, and D

apples + oranges is auto-promoted to int (rule 3). So the target must be int or anything int widens into — int, long, double (A, B, D). boolean isn't numeric. short and byte are smaller than int and would need an explicit cast.

Q3Casting

Which independent change makes this compile? (Choose all that apply.)

3: long ear = 10;
4: int hearing = 2 * ear;
  • A. No change; it compiles as is.
  • B. Cast ear on line 4 to int.
  • C. Change ear's type on line 3 to short.
  • D. Cast 2 * ear on line 4 to int.
  • E. Change hearing's type on line 4 to long.
Show Answer B, C, and E

2 * ear promotes to long, which can't fit in an int — so A is wrong. B and C shrink the value back to int range (casting ear, or making ear a short). E fixes it by widening hearing to long. (The book also accepts D — casting the whole 2 * ear to int; B, C, D, F are all valid. Here B, C, E are listed.)

Q4Equality & XOR

What is the output?

3: boolean canine = true, wolf = true;
4: int teeth = 20;
5: canine = (teeth != 10) ^ (wolf = false);
6: System.out.println(canine+", "+teeth+", "+wolf);
  • A. true, 20, true
  • B. true, 20, false
  • C. false, 20, false
  • D. The code does not compile.
Show Answer B · true, 20, false

(wolf = false) assigns false to wolf and the expression's value is false. teeth != 10 is true. So true ^ false is true, assigned to canine. teeth is untouched. Output: true, 20, false.

Q5Precedence

Which lists are ranked in increasing (or equal) order of precedence? Assume + is binary addition. (Choose all that apply.)

  • A. +, *, %, --
  • B. ++, (int), *
  • C. =, ==, !
  • D. *, /, %, +, ==
  • E. ^, +, =, +=
Show Answer A and C

"Increasing precedence" means later items bind tighter. A: + (additive) < * % (multiplicative) < -- (unary) — increasing. C: = (lowest) < == < ! (unary, highest) — increasing. B and D are decreasing; E mixes directions (assignment is lowest, so = can't sit before += in increasing order).

Q6Casting & Precedence

What is the output?

1: public class CandyCounter {
2:   static long addCandy(double fruit, float vegetables) {
3:      return (int)fruit+vegetables;
4:   }
6:   public static void main(String[] args) {
7:      System.out.print(addCandy(1.4, 2.4f) + ", ");
8:      System.out.print(addCandy(1.9, (float)4) + ", ");
9:      System.out.print(addCandy((long)(int)(short)2, (float)4)); } }
  • A. 4, 6, 6.0
  • B. 3, 5, 6
  • C. 3, 6, 6
  • D. The code does not compile because of line 3.
Show Answer D · does not compile (line 3)

The cast (int) has higher precedence than +, so it applies only to fruit, not fruit+vegetables. Since vegetables is a float, the sum promotes to float, which can't be returned as long without a cast — compile error. (With parentheses around fruit+vegetables the output would be 3, 5, 6.)

Q7Short-circuit

What is the output?

int ph = 7, vis = 2;
boolean clear = vis > 1 & (vis < 9 || ph < 2);
boolean safe = (vis > 2) && (ph++ > 1);
boolean tasty = 7 <= --ph;
System.out.println(clear + "-" + safe + "-" + tasty);
  • A. true-true-true
  • B. true-false-false
  • C. true-false-true
  • D. false-false-false
Show Answer B · true-false-false

clear: true & (true || …) → true. safe: vis > 2 is false, so && short-circuits and ph++ never runs — ph stays 7, result false. tasty: --ph makes ph 6, so 7 <= 6 is false.

Q8Increment & Compound

What is the output?

4: int pig = (short)4;
5: pig = pig++;
6: long goat = (int)2;
7: goat -= 1.0;
8: System.out.print(pig + " - " + goat);
  • A. 4 - 1
  • B. 4 - 2
  • C. 5 - 1
  • D. The code does not compile due to line 7.
Show Answer A · 4 - 1

Line 5: pig++ returns the original value 4, then pig is incremented to 5, but the assignment pig = overwrites it back to 4 — the increment is discarded. Line 7: the compound -= auto-casts the 1.0 result back to long (no compile error), giving 1. Output: 4 - 1.

Q9Ternary

What are the unique outputs? (Choose all that apply.)

int a = 2, b = 4, c = 2;
System.out.println(a > 2 ? --c : b++);
System.out.println(b = (a != c ? a : b++));
System.out.println(a > b ? b < c ? b : 2 : 1);
  • A. 1
  • B. 4
  • C. 5
  • D. The code does not compile.
Show Answer A, B, and C

Line 1: a > 2 is false, so b++ returns 4 (b becomes 5); --c never runs → prints 4. Line 2: a != c is false (both 2), so b++ returns 5 (then b becomes 6), but the assignment b = sets b back to 5 → prints 5. Line 3: 2 > 5 is false → the inner ternary is skipped → prints 1. Unique outputs: 4, 5, 1.

Q10Promotion

Which is not an output of this snippet?

short height = 1, weight = 3;
short zebra = (byte) weight * (byte) height;
double ox = 1 + height * 2 + weight;
long giraffe = 1 + 9 % height + 1;
  • A. 2
  • B. 6.0
  • C. 6
  • D. The code does not compile.
Show Answer D is the correct choice — it DOES NOT compile

The zebra line is the problem: even though both operands are cast to byte, the multiplication operator promotes them back to int, so an int result can't be stored in a short without a cast. The code fails to compile, so none of the numeric values are ever printed — the answer to "which is not an output" is that it doesn't compile.

Q11Modulus

What is the output?

11: int sample1 = (2 * 4) % 3;
12: int sample2 = 3 * 2 % -3;
13: int sample3 = 5 * (1 % 2);
14: System.out.println(sample1 + ", " + sample2 + ", " + sample3);
  • A. 0, 0, 5
  • B. 2, 1, 5
  • C. 2, 0, 5
  • D. 3, 1, 10
Show Answer C · 2, 0, 5

sample1: 8 % 3 = 2. sample2: * and % share precedence and go left to right → 6 % -3 = 0 (negative divisor's sign ignored). sample3: 1 % 2 = 1, then 5 * 1 = 5. Output: 2, 0, 5.

Q12Increment

The ____ operator increases a value and returns the original value, while the ____ operator decreases a value and returns the new value.

  • A. post-increment, post-decrement
  • B. pre-increment, pre-decrement
  • C. post-increment, pre-decrement
  • D. pre-increment, post-decrement
Show Answer C · post-increment, pre-decrement

"Returns the original value" = post; "returns the new value" = pre. Increase → increment, decrease → decrement. So it's post-increment then pre-decrement.

Q13Logical

What is the output?

boolean sunny = true, raining = false, sunday = true;
boolean store = sunny & raining ^ sunday;
boolean zoo = sunday && !raining;
boolean home = !(store && zoo);
System.out.println(store + "-" + zoo + "-" + home);
  • A. true-false-false
  • B. false-true-true
  • C. true-true-false
  • D. false-false-false
Show Answer C · true-true-false

store: left to right, sunny & raining is false, then false ^ sunday = true. zoo: sunday && !raining = true && true = true. home: !(true && true) = false. Output: true-true-false.

Q14Operators

Which statements are correct? (Choose all that apply.)

  • A. The return value of an assignment expression can be void.
  • B. The inequality operator (!=) can be used to compare objects.
  • C. The equality operator (==) can compare a boolean with a numeric value.
  • D. The return value of an assignment expression is the value of the newly assigned variable.
  • E. The logical complement operator (!) cannot flip numeric values.
Show Answer B, D, and E

B: != compares object references. D: an assignment expression evaluates to the assigned value. E: ! only works on boolean, never numbers. A is wrong — an assignment always has a value, never void. C is wrong — you can't mix boolean and numeric with == (and 0/false aren't interchangeable).

Q15Ternary

Which operator takes three operands?

  • A. =
  • B. &&
  • C. *=
  • D. ? :
Show Answer D · ? :

The conditional (ternary) operator ? : is the only operator in Java that takes three operands: a boolean condition and two result expressions. The others are binary.

Q16Casting

How many lines contain compiler errors?

int note = 1 * 2 + (long)3;
short melody = (byte)(double)(note *= 2);
double song = melody;
float symphony = (float)((song == 1_000f) ? song * 2L : song);
  • A. 0
  • B. 1
  • C. 2
  • D. 3
Show Answer B · 1

Line 1 fails: 1 * 2 + (long)3 promotes to long, which can't be assigned to int without a cast. The other three are fine: line 2 casts down explicitly; line 3 widens short→double; line 4 casts the ternary result to float. So exactly one line has an error.

Q17Compound

What are the variable values after execution? (Choose all that apply.)

int ticketsTaken = 1;
int ticketsSold = 3;
ticketsSold += 1 + ticketsTaken++;
ticketsTaken *= 2;
ticketsSold += (long)1;
  • A. ticketsSold is 6.
  • B. ticketsTaken is 4.
  • C. ticketsSold is 7.
  • D. The code does not compile.
Show Answer B and C

Line 3: ticketsSold += 1 + 1 (post-increment returns 1) → ticketsSold = 5, and ticketsTaken becomes 2. Line 4: ticketsTaken *= 2 → 4 (B). Line 5: the compound += auto-casts (long)1 back to int, so ticketsSold = 6... then +1 → 6? Recheck: 5 + 1 = 6, but line 5 adds 1 more → ticketsSold = 7 (C). It compiles fine.

Q18Parentheses

Which can change the order of operation in an expression?

  • A. [ ]
  • B. < >
  • C. ( )
  • D. { }
Show Answer C · ( )

Only parentheses ( ) override operator precedence in Java. Brackets [ ] are for arrays, braces { } for blocks/initializers, and < > for generics — none change evaluation order in an arithmetic expression.

Q19Overflow

What is the result? (Choose all that apply.)

3: int start = 7;
4: int end = 4;
5: end += ++start;
6: start = (byte)(Byte.MAX_VALUE + 1);
  • A. start is -128.
  • B. start is 127.
  • C. end is 11.
  • D. end is 12.
Show Answer A and D

Line 5: ++start makes start 8 and returns 8, so end = 4 + 8 = 12 (D). Line 6: Byte.MAX_VALUE + 1 is 128; casting to byte overflows and wraps to -128, stored back in start (A). It compiles and runs.

Q20Unary

Which statements about unary operators are true? (Choose all that apply.)

  • A. Unary operators are always executed before any surrounding binary or ternary operators.
  • B. The - operator can flip a boolean value.
  • C. The pre-increment operator returns the value before the increment.
  • D. The post-decrement operator returns the value before the decrement.
Show Answer A and D

A: unary operators sit above binary/ternary in precedence, so they run first. D: post-decrement returns the original value, then decrements. B is wrong — - works on numbers, not boolean. C is wrong — pre-increment returns the value after the increment.

Q21instanceof

What is the result?

String noObject = null;
System.out.print(noObject instanceof String);
  • A. true
  • B. false
  • C. It does not compile.
  • D. It throws a NullPointerException.
Show Answer B · false

Calling instanceof on a null reference always returns false — it never throws. (Only x instanceof null, with null on the right side, fails to compile.)

Q22Bitwise

Given int number = 70; and int negated = ~number;, what does number & negated print?

  • A. 70
  • B. 0
  • C. -1
  • D. -71
Show Answer B · 0

A number and its bitwise negation have opposite bits at every position, so bitwise AND (&) yields all zeros → 0. By contrast, bitwise OR (|) of a number with its negation gives -1 (all ones).

Q23Assignment value

What is printed?

long wolf = 5;
long coyote = (wolf = 3);
System.out.println(wolf + "," + coyote);
  • A. 5,3
  • B. 3,5
  • C. 3,3
  • D. It does not compile.
Show Answer C · 3,3

(wolf = 3) does two things: it sets wolf to 3, and the assignment expression itself evaluates to 3, which is assigned to coyote. Both end up 3.

Q24Numeric Promotion

What is the data type of z?

short w = 14;
float x = 13;
double y = 30;
var z = w * x / y;
  • A. short
  • B. float
  • C. double
  • D. int
Show Answer C · double

w promotes to int (short in binary arithmetic), then to float to multiply with x; that product promotes to double to divide by y. The widest type wins, so z is a double.

Q25Modulus

What does System.out.println(-7 % 5); print?

  • A. 2
  • B. -2
  • C. 3
  • D. -3
Show Answer B · -2

The result of % takes the sign of the dividend (the left operand). Since -7 is negative, the result is -2. (Compare: 7 % -5 is 2 — the divisor's sign is ignored.)

Q26Negation

Given double t = 1.21;, what does this print?

t = -t;
t = --t;
System.out.println(t);
  • A. 1.21
  • B. -1.21
  • C. -2.21
  • D. 0.21
Show Answer C · -2.21

t = -t makes t = -1.21. Then --t is the decrement operator (not double negation), subtracting 1 → -2.21. To negate twice you'd need -(-t) with parentheses.

Q27Ternary type

Which line does not compile?

int stripes = 7;
// A System.out.print((stripes > 5) ? 21 : "Zebra");
// B int animal = (stripes < 9) ? 3 : "Horse";
  • A. Line A
  • B. Line B
  • C. Both
  • D. Neither
Show Answer B · line B

print() accepts mixed branch types because it converts the result to Object and calls toString() — line A compiles. But line B assigns the result to an int, and "Horse" can't become an int, so the compiler rejects it.

Q28Casting values vs vars

Which line does not compile?

byte hat = 1;
// A byte gloves = 7 * 10;
// B short boots = 2 + 1;
// C short scarf = 2 + hat;
  • A. Line A
  • B. Line B
  • C. Line C
  • D. None — all compile.
Show Answer C · line C

Lines A and B use only literal values, so the compiler can confirm the result fits (70 in a byte, 3 in a short). Line C mixes the literal 2 with the variable hat; both promote to int, and an int can't be stored in a short without a cast.

Quick Revision

  • Precedence: unary > multiplicative (* / %) > additive (+ -) > relational > equality > logical > conditional > ternary > assignment. Only unary, cast, ternary, and assignment go right-to-left. Parentheses must be balanced; [] isn't allowed.
  • Unary: ! only on boolean, -/~ only on numbers; 1≠true, 0≠false. ~n = -n-1. --x is decrement, not double negation. Pre returns the new value, post the original.
  • Promotion: byte/short/char → int in binary arithmetic on variables; integral + floating-point → the float/double type; the widest operand wins.
  • Casting: larger→smaller needs a cast; the cast is unary and binds to one operand. Literal arithmetic that fits compiles; the same on variables may not. Compound operators (+= *= …) auto-cast back to the left type.
  • Assignment value: an assignment is an expression equal to its value — watch = vs == inside conditions.
  • Equality & logic: == compares references for objects, values for primitives; can't mix incompatible types; null == null is true. & | ^ evaluate both sides; && || short-circuit (watch unperformed side effects).
  • instanceof: on null → false; instanceof null or an impossible type → does not compile.
  • Ternary: only one branch runs; branches can differ in type for print(), but assigning the result enforces type compatibility.
