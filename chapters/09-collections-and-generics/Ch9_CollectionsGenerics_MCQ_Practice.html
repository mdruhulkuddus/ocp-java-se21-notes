Q01COLLECTION API

Which statements about the four main interfaces are true? (Choose all that apply.)

  • A. Map implements Collection
  • B. A List allows duplicates and index access
  • C. A Set does not allow duplicates
  • D. Deque extends Queue
Show Answer
ANSWER: B, C, D

A is false — Map does not implement Collection; it's part of the framework but keeps key/value methods. The rest are correct.

Q02GENERICS

What is the type of map?

var map = new HashMap<>();
  • A. Does not compile
  • B. HashMap<Object, Object>
  • C. HashMap<String, String>
  • D. HashMap<?, ?>
Show Answer
ANSWER: B

Q03GENERICS

Which line does not compile?

2: var b = new ArrayList<Integer>();

4: List<Integer> d = new ArrayList<Integer>();

  • B. Line 2

The diamond operator can't be used on the left of a declaration — List<> is illegal. It works only on the right of an assignment.

Q06LIST

Which factory method produces a fixed-size list backed by the array, where set() works but add() throws?

  • A. List.of(array)
  • B. List.copyOf(coll)
  • C. Arrays.asList(array)
  • D. new ArrayList<>(coll)
Show Answer
ANSWER: C

Arrays.asList() is backed by the array: you can replace elements with set(), but adding or removing throws UnsupportedOperationException. List.of/copyOf are fully immutable.

Q07LIST

What is printed?

var list = new LinkedList<Integer>();
list.add(3); list.add(2); list.add(1);
list.remove(2);
list.remove(Integer.valueOf(2));
System.out.println(list);
  • A. [3]
  • B. [3, 1]
  • C. [2, 1]
  • D. [1]
Show Answer
ANSWER: A

remove(2) takes an int index, deleting the element at index 2 (the 1) → [3, 2]. Then remove(Integer.valueOf(2)) deletes the value 2 → [3].

Q08LIST

For an Integer list, how do remove(int) and remove(Object) behave when the target is missing?

  • A. Both return false
  • B. remove(int) throws IndexOutOfBoundsException; remove(Object) returns false
  • C. Both throw an exception
  • D. remove(int) returns false; remove(Object) throws
Show Answer
ANSWER: B

The index version throws if the index is out of range; the object version simply returns false when no matching element is found.

Q09SET

The values 66, 10, 8 are added in that order. Which implementation prints them as 8, 10, 66?

  • A. HashSet
  • B. LinkedHashSet
  • C. TreeSet
  • D. ArrayList
Show Answer
ANSWER: C

TreeSet keeps elements in natural sorted order. HashSet is arbitrary, LinkedHashSet uses insertion order (66, 10, 8).

Q10SET

What does set.add(66) return the second time 66 is added to a HashSet?

  • A. true

  • D. null

Q11QUEUE/DEQUE

Which Queue methods return a sentinel value (rather than throwing) on failure? (Choose all that apply.)

  • A. offer(e)
  • B. element()
  • C. peek()
  • D. poll()
Show Answer
ANSWER: A, C, D

offer returns a boolean, peek and poll return null when empty. element() (like add and remove) throws on failure.

Q12QUEUE/DEQUE

When using a Deque as a stack (LIFO), which methods do you use?

  • A. offer, poll, peek
  • B. push, pop, peek
  • C. addLast, removeLast, getLast
  • D. add, remove, element
Show Answer
ANSWER: B

A stack adds and removes at the front/top: push(e), pop(), and peek().

Q13QUEUE/DEQUE

Which classes implement the Deque interface? (Choose all that apply.)

  • A. ArrayDeque
  • B. LinkedList
  • C. ArrayList
  • D. HashSet

ArrayDeque and LinkedList both implement Deque. LinkedList also implements List; use ArrayDeque when you don't need List methods.

Q15MAP

After this code, what is printed?

Map<String,String> m = new HashMap<>();
m.put("Jenny", "Bus Tour");
m.put("Tom", null);
m.putIfAbsent("Jenny", "Tram");
m.putIfAbsent("Tom", "Tram");
System.out.println(m.get("Jenny") + "," + m.get("Tom"));
  • A. Tram,Tram
  • B. Bus Tour,Tram
  • C. Bus Tour,null
  • D. Tram,null
Show Answer
ANSWER: B

putIfAbsent skips Jenny (already a non-null value) but updates Tom, whose value was null → Bus Tour,Tram.

Q16MAP

Using a longest-name mapper, what is the value of Tom after these calls?

BiFunction<String,String,String> mapper =
   (v1, v2) -> v1.length() > v2.length() ? v1 : v2;
favorites.put("Tom", "Tram");
favorites.merge("Tom", "Skyride", mapper);

  • B. Skyride
  • C. null
  • D. Key is removed

ANSWER: B

Both values are non-null, so the mapper runs. "Skyride" (7) is longer than "Tram" (4), so the value becomes Skyride.

Q19COMPARATOR

Which statements about Comparable vs Comparator are correct? (Choose all that apply.)

  • A. Comparable.compareTo takes 1 parameter; Comparator.compare takes 2
  • B. Comparable is in java.lang; Comparator is in java.util
  • C. Comparator must be implemented by the class being compared
  • D. Comparator is commonly written as a lambda
Show Answer
ANSWER: A, B, D

C is false — Comparator is defined outside the class (often as a lambda), which is exactly why it's useful for sorting types you don't control.

Q20COMPARATOR

What is wrong with this code?

var byWeight = new Comparator<Duck>() {
   public int compareTo(Duck d1, Duck d2) {
      return d1.weight() - d2.weight();
   }
};
  • A. Nothing — it compiles
  • B. A Comparator must implement compare(), not compareTo()
  • C. Comparator can't be an anonymous class
  • D. compare takes only one parameter
Show Answer
ANSWER: B

The method name is wrong. Comparator's abstract method is compare(T, T); compareTo belongs to Comparable.

Q21COMPARATOR

Which builds a comparator that sorts by species ascending, breaking ties by weight?

  • A. Comparator.comparing(Squirrel::species).reversed()
  • B. Comparator.comparing(Squirrel::species).thenComparingInt(Squirrel::weight)
  • C. Comparator.comparingInt(Squirrel::weight).thenComparing(Squirrel::species)
  • D. Comparator.naturalOrder()
Show Answer
ANSWER: B

Start with comparing(species), then chain thenComparingInt(weight) for the tie-breaker. Option C reverses the priority.

Q23SORTING

What is printed?

List<Integer> list = Arrays.asList(6,9,1,8);
Collections.sort(list);
System.out.println(Collections.binarySearch(list, 3));
  • A. 3
  • B. -2
  • C. 1
  • D. -1

Q24SORTING

Rabbit does not implement Comparable. What happens in each case?

// Case 1 Collections.sort(rabbitList);
// Case 2 new TreeSet<Rabbit>().add(new Rabbit(1));
  • A. Both fail to compile
  • B. Case 1 doesn't compile; Case 2 throws ClassCastException at runtime
  • C. Both throw at runtime
  • D. Both compile and run fine
Show Answer
ANSWER: B

Collections.sort requires Comparable at compile time. A TreeSet doesn't check at compile time, so it throws ClassCastException on the first add.

Q25SEQUENCED

Which is true about Java 21 sequenced collections? (Choose all that apply.)

  • A. A HashMap cannot be used as a SequencedMap
  • B. TreeSet.addLast() throws UnsupportedOperationException
  • C. An ArrayList is a SequencedCollection
  • D. Sequenced means the same thing as sorted
Show Answer
ANSWER: A, B, C

D is false — sequenced means a well-defined encounter order, not necessarily sorted. HashMap has no order (so it can't be a SequencedMap), and you can't force position in a sorted TreeSet.

Q26GENERICS

Why doesn't this class compile?

public class LongTailAnimal {
   protected void chew(List<Object> input) {}
   protected void chew(List<Double> input) {}
}
  • A. List can't be a parameter type
  • B. After type erasure both become chew(List)
  • C. Double is not a subtype of Object
  • D. Methods can't be protected
Show Answer
ANSWER: B

Type erasure removes the generic type, so both methods reduce to chew(List) — an illegal duplicate. (chew(List<Object>) + chew(ArrayList<Double>) would be fine.)

Q27GENERICS

Mammal declares List<CharSequence> play(). Which overrides compile? (Choose all that apply.)

  • A. ArrayList<CharSequence> play()
  • B. List<String> play()
  • C. List<CharSequence> play()
  • D. Collection<CharSequence> play()
Show Answer
ANSWER: A, C

Return types may be covariant (a subtype), but the generic parameter must match exactly. ArrayList<CharSequence> (subtype of List) works; List<String> fails because String ≠ CharSequence; Collection is a supertype, not a subtype.

5: public static T noGood(T t) { return t; }
  • A. Line 3
  • B. Line 4
  • C. Line 5
  • D. None — all compile
Show Answer
ANSWER: C

Line 5 uses T without declaring the formal type parameter <T> before the return type. Lines 3 and 4 declare it correctly.

Q29WILDCARDS

What is the result?

List<? extends Bird> birds = new ArrayList<Bird>();
birds.add(new Sparrow());
  • A. Compiles and adds the Sparrow
  • B. Does not compile — upper-bounded lists are effectively read-only
  • C. Throws at runtime
  • D. Compiles only if Sparrow extends Bird
Show Answer
ANSWER: B

With ? extends Bird, Java can't know the exact list type (it could be List<Sparrow> or another), so adding is disallowed. Upper-bounded and unbounded wildcards are logically immutable.

Q30WILDCARDS

Which lines compile? (Choose all that apply.)

A: List<? super IOException> e = new ArrayList<Exception>();
B: e.add(new Exception());
C: e.add(new IOException());
D: e.add(new FileNotFoundException());
  • A. Line A
  • B. Line B
  • C. Line C
  • D. Line D
Show Answer
ANSWER: A, C, D

The list could be List<IOException>, so a plain Exception (a supertype) can't be added (line B fails). IOException and its subclass FileNotFoundException are always safe to add.

Quick Revision

  • Diamond/var: diamond infers left, var infers right; var x = new HashMap<>() → <Object,Object>.
  • Lists: Arrays.asList = fixed-size backed array; List.of/copyOf = immutable; remove(int) by index vs remove(Object) by value.
  • Order: Hash* unordered, Linked* insertion, Tree* sorted (no null keys/values).
  • Queue/Deque: throwing vs sentinel pairs; stack = push/pop/peek.
  • Map: merge runs the function only with two real values (returns null → remove); map.contains() doesn't compile.
  • Sorting: Comparable = compareTo/1 param/in class; Comparator = compare/2 params/lambda. Non-Comparable: sort won't compile, TreeSet throws at runtime.
  • Generics: erasure blocks generic-type overloading and needs exact generic match for covariant overrides; declare <T> before the return type.
  • Wildcards: ? extends/unbounded are read-only; ? super lets you add the bound type or its subclasses.
