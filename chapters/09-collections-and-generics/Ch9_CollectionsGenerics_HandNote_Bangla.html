  • Bounded wildcard (?, ? extends, ? super) প্রয়োগ।

SECTION 01

Collection API ও Generics বেসিক

Java Collections Framework (java.util-এ) এর চারটি প্রধান interface আছে। Map, Collection implement করে না — এটি framework-এর অংশ কিন্তু নিজস্ব key/value method রাখে।

Interfaceঅর্থ
Listordered, duplicate অনুমোদিত, int index দিয়ে access
Setকোনো duplicate entry নেই
Queue / Dequeprocessing-এর জন্য ক্রমে রাখে; Deque উভয় প্রান্তে অনুমতি দেয়
Mapkey থেকে value; duplicate key নেই

Generics শর্টহ্যান্ড: diamond ও var

Generics হলো parameterized type। Diamond operator <> বাঁ দিক থেকে infer করে এবং কেবল assignment-এর ডান পাশে অনুমোদিত; var ডান দিক থেকে infer করে। দুই পাশে type দেবেন না।

List<Integer> list = new ArrayList<>();        // diamond infers from left
var list = new ArrayList<Integer>();            // var infers from right

List<> bad = new ArrayList<Integer>();          // DOES NOT COMPILE (diamond on left)
var map = new HashMap<>();                       // compiles -> HashMap<Object,Object>
Exam Trap — var + diamond একসাথে

var map = new HashMap<>(); compile হয়, কিন্তু infer করার মতো কিছু না থাকায় type হয় HashMap<Object, Object>।

List<Integer>-এ null add করা বৈধ, কিন্তু primitive-এ unbox করলে throw করে: int h = heights.get(0); → NullPointerException।

SECTION 02

List Interface

একটি List ordered ও duplicate অনুমোদিত, index-ভিত্তিক access সহ। দুটি implementation জানতে হবে:

ArrayList

  • Resizable array; index দিয়ে constant-time lookup।
  • লেখার চেয়ে পড়া বেশি (বা সমান) হলে ভালো।
  • সন্দেহ হলে "default" পছন্দ।

LinkedList

  • List ও Deque দুটোই implement করে।
  • প্রান্তে add/remove/access constant-time।
  • যেকোনো index-এ access linear time।

Factory method

MethodফলাফলReplace?Add/Delete?
Arrays.asList(varargs)array-backed fixed-size listহ্যাঁনা
List.of(varargs)immutable listনানা
List.copyOf(collection)immutable copyনানা
Key Rule — backed array বনাম immutable

Arrays.asList() array দ্বারা backed: array বদলালে list বদলায় এবং set() array-কে আবার বদলায়। List.of() ও List.copyOf() immutable — যেকোনো set(), add(), বা remove() UnsupportedOperationException throw করে। (আর Arrays.asList().add()-ও throw করে, কারণ size fixed।)

List method ও overloaded remove() ফাঁদ

inherited Collection method ছাড়াও: add(int, E), get(int), indexOf, lastIndexOf (না পেলে -1), remove(int), set(int, E), replaceAll(UnaryOperator), ও sort(Comparator)। range-এর বাইরে index-এ set() IndexOutOfBoundsException throw করে।

var list = new LinkedList<Integer>();
list.add(3); list.add(2); list.add(1);   // [3, 2, 1]
list.remove(2);                  // INDEX 2 মুছে -> [3, 2]
list.remove(Integer.valueOf(2));  // VALUE 2 মুছে -> [3]
Exam Trap — remove(int) বনাম remove(Object)

Integer list-এ remove(int) index দিয়ে মুছে এবং না পেলে IndexOutOfBoundsException throw করে; remove(Object) value দিয়ে মুছে এবং না পেলে false দেয়। remove(Integer.valueOf(2)) value version-কে বাধ্য করে।

var nums = Arrays.asList(1, 2, 3);
nums.replaceAll(x -> x*2);   // [2, 4, 6]

String[] arr = list.toArray(new String[0]);  // typed array; size 0 দিলে Java নিজে size দেয়

argument ছাড়া toArray() Object[] ফেরত দেয়। ফেরত array একটি copy — list clear করলে array-তে প্রভাব পড়ে না।

SECTION 03

Set, Queue ও Deque

Set implementation

ClassStorage / order
HashSethash table; hashCode() ব্যবহার; arbitrary order
LinkedHashSethash table + insertion (encounter) order
TreeSetsorted tree; সবসময় sorted; add/remove ধীর

66, 10, 8 add করা হলে

  • HashSet → 66,8,10 (arbitrary)
  • LinkedHashSet → 66,10,8 (insertion)
  • TreeSet → 8,10,66 (sorted)

Immutable set আসে Set.of(...) ও Set.copyOf(...) থেকে। সমতা equals() দিয়ে; hashCode() bucket বেছে নেয়।

Queue ও Deque

একটি Queue হলো FIFO। LinkedList ও ArrayDeque দুটোই Deque implement করে (যা Queue extend করে)। List method না লাগলে ArrayDeque ব্যবহার করুন।

প্রতিটি operation-এর দুই রূপ

প্রতিটি ক্ষমতার একটি version ব্যর্থতায় throw করে, আরেকটি sentinel (null/false) ফেরত দেয়:

  • পেছনে add: add(e) (throws) · offer(e) (boolean দেয়)
  • সামনে read: element() (throws) · peek() (null দেয়)
  • সামনে থেকে remove: remove() (throws) · poll() (null দেয়)
Deque উদ্দেশ্যসামনের methodপেছনের method
AddaddFirst / offerFirstaddLast / offerLast
Read (peek)getFirst / peekFirstgetLast / peekLast
RemoveremoveFirst / pollFirstremoveLast / pollLast
Remember — Deque-কে stack হিসেবে (LIFO)

Stack হিসেবে method গুলো হলো push(e) (front/top-এ add), pop() (front/top থেকে remove), ও peek()। FIFO queue হলো মানুষের লাইন; LIFO stack হলো প্লেটের স্তূপ।

SECTION 04

Map Interface

একটি Map key/value pair রাখে এবং Collection extend করে না। Generics হলো K (key) ও V (value)।

ClassOrder
HashMapkey-এর hashCode(); arbitrary order
LinkedHashMapinsertion (encounter) order
TreeMapkey অনুযায়ী sorted; null key নেই
 
Map.of("key1", "value1", "key2", "value2");     // up to 10 pairs
Map.ofEntries(Map.entry("k1", "v1"), Map.entry("k2", "v2"));  // নিরাপদ
Map.copyOf(map);

যে Map method জানতে হবে

Exam Trap — contains() Map-এ নেই

map.contains("lion") compile হয় না — contains() একটি Collection method। Map-এ আছে containsKey() ও containsValue()।

merge() আচরণ

merge(key, value, fn) যা করে

  • Key absent, বা null mapped → value সেট (function কল হয় না)।
  • Key-এর non-null value আছে → fn(old, value) সেট।
  • Function null দিলে → key মুছে যায়।

সবচেয়ে লম্বা নামের mapper

  • favorites.put("Jenny","Bus Tour"); merge("Jenny","Skyride", longest) → Bus Tour রাখে (লম্বা)।
  • favorites.put("Tom","Tram"); merge("Tom","Skyride", longest) → Skyride (লম্বা)।

Map-এ forEach দুটি parameter নেয়: map.forEach((k, v) -> ...)। entrySet() Map.Entry<K,V> object দেয় যাতে getKey()/getValue() থাকে।

Sorting, Searching ও Sequenced Collections

Comparable বনাম Comparator

Method নামcompareTo(T)compare(T, T)

Key Rule — compareTo() রিটার্ন মান

সমান হলে 0, this ছোট হলে negative, বড় হলে positive। id - d.id ascending; d.id - id descending।

Exam Trap — ভুল method নাম

Comparator-কে compare() implement করতে হয়, compareTo() নয়। new Comparator<Duck>(){ public int compareTo(...){} } compile হয় না। দুটোই functional interface (এক abstract method), কিন্তু Comparable-এর জন্য lambda অর্থহীন।

Comparator তৈরি

Comparator<Duck> byWeight = Comparator.comparing(Duck::weight);
Comparator<Squirrel> c = Comparator.comparing(Squirrel::species)
                                   .thenComparingInt(Squirrel::weight);
var desc = Comparator.comparing(Squirrel::species).reversed();

Static builder: comparing, comparingInt/Double/Long, naturalOrder(), reverseOrder()। Chainable default method: reversed(), thenComparing, thenComparingInt/Double/Long।

Sorting ও binarySearch

Exam Trap — sort-এ Comparable লাগে

non-Comparable type-এ Collections.sort(rabbits) compile হয় না। কিন্তু TreeSet-এ non-Comparable element add করা compile হয় ও runtime-এ (প্রথম add-এ) ClassCastException throw করে। উভয় ক্ষেত্রে Comparator পাস করে ঠিক করুন।

List<Integer> list = Arrays.asList(6,9,1,8);
Collections.sort(list);                       // [1, 6, 8, 9]
Collections.binarySearch(list, 6);            // 1  (index-এ পাওয়া গেল)
Collections.binarySearch(list, 3);            // -2  (-(insertion point)-1)

binarySearch-এর জন্য sorted list লাগে; পাওয়া গেলে index দেয়, নইলে -(insertion point) - 1। list-এর order-এর সাথে না মেলা comparator দিয়ে search করলে ফল undefined।

Sequenced collections (Java 21)

SequencedCollection, SequencedSet, ও SequencedMap-এর well-defined encounter order আছে (sorted হতেই হবে এমন নয়)। Method-এ আছে addFirst/addLast, getFirst/getLast, removeFirst/removeLast, ও reversed(); map-এ আছে firstEntry/lastEntry, pollFirstEntry/pollLastEntry, putFirst/putLast।

Remember — Hash type sequenced নয়

HashSet ও HashMap উল্লেখযোগ্য ব্যতিক্রম যাদের encounter order নেই, তাই HashMap SequencedMap হতে পারে না (compile হয় না)। TreeSet-এ addLast() UnsupportedOperationException throw করে — sorted structure-এ position চাপানো যায় না। Sorted set-এ null value নিষেধ; sorted map-এ null key নিষেধ।

একটি unmodifiable view (Collections.unmodifiableList/Set/Map/Collection) view দিয়ে বদলানো যায় না, কিন্তু underlying collection-এর পরিবর্তন view-তে দেখা যায়।

SECTION 06

Generics ও Wildcards

angle bracket-এ formal type parameter ঘোষণা করুন। Convention: E element, K key, V value, N number, T type, S/U/V আরও বেশি।

public class Crate<T> {
   private T contents;
   public T lookInCrate() { return contents; }
   public void packCrate(T contents) { this.contents = contents; }
}
public record CrateRecord<T>(T contents) {}   // generic record

Type erasure

compile time-এ compiler প্রতিটি T-কে Object (বা তার bound) দিয়ে প্রতিস্থাপন করে ও cast যোগ করে। একটিই class file থাকে। এর প্রভাব overloading ও overriding-এ পড়ে।

Exam Trap — erasure overloading ভাঙে

chew(List<Object>) ও chew(List<Double>) দুটো একসাথে রাখা যায় না — erasure-এর পরে দুটোই chew(List) → compile হয় না। parent/child-এও একই। কিন্তু chew(List<Object>) ও chew(ArrayList<Double>) একসাথে থাকতে পারে (ভিন্ন raw type)।

Exam Trap — covariant return, exact generic

overriding method-এর return type subtype হতে পারে, কিন্তু generic parameter হুবহু মিলতে হবে। List<CharSequence> play()-কে override করে ArrayList<CharSequence> play() compile হয়; List<String> play() হয় না (String ≠ CharSequence), যদিও CharSequence sleep()-কে override করে String sleep() ঠিক আছে।

Generic method

public static <T> void prepare(T t) { }
public static <T> Crate<T> ship(T t) { return new Crate<T>(); }
public static T noGood(T t) { return t; }   // DOES NOT COMPILE (<T> নেই)

formal type parameter <T> return type-এর ঠিক আগে বসে (class থেকে inherited না হলে)। method-এর T class-এর T থেকে স্বাধীন।

Wildcards

BoundSyntaxউদাহরণ
Unbounded?List<?> a = new ArrayList<String>();
Upper bound? extends typeList<? extends Number> a = new ArrayList<Integer>();
Lower bound? super typeList<? super Exception> a = new ArrayList<Object>();
Key Rule — upper/unbounded read-only

List<String>-কে List<Object>-এ assign করা যায় না (type safety)। "যেকোনো কিছুর list"-এর জন্য List<?> ব্যবহার করুন। unbounded বা upper-bounded wildcard-এ list logically immutable: List<? extends Bird>-এ birds.add(new Sparrow()) compile হয় না। Upper bound class ও interface উভয়ের জন্য extends ব্যবহার করে।

Key Rule — lower bound add করতে দেয়

List<? super String> একটি List<String> বা যেকোনো supertype list গ্রহণ করে, তাই String add করা নিরাপদ। List<? super IOException>-এ IOException বা তার subclass FileNotFoundException add করা যায়, কিন্তু সাধারণ Exception নয় (এটি আসলে List<IOException> হতে পারে)।

List<? extends B> l4 = new ArrayList<A>();    // DOES NOT COMPILE (A, B/sub নয়)
List<?> l6 = new ArrayList<? extends A>();    // DOES NOT COMPILE (wildcard instantiate করা যায় না)
<B extends A> B third(List<B> list){ return new B(); }  // DOES NOT COMPILE (type param new করা যায় না)
<X> void fifth(List<X super B> list){ }           // DOES NOT COMPILE (wildcard-এ ? লাগে)
Remember — generics দিয়ে যা করা যায় না

erasure-এর কারণে যা করা যায় না: new T(), new T[], parameterized type-এ instanceof, primitive-কে type argument, type parameter-এর static field, বা T catch করা।

সংক্ষিপ্ত রিভিউ

  • চার interface: List (ordered, dup), Set (no dup), Queue/Deque (processing order), Map (key/value, Collection নয়)।
  • Diamond <> বাঁ থেকে infer; var ডান থেকে; var x = new HashMap<>() দেয় <Object,Object>।
  • Arrays.asList = fixed-size, backed array (set ok, add/remove throw); List.of/copyOf = পূর্ণ immutable।
  • remove(int) index দিয়ে (না পেলে throw); remove(Object) value দিয়ে (false দেয়)। Integer.valueOf(n) value version বাধ্য করে।
  • HashSet/HashMap unordered, Linked* insertion order, Tree* sorted (null key/value নিষেধ)।
  • Queue/Deque: throw বনাম sentinel জোড়া (add/offer, element/peek, remove/poll); stack-এ push/pop/peek।
  • merge: দুটি real value থাকলেই function চলে; null দিলে remove। putIfAbsent null mapping update করে। map.contains() compile হয় না।
  • Comparable: compareTo, ১ param, java.lang, class-এ। Comparator: compare, ২ param, java.util, সাধারণত lambda।
  • non-Comparable-এ Collections.sort compile হয় না; TreeSet add runtime-এ ClassCastException throw করে।
  • Type erasure T-কে Object করে; generic type দিয়ে overloading আটকায়; covariant override-এ generic হুবহু মিলতে হয়।
  • Wildcard: ? / ? extends (read-only) / ? super (add করা যায়)। Generic method return type-এর আগে <T> ঘোষণা করে।
